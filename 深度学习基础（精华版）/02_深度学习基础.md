# 深度学习基础概念

<h2 id="1.机器学习与深度学习有哪些区别？">1.机器学习与深度学习有哪些区别？</h2>

## 面试问题：机器学习与深度学习有哪些区别？

**难度评分：⭐⭐ (2/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

机器学习是广义的方法论集合，深度学习是机器学习的一个子集，特指基于深层神经网络的特定实现路径。

**核心区别对照表**

| 对比维度 | 机器学习 | 深度学习 |
| :--- | :--- | :--- |
| **特征工程** | **人工设计为主**（决定模型上限） | **自动提取**（端到端学习） |
| **数据需求量** | **小样本**也能工作（几百上千条） | **海量数据**驱动（通常需百万级） |
| **硬件依赖性** | CPU 即可训练 | **依赖 GPU/TPU** 并行计算 |
| **可解释性** | **白盒**（决策树、回归系数直观） | **黑盒**（难以解释中间层权重含义） |
| **训练时间** | 短（秒~分钟级） | 长（小时~天级） |
| **人工干预** | 高（需领域专家构建特征） | 低（原始数据直接输入） |
| **典型算法** | SVM、随机森林、XGBoost、K-Means | CNN、RNN、Transformer、GAN |


**关键差异点**

(1)特征工程：人工抽取 vs 自动表征学习，这是两者最本质的哲学差异。

- **机器学习（以 SVM 分类图片为例）：**
    - 工程师需要先写代码提取特征：**HOG**（方向梯度直方图）、**SIFT**（尺度不变特征变换）、**颜色直方图**。
    - 然后将这些人工定义的数字向量喂给 SVM 分类。
    - **弊端：** 特征提取器是固定的，不参与训练优化。如果人工选错了特征，模型再强也没用。

- **深度学习（以 CNN 分类图片为例）：**
    - 输入**原始像素矩阵**。
    - 网络浅层自动学会提取边缘、纹理；中层学会提取形状、部件；深层学会提取语义概念。
    - **核心优势：** **特征提取器本身也是通过反向传播学习出来的参数**，随着 Loss 下降，特征表示会变得越来越适合当前任务。

(2)数据量与性能的边际曲线：这是一个关键的工程选型依据，**传统机器学习在面对海量数据时，模型容量有限，会出现欠拟合饱和；而深度学习的伸缩性允许它利用更多数据持续提升精度**

| 数据量规模 | 推荐方案 | 原因 |
| :--- | :--- | :--- |
| **< 1万条表格数据** | **XGBoost / 随机森林**（机器学习） | 深度学习参数过多，极度过拟合。 |
| **1万 ~ 10万条** | 两者皆可，调参决定 | 机器学习可能上限更低但更稳。 |
| **> 100万条（图像/文本）** | **深度学习** 完胜 | 传统 ML 性能会**过早进入平台期**，增加数据不再提升；DL 性能随数据量**对数增长**。 |


(3)硬件与算力逻辑：

- **机器学习（SVM/XGBoost）：** 大多是**凸优化问题**或**统计计算**，CPU 的复杂逻辑控制能力足够应对，且内存占用可控。
- **深度学习：** 核心计算是**矩阵乘法**，这天然适合 GPU 成千上万个简单核心同时计算。如果没有 GPU，深层的反向传播计算量会大到不可接受。

 可解释性与工业落地偏好
- **金融风控、医疗诊断辅助：** 监管和医生往往**更信任机器学习**（如逻辑回归的系数权重、决策树的分裂路径）。因为模型需要解释“为什么拒绝贷款”或“为什么判断为高风险结节”。
- **图像识别、自然语言理解：** 追求极致准确率时，只能牺牲可解释性，**必须上深度学习**。

主要可以总结为三个维度来阐述它们的区别：

> **第一，是包含关系。** 深度学习是机器学习中**神经网络**分支的延伸，属于特化子集。
> 
> **第二，也是最核心的区别，在于特征工程。** 传统机器学习需要**专家手工设计特征**，这决定了它的上限；而深度学习能做到**端到端的特征自动抽取**，浅层抽细节，深层抽语义。
> 
> **第三，是应用场景的选择逻辑。** 在**结构化表格数据、小样本、强解释性**场景下（比如金融评分卡），机器学习中的 **XGBoost** 依然占据统治地位；而在**非结构化数据、大样本、追求感知精度**的场景下（比如人脸识别、ChatGPT），**深度学习**无可替代。
总结来说，不能说深度学习全面优于机器学习，它们是工业界解决不同问题的互补工具。**



<h2 id="2.介绍一下神经网络，它是由哪些模块组成，有哪些常用模型结构？">2.介绍一下神经网络，它是由哪些模块组成，有哪些常用模型结构？</h2>

## 面试问题：介绍一下神经网络，它是由哪些模块组成，有哪些常用模型结构？

**难度评分：⭐⭐ (2/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

神经网络（Neural Network） 是一种受人脑神经元连接方式启发的计算模型，通过大量简单的处理单元（神经元）相互连接，以学习输入数据与输出目标之间的复杂映射关系。它本质上是一个可微分的、由参数构成的非线性函数逼近器。

#### 核心组成模块


模块|核心作用|常见类型 / 关键参数|面试高频考点|
|---|---|---|---|
|**神经元（感知机）**|最基本计算单元，接收输入并输出结果|线性变换 \+ 偏置 \+ 激活函数|感知机的局限性（无法解决异或问题）|
|**网络层**|按功能组织神经元，实现特征变换|输入层、隐藏层、输出层|深度 vs 宽度的权衡、过拟合与层数的关系|
|**激活函数**|引入非线性，使网络能拟合复杂函数|Sigmoid、Tanh、ReLU、LeakyReLU、GELU、Swish|各激活函数优缺点、为什么 ReLU 解决梯度消失、GELU 在 Transformer 中流行的原因|
|**损失函数**|衡量模型预测与真实值的差距，指导优化方向|交叉熵（分类）、MSE（回归）、KL 散度（分布匹配）、对比损失、扩散损失|不同任务的损失函数选型、交叉熵与 MSE 的适用场景|
|**优化器**|更新网络参数，最小化损失函数|SGD、Adam、AdamW、RMSprop|AdamW 与 Adam 的区别（权重衰减）、学习率调度策略、梯度消失 / 爆炸的原因与解决方法|

**AIGC时代专属模块**：注意力机制、归一化层（LayerNorm）、残差连接、位置编码、采样器（扩散模型）、生成器 / 判别器（GAN）

### 常用模型结构

####  传统深度学习时代（判别式为主）
|模型结构|提出时间|核心思想|代表模型|典型应用|
|---|---|---|---|---|
|多层感知机（MLP）|1986 年|全连接层堆叠，逐层特征变换，通过激活函数引入非线性，实现简单的特征映射与拟合|BP神经网络、全连接神经网络|简单分类、回归、特征提取|
|卷积神经网络（CNN）|1998 年（LeNet）|利用卷积操作提取局部空间特征，通过参数共享减少模型参数，降低过拟合风险，提升训练效率|LeNet、AlexNet、VGG、ResNet|图像分类、目标检测、语义分割|
|循环神经网络（RNN/LSTM/GRU）|1997 年（LSTM）|处理序列数据，通过循环结构保留历史信息，解决时序依赖问题，适配变长序列输入|Elman RNN、LSTM、GRU、Seq2Seq（基础版）|文本分类、机器翻译、语音识别|

####  AIGC 时代（生成式为主）
|模型结构|提出时间|核心思想|代表模型|典型 AIGC 应用|
|---|---|---|---|---|
|Transformer|2017 年|基于自注意力机制，并行处理序列，捕捉长距离依赖；搭配位置编码、前馈网络（FFN）、层归一化（LayerNorm），彻底取代传统RNN/CNN在序列、图像任务中的主导地位，提升训练效率。|BERT、GPT 系列、LLaMA、Qwen、ViT|大语言模型、文本生成、机器翻译、图像理解|
|生成对抗网络（GAN）|2014 年|生成器与判别器博弈训练，生成器负责生成逼真数据，判别器负责区分真实数据与生成数据，通过对抗过程不断优化模型，提升生成质量。|StyleGAN、CycleGAN、Pix2Pix|图像生成、风格迁移、人脸生成|
|变分自编码器（VAE）|2013 年|基于概率生成，学习数据的潜在分布，通过编码器将数据映射到潜在空间，再通过解码器从潜在空间重构数据，实现数据生成与压缩。|VQ\-VAE、VQ\-VAE\-2|图像生成、语音合成、数据压缩|
|扩散模型（Diffusion）|2015 年（基础）、2020 年（DDPM）|逐步加噪再去噪的生成过程，前向过程向原始数据逐步添加高斯噪声，反向过程训练噪声预测网络（常用U\-Net或Transformer架构），训练稳定、生成质量高。|Stable Diffusion、DALL\-E 3、Sora、DDPM|文生图、文生视频、图像编辑|
|多模态融合结构|2021年（CLIP为代表）|核心是跨注意力（cross\-attention，实现不同模态数据的关联映射）、对比学习（如CLIP的图文对比），打破单一模态局限，实现多模态协同生成与理解。|CLIP、Flamingo、ImageBind、GPT\-4V|图文生成、跨模态检索、多模态对话|
|MoE（混合专家模型）|2017年（基础）、2023年（Mixtral为代表）|核心是稀疏激活，将模型分为多个独立“专家网络”，输入仅激活部分相关专家，无需全量参数参与计算，降低大模型训练与推理成本，支持规模扩展。|Mixtral 8x7B、GPT\-4（部分采用）、PaLM\-E、Qwen\-MoE|大语言模型高效推理、多模态生成、大规模模型部署|
|Hybrid结构（CNN\+Transformer结合）|2020年（Swin Transformer为代表）|融合CNN的局部特征提取优势（高效捕捉图像、音频局部细节）与Transformer的长距离依赖捕捉能力，兼顾效率与性能，解决纯Transformer局部特征提取冗余问题。|ViViT、CoaT、Swin Transformer、ConvNeXt\-Transformer|视频生成、图像编辑、多模态特征提取|


<h2 id="3.模型泛化能力、拟合能力、表达能力分别是什么？三者有什么关系？">3.模型泛化能力、拟合能力、表达能力分别是什么？三者有什么关系？</h2>

## 面试问题：3.模型泛化能力、拟合能力、表达能力分别是什么？三者有什么关系？

**难度评分：⭐⭐⭐ \(3/5\)  \|  考察频率：⭐⭐⭐⭐ \(4/5\)**

#### 三者定义：

**表达能力**

    - 定义：模型**理论上能够拟合的函数空间的大小与丰富程度**，是模型能**学会什么的**绝对上限。

    - 核心决定因素：仅由模型本身的固有属性决定，包括参数量、网络深度 / 宽度、架构设计（如注意力机制）、激活函数类型等。

    - 示例：10B 参数的 GPT\-3\.5 表达能力远强于 1B 参数的模型，能够拟合更复杂的人类语言分布和逻辑规律。

**拟合能力**

    - 定义：模型**在训练集上的实际学习效果**，是模型**实际学会**了多少训练数据中的规律的能力，通常用训练误差衡量

    - 核心决定因素：是表达能力的子集，受训练时长、优化器选择、数据量、数据质量、学习率调度等训练过程因素影响。

    - 示例：同一个模型，训练 100 轮通常比训练 10 轮的拟合能力更强，表现为训练集损失更低、准确率更高。

**泛化能力**

    - 定义：模型**在未见过的测试集 / 真实分布新数据上的表现**，是模型**举一反三**的能力，通常用测试误差或泛化误差衡量

    - 核心地位：这是所有 AI 模型的**最终核心目标**，在 AIGC 领域直接决定生成内容的通用性、合理性、鲁棒性和商业价值。

    - 核心判定指标：训练集与测试集的性能差距（泛化 Gap）、分布外（OOD）数据表现。

在AIGC领域扩散模型、LLM、VAE等生成模型中，表达能力决定了生成分布的复杂程度；拟合能力体现为训练集上的重建/生成质量；泛化能力则表现为对prompt/条件的鲁棒响应及新样本的多样性/真实性。
  
#### 三者的核心关系：

**表达能力是拟合能力的基础**：若模型表达能力不足（如过于简单的线性层），即使训练数据再多也无法充分拟合（欠拟合），在训练集和测试集表现都差。表达能力过剩则存在**过拟合风险**，模型有能力记住训练数据中的所有细节，包括噪声和特异性特征，这会导致模型在未见数据表现差。

**泛化能力依赖表达能力与数据分布、训练策略的匹配**：在理想情况下，如果模型的表达能力刚好能拟合数据的真实分布，同时不过度拟合训练噪声，此时可同时实现强拟合与强泛化。但是大多数情况是在 "足够的表达能力拟合数据" 和 "控制过拟合保证泛化" 之间寻找最优平衡点。

传统机器学习理论认为，模型表达能力过强时，能够完美拟合训练集，但泛化能力会随之下降，即发生过拟合。然而，大模型时代出现了双下降（Double Descent）现象：在过参数化区域（参数量远大于样本量），即使训练损失已降至零，继续扩大模型规模或增加训练数据，测试误差反而会再次下降，泛化能力持续提升。这一现象打破了偏差-方差权衡的传统直觉，被称为“现代过拟合悖论”。
在AIGC中，泛化能力的评价维度更为丰富：不仅包括传统的OOD准确率，还涉及幻觉率、长文本逻辑一致性、跨领域迁移能力、指令遵循等。大模型正是利用双下降性质，通过规模化（Scaling）在保持极低训练损失的同时，持续提升泛化能力
