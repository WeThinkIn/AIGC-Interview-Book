# 《三年面试五年模拟》之开放性问题高频考点

开放性问题不是面试里的“闲聊题”，而是判断候选人是否具备长期AI行业认知的关键入口。技术题考察的是知识点掌握，开放性问题考察的是对技术周期、业务价值、组织协作、职业成长和行业趋势的综合判断。

Rocky认为，AI行业越往后发展，越不能只停留在“会不会某个模型、熟不熟某个框架”。真正能穿越周期的能力，是理解技术为什么出现、业务为什么需要、产品为什么成立、公司为什么能活、个人为什么能持续成长。

本板块围绕三类问题构建：

<div align="center">

| 章节 | 核心问题 | 适合训练的能力 |
|---|---|---|
| [AI行业发展趋势问题](AI行业发展趋势问题.md) | 如何判断AI技术周期、公司价值、落地方向和跨周期技术 | 行业判断、技术趋势、周期认知 |
| [AI业务场景问题](AI业务场景问题.md) | 如何把AI技术放进真实业务、客户流程和商业闭环 | 业务理解、产品落地、ROI意识 |
| [算法工程师个人成长问题](算法工程师个人成长问题.md) | 算法工程师如何从技术执行走向系统负责和跨周期成长 | 职业规划、能力结构、认知复利 |

</div>

建议阅读顺序：

1. 先读 [AI行业发展趋势问题](AI行业发展趋势问题.md)，建立对AIGC、传统深度学习、自动驾驶、AI Agent和未来技术周期的宏观判断。
2. 再读 [AI业务场景问题](AI业务场景问题.md)，把行业判断落到真实客户、真实成本、真实流程和真实交付里。
3. 最后读 [算法工程师个人成长问题](算法工程师个人成长问题.md)，反推个人应该沉淀哪些跨周期能力。

这些问题没有唯一标准答案，但不能没有结构化答案。面试中最好的表达，不是给出一堆口号，而是能说清楚：**问题本质是什么，底层机制是什么，业务后果是什么，长期价值是什么，自己会如何行动。**
