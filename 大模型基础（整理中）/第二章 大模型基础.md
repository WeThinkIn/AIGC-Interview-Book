# 第二章 大语言模型 核心基础知识高频考点


<h1 id="1.介绍一下大语言模型的基础概念与概述">1.介绍一下大语言模型的基础概念与概述</h1>


## 面试问题：大语言模型的基本定义是什么？

**难度评分：⭐ (1/5) | 考察频率：⭐⭐⭐⭐⭐ (5/5)**

### 1. 什么是大语言模型？其核心功能是什么？

**大语言模型**是指**参数规模≥10 Billion**、以**自回归**或**离散扩散**为核心范式，具备**自然语言理解（NLU）**、**自然语言生成（NLG）**能力，可延伸支持代码、图像、音频等多模态 token 生成的**深度学习模型**。（来源：《2025 中国大语言模型（LLM）产业白皮书》）

[!](./imgs/1-1.png)

本章以**文本模态**为核心展开：

a. **核心特征**主要包括：

- **规模**：参数量达数十亿以上，确保模型具备足够的学习容量。

- **架构**：基于 Transformer，支持并行处理与长距离依赖建模。

- **训练**：采用自监督学习，在海量无标注文本上进行预训练。

- **能力**：具备能够理解、生成和操作人类语言的通用能力。

b. **核心功能**主要包括：

- **自然语言理解**：理解人类自然语言意图；

- **自然语言生成**：生成流畅连贯、逻辑严密的文本；

- **知识检索与推理决策**：具备事实性知识的存储与检索能力，能进行常识推理、逻辑推理和上下文学习，可执行问答、翻译、摘要、创作、代码生成等复杂语言任务。

### 2.  词元为什么是大语言模型的基石概念？

**词元（Token）**是指大语言模型**理解人类语言**的**最小单位**。词元的切分**没有固定规律**，通常英文按照词缀、字符或单词，中文按照子词、字符或词语进行切分。（例：“人工智能”可能被分为“人工”和“智能”两个词元）

[!](./imgs/1-2.png)

词元是AIGC时代关键的术语，为**参数量**，**数据量**和现在的**收费模式**铆钉了基础。（例：OpenAI的月度会员为用户调用Codex设定了限额。该限额的收费标准就是百万tokens多少钱）

### 3.为什么称为 “大” 模型？“大” 体现在哪些维度？

之所以称为 “大模型” ，是因为其在**三个关键维度**上均达到了**前所未有的规模**：

[!](./imgs/1-3.png)

- **参数量大**：包含数十亿、千亿乃至万亿级的可学习参数（例：GLM-5.1，约744B）；

- **数据量巨大**：基于互联网、书籍、文档等来源的海量文本进行训练（例：GLM-5.1，28.5T Tokens)；

- **计算量庞大**：训练和推理都需要大规模 GPU/TPU 集群提供的强大算力支持（例：GLM-5.1推理，28个H100 GPU）。

**三大维度**的提升促使大语言模型**涌现**出传统小模型不具备的**核心功能（1）**。

### 4. 大语言模型与传统 自然语言处理（NLP) 模型的核心区别是什么？

- **架构差异**：传统NLP模型采用规则、统计或浅层机器学习，结构简单、参数不及百万；大语言模型基于Transformer，参数达数亿至数千亿。

- **训练范式**：传统NLP模型在小规模人工标注数据上监督学习；大语言模型在海量无标注数据上自监督预训练。

- **能力边界**：传统NLP模型任务专用，按场景训练；大语言模型具备通用语言能力，可以一个模型服务多个场景。

**难度评分：⭐ (1/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

### 1. 当前主流的大语言模型有哪些？

- **GPT、Claude、Gemini** 系列：海外标杆模型，综合能力稳居全球第一梯队。

- **Qwen、GLM、DeepSeek** 系列：国产顶尖模型，架构透明、能力优异，是私有化开发的首选。

[!](./imgs/2-1.png)

以上模型在各大评测榜单中均位列前茅，表现出世界顶尖的通用能力。

其他模型请参见[评测榜单](https://artificialanalysis.ai/)。

### 2. 什么是闭源/开源大模型？什么是通用/垂直大模型？

**模型是否开源（开源属性）**和**模型的能力覆盖范围**是最常见的两类分类标准。

- **按开源属性分**：

| 分类 | 核心定义 | 代表模型 | 核心特点 |
| :--- | :--- | :--- | :--- |
| 闭源大模型 | 不开放模型权重，仅通过 API 接口和在线平台提供服务，无法二次修改和本地部署 | GPT、Claude、Gemini系列| 开箱即用，无需算力投入；缺点定制化自由度低，数据隐私可控性弱 |
| 开源大模型 | 开放模型权重，可免费 / 商用授权下载，支持本地部署、二次微调和架构修改 | Qwen、DeepSeek系列 | 定制化自由度高，数据隐私可控；需要算力和技术投入|

- **按能力范围分**：

| 分类 | 核心定义 | 代表模型 | 核心特点 |
| :--- | :--- | :--- | :--- |
| 通用大模型 | 覆盖全场景语言能力，无明确行业偏向 | GPT-5.x，GLM-5.x | 能力均衡，泛化性强，可作为底座适配各类场景；专业度不够 |
| 开源大模型 | 基于专业数据做专项训练，聚焦解决特定领域的专业问题 | DeepSeek-Coder | 领域专业度高，行业知识精准；场景泛化能力差|

其他分类标准还包括：**参数规模**

- **参数规模**：按参数量近似10B，100B，1T，将大语言模型分为小模型、中型模型、大模型和超大模型。也可以将7B，14B，32B，100B，1T设定为阈值划分标准。

### 3. 什么是基座模型？什么是对话模型？什么是推理模型？

三者是大语言模型研发全流程不同阶段的产物，而**基座模型**是所有**衍生模型**（对话模型、推理模型）的基础。

[!](./imgs/2-3.png)

- **基座模型**：也叫基础模型，是在海量无标注的通用文本预料上，通过自监督学习完成预训练的**通用语言表示模型**。该类模型的核心目标是学习**语言的统计规律、语法结构、语义关系以及世界知识**，从而具备强大的语言理解与生成能力。但输出通常是**文本续写**。（例：DeepSeek-Base）

- **对话模型**：是在基座模型的基础上，经过**指令微调**、基于人类反馈的**强化学习**和**对齐训练**后得到的，专门适配人机对话的衍生模型。（例：ChatGPT、豆包）

- **推理模型**：在基座模型基础上，通过**思维链微调**（让大模型学习思考过程）等技术增强逻辑推理能力的模型。该技术引入”问题 + 推理步骤 + 答案”和包含“错误路径”的示例来强化大模型的逻辑推理能力。（例：GPT-5 thinking）

### 4. 如何使用大语言模型？

根据使用主体、技术能力和应用场景，主流使用方法分为4类。

- **在线对话平台**（零门槛）：直接使用大模型厂商提供的官方在线平台，无需任何技术和算力投入。（例：使用豆包执行办公辅助）

- **API接口调用**（商用落地）：开发者通过代码调用大模型的API接口，将模型能力集成到产品、系统和业务流程中。（例”：使用OpenAI API协助管理数据库）

- **开源模型本地部署**（隐私场所/二次开发）：下载开源大模型的权重文件，在本地部署模型，可完全自主控制模型，保障数据安全。（例：政企私有化部署DeepSeek模型）

- **二次微调/定制化训练**（垂直领域落地）：基于通用基座模型，用行业数据做微调/全参训练，得到适配专业任务的垂直大模型。（例：针对稀有病例分析的医疗大模型）

注：API调用代码，请参照官网案例；本地部署工具，请参考大模型部署章节。


## 面试问题：什么是大语言模型的涌现特性与缩放定律？
**难度评分：⭐⭐ (2/5)  |  考察频率：⭐⭐⭐⭐ ⭐(5/5)**

### 1.  什么是 “涌现能力”？请举例说明。

**涌现能力**（Emergent Abilities）是指模型在**小规模参数**下**完全不具备**、**无法**通过小模型性能**线性外推**，只有当**模型规模**（参数量、训练数据量和计算量）**突破**某个**临界阈值**后，才突然解锁的**复杂能力**。**涌现能力**的出现伴随着**性能曲线的非线性跃升**。（来源：[《Emergent Abilities of Large Language Models》](https://arxiv.org/pdf/2206.07682)）

[!](./imgs/3-1.png)

a. **核心特征**包括：

- **非线性**：能力不是随规模逐步提升，而是突破阈值后从**几乎为0**跃升至**可用水平**；

- **不可预测性**：无法通过先模型的表现，外推出涌现能力何时以何种程度出现；

- **任务特异性**：不同能力的涌现阈值不同，越复杂的任务，需要的规模阈值越高。

b. **典型能力**包括：

- **指令跟随能力**（基础涌现）：理解并执行多步复合的自然语言指令。（例：把这段产品说明改为小红书种草文案，并提炼3个核心卖点，最后翻译成英文）

- **上下文学习能力**（核心涌现）：通过几个示例，大模型就能实现举一反三完成全新的、从未训练过的任务。（例：给 3 个“情感分析” 的标注示例，模型就能直接对全新的文本做情感分类）

- **多步逻辑与数学推理**（高阶涌现）：通过推理数据的训练后，能够完成 IMO 级别的数学竞赛题和复杂的因果逻辑推导。（例：在IMO-AnswerBenchshuju竞赛题求解任务中，GLM 5.1获得83.8的分数）

- **复杂工具调用与工程代码能力**（顶级涌现）：自主调用工具完成 API 对接、数据分析、自动化任务开发等复杂工程工作。（例：在SWE-beanch工业级代码库修复任务中，Claude 4.5 Opus解决76.8%的任务）

### 2. 为什么小模型也开始具备一定的涌现能力？

**涌现能力**已**不再是模型规模的专属特征**，而是取决于**预训练损失是否降至某个特定阈值**以下：只有当损失低于该阈值时，模型才会在某些任务上展现出涌现能力。

小体量的大语言模型（如Gemma 4 31B）涌现能力（在很多任务有远超大体量大语言模型的性能）主要归功于**训练数据的质量提升**（如高质量的合成数据），**模型架构的优化**（如分组注意力降低了模型参数的量级），**训练策略的改革**（如小模型学习大模型的思考过程）。

### 3.  什么是缩放定律？

**缩放定律**是指在**模型架构不变、训练数据质量达标**的前提下，大语言模型的**最终性能**（以测试集的负对数困惑度 Loss 为核心指标，Loss 越低，模型语言建模能力、通用能力越强），与**参数量**（N）、**训练数据量**（D）、**计算量**（C)，三个核心维度，呈稳定的幂律缩放关系。简单来说，**三个维度只要均衡扩大，模型性能就会平滑、可预测地提升，没有明显的上限**。

[!](./imgs/3-3-1.png)

a. OpenAI**原始缩放定律**（来源：[《Scaling Laws for Neural Language Models》](https://arxiv.org/pdf/2001.08361)）

**单变量独立幂律公式**：当**其他两个因子足够充足**、**不成为性能瓶颈**时，**模型损失**与**单一变量**呈严格的**幂律下降关系**，该规律跨越 6 个以上数量级（从千级到十亿级）依然成立。

- **参数量公式**：

$$
L(N)={ ( N_c / N ) } ^ { ( {\alpha}_N) }
$$

$L(N)$：模型的交叉熵损失（单位：nats)，衡量模型的语言性能，值越低表示模型越好。

$N$：模型所有非嵌入层的参数总量（单位：个）。（注：嵌入层是大语言模型的一个模块，参见2.2节）

$N_c$：比例常数（约$8.8 \times 10^13$），仅与数据集有关。

${\alpha}_N$：幂律指数（约0.076）。

- **数据量公式**：

$$
L(D)={ ( D_c / D ) } ^ { ( {\alpha}_D) }
$$

$L(D)$：模型的交叉熵损失，衡量模型的语言性能，值越低表示模型越好。

$D$：训练数据集的数量（单位：tokens）。

$D_c$：比例常数（约$5.4 \times 10^13$），仅与数据集有关。

${\alpha}_D$：幂律指数（约0.095）。

- **计算量公式**：

$$
L(C_{min})={ ( C_{minc} / C_{min} ) } ^ { ( {\alpha}_{mCin}) }
$$

$L(C_{min})$：模型的交叉熵损失（单位：nats)，衡量模型的语言性能，值越低表示模型越好。

$C_{min}$：训练所需的最小计算量（单位：PF-days）。

$C_{minc}$：比例常数（约$ 3.1 \times 10^8$），仅与数据集有关。

${\alpha}_{mCin}$：幂律指数（约0.050）。

**联合缩放公式**：同时考虑**参数量 N **和**数据量 D** 的联合损失公式。模型的最终损失，由**“参数量不足带来的损失”**和**“数据量不足带来的损失”**两部分**加和**而成，两部分是独立，可拆分的。因此，模型性能的瓶颈，由**相对更稀缺的部分决定**。（例：如果参数量很大但数据量不足，损失就会被数据项锁死，无论怎么堆参数都无法提升性能）
$$
L(N, D)= { ( N_c / N ) } ^ { ( {\alpha}_N) } + { ( D_c / D ) } ^ { ( {\alpha}_D) }
$$

该公式进一步表示**模型参数量增长和算力增长的0.73次方成正比**，**数据量增长和算力增长的0.27次方成正比**，错误地导致了”参数竞赛“（模型越做越大，数据增长缓慢）。

$$
N_{opt} \propto C^{0.73}, D_{opt} \propto C^{0.27}
$$

b. DeepMind**修正缩放定律**（来源：[《Training Compute-Optimal Large Language Models》](https://arxiv.org/pdf/2203.15556)）

DeepMind团队发现OpenAI 的原始缩放定律存在核心偏差，导致当时几乎所有大模型都处于 **“参数过大、数据不足、严重欠训练”的状态**，算力投入的性价比极低。修正后的损失公式如下：

$$
L(N, D) = L_{\infty} + { ( A / N_{dm}^{\alpha} ) } + { ( B / D^{\beta})}
$$

$L_{\infty}$：不可约损失，指无限算力，无限数据下能达到的理论最低损失。

$A$、$B$：拟合常数（A=406.4，B=410.7）。

$alpha$、$beta$：幂律指数（$alpha=0.34$，$beta=0.28$）。

$N_{dm}$：所有可训练参数的总和（包括嵌入层）。

修正后的损失公式意味着在**固定算力预算**下，**最小化损失**的最优解是**参数量与数据量近乎同步缩放**。而所需的总计算量公式则优化为：

$$
C \approx 6 \times N \times D
$$

$$
N_{opt} \propto C^{0.5}, D_{opt} \propto C^{0.5}
$$

即，最优参数量$N_{opt}$和最优数据量$D_{opt}$的增长应该正比于算力增长的0.5次方。

### 4. 缩放定律对模型规模的扩展有何指导意义？

- **模型研发的资源规划**：给定算力预算，可直接算出最优的参数量和数据量配比，避免数千万甚至数亿元的算力浪费。

- **解释了大模型涌现能力的底层逻辑**：涌现能力是模型损失随规模缩放持续降低，突破了复杂任务的性能阈值后的外在表现。只要持续按规律扩大规模，模型就会持续解锁更复杂的能力。

- **模型选型与微调的依据**：优先选择 “训练充分、符合 Chinchilla 最优配比” 的基座模型。同样算力下，训练充分的低参数量模型，微调后的效果会远好于欠训练的高参数量模型，同时推理成本更低。

- **明确了大模型性能提升的边界**：当模型规模扩大到一定程度后，会出现边际效益递减，同时性能提升会受限于数据质量、数据多样性，而非单纯的规模堆砌。


## 面试问题：大语言模型存在哪些内在的局限性？
**难度评分：⭐⭐ (2/5)  |  考察频率：⭐⭐⭐⭐ ⭐(5/5)**

### 1. 什么是“幻觉”？其产生的主要原因有哪些？

a. **幻觉的定义和分类**

大语言模型的**幻觉**（Hallucination）是指模型生成**看似流畅、语法正确**但**事实不准确**或**缺乏外部证据支持**的**内容**的**现象**。（来源：[《Large Language Models Hallucination: A Comprehensive Survey》](https://arxiv.org/pdf/2510.06265)

[!](./imgs/4-1.png)

从技术定义来看，幻觉分为：**内在幻觉**和**外在幻觉**。（来源：[《HalluLens: LLM Hallucination Benchmark》](https://aclanthology.org/2025.acl-long.1176.pdf) 当然幻觉也可以分为“事实不一致“（与现实真实信息矛盾），”逻辑不一致“（输出内部逻辑矛盾），”上下文不一致“（输出与输入提示矛盾）三类。

- **内在幻觉**：生成内容与**输入上下文不一致**，可以直接对照输入验证，属于推理时的一致性问题。

- **外在幻觉**：生成内容与**训练数据不一致**，输入上下文无法证实或证伪，反映模型知识吸收不足、无法识别知识边界。

b. **幻觉产生的原因**

大语言模型产生幻觉的主要原因可以从**训练数据**、**模型设计逻辑**和**推理机制**三个层面进行分析：

- **训练数据**：训练数据的**覆盖程度**、**质量**、**完整性**和**一致性**直接影响模型的输出质量。如果训练数据包含**错误、过时或虚构的信息**，会导致模型学习到这些错误知识。（例：知识覆盖不全、数据噪声和矛盾，数据分布偏移）

- **模型设计逻辑**：模型的优化目标是“预测下一个词”而非”保证事实准确性“；知识以分布式、非结构化的方式存储在权重中，缺乏显式的检索验证机制；Transformer的自回归建模方式导致曝光偏差（训练时依赖真实标签，推理时却依赖自身预测滚动生成，早期微小误差被逐词指数级放大）。这些设计逻辑共同决定了：**大语言模型**本质上是一个**高维、概率性的联想引擎**，擅长基于统计共现生成流畅的文本，却**并非一个能够存储、检索与验证事实的可靠系统**。

- **推理机制**：大语言模型受限于**上下文窗口**，难以完整保留长程事实信息，导致生成内容与上下文脱节；解码策略的决策可能选中概率较低但语义突兀的错误词。

c. **幻觉的示例**

- 在医疗领域，大语言模型生成了看似正确的模糊治疗建议；在代码生成任务中，生成了存在逻辑漏洞的代码；在教育领域，编造一些不存在的科学理论来解释自然现象。

d. **幻觉的检测手段**

- **检索验证**：将模型输出与外部知识库/搜索结果进行对比，验证内容的真实性。

- **不确定性量化**：通过评估模型输出的置信度（“模型心里有没有底”）来识别可能的幻觉。（例：评估单次输出所有可能词的熵。如果熵很大，模型心里没底，则存在幻觉的概率大）

- **嵌入相似度比较**：通过比较生成内容与已知真实内容在向量空间中的相似性来判断真实性。

- **自一致性检测**：通过多次生成并比较结果的一致性来识别幻觉。如果模型对同一问题的多次回答存在显著差异，则可能存在幻觉问题。

e. **幻觉的缓解策略**

通过精心设计的输入来引导模型生成更准确的内容。（例：在上下文中提供示例，额外的知识或额外的思考流程。）

注：该策略被进一步发展为提示工程和检索增强技术，详细参见2.6节。

### 2.大模型的知识为什么存在截止日期？如何获取实时信息？

a. 什么是**知识截止日期**？为什么？

**大语言模型的知识截止日期**是指**模型训练数据结束的时间点**，在这个时间点之后发生的事件、产生的知识或出现的信息，模型都无法获知。

这一局限性来源于大语言模型的训练机制。模型通过学习训练数据中的模式来获取知识，而**训练数据本身是某个时间点的”快照“**。随着时间推移，新的事件不断发生，知识不断更新，但模型无法自动吸收这些新信息。这种静态知识与动态世界之间的矛盾，是大语言模型面临的根本性挑战之一。

b. 有哪些**实时信息获取**的技术？

在推理层面，通过**外部的数据库提供实时的信息**，来知道大语言模型的推理；通过**调用搜索引擎**等工具，让大语言模型实时了解外界的最新资讯。

注：这两个技术被进一步发展为检索增强技术和工具调用，详情参见2.6节。

在训练层面，采用**实时微调**和**持续学习**技术，让模型逐步学习新信息而不忘记旧知识。

[!](./imgs/4-2.png)

### 3.大语言模型可能产生哪些偏见或不安全内容？

**训练数据被污染**是大语言模型产生有害行为（如**仇恨言论、歧视、暴力、威胁、虚假信息、隐私侵犯、教唆**等）的主要根源。随着**内容过滤、对抗性测试、人类反馈对齐**等技术手段的广泛应用，这类有害输出已得到有效抑制，目前在日常生活中已不常出现。

### 4.大语言模型在哪些复杂任务上表现仍不理想？

尽管大语言模型在许多任务上表现出色，但仍然存在能力边界，主要包括：

- **数学推理能力的缺陷**：大语言模型本质是概率生成模型，缺乏真正的数学理解，因此在处理**基础算术**、**复杂数学推理**、**数字表示**等严谨数学问题时会犯错误。（例：用错公式、逻辑混乱、答案对过程错）

- **因果推断的困难**：大语言模型的设计本质是**统计模式匹配**（学习文本中的共现规律），而**非符号计算或逻辑推演**。在**逻辑谜题**、**形式证明**和**复杂文字问题**等严格推理任务上表现不佳。

- **过度讨好人类**：大语言模型由于基于人类反馈进行训练，容易出现**盲目迎合**和**过度妥协**人类的**强硬性输入**，甚至出现违背客观事实的情况。（例：在辅助代码生成的任务中，强制性地让大语言模型把两个完全无关的技术进行融合）

## 面试问题：文本表示：什么是分词器？怎么将文本进行分词？
**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐(3/5)**

### 1.什么是分词器？什么是词表？

a. 什么是**分词器**（Tokenizer）？

分词器是大语言模型中最基础、最关键的组件之一，它的核心作用是将**人类可读的文本**（字符串）转换为**模型可以处理的数字序列**（通常是整数 ID）。

分词器是连接原始文本与算法模型的桥梁，在整个文本处理流水线中扮演着 **“翻译官“**的角色，负责将人类语言翻译成模型可理解的数字形式。

b. 什么是**词表**（Vocabulary）？

**词表**是一个**查找表**，包含**子词词汇**或指向** vocab.txt 文件路径**的**字符串**。它定义了词元与整数 ID 之间的映射关系，每个唯一词元被分配一个 ID，从 0 开始一直到词汇表的大小。

[!](./imgs/5-1.png)

### 2.分词器是如何处理文本数据的？

- **归一化**：将文本统一为标准形式，减少无关差异。（例：Unicode规范化，格式规划化）

- **预分词**：基于规则将**输入本文**初步切分成**相互独立的词块**，决定了分词的基本边界，防止跨块的分词。

- **正式分词**：分词器会利用预定义的”词表“将**相互独立的词块**切割为**词元**。

- **编码**： 通过**查词表**，将每个词元映射为唯一的整数编号，即Token ID。

- **添加特殊标记**：在整数 ID 序列的开头、结尾、中间插入对应固定 ID，特殊 token 在词表训练时就会被预留固定位置，永远不会被子词拆分。（例：GPT系列模型的起始标记`<s>`和结束标记`<\s>`）

### 3. 词表大小如何影响模型性能？

- 词表是**词元的”字典“**，决定了模型能够理解和生成的**语言范围**。（例：基本英语单词的词表一般不包含生僻的专业术语，无法对医疗行业的专业术语进行分词）

- 由于大语言模型需要分析所有词表含有的词，因此**词表大小直接影响模型的参数规模**。词表越大，模型越大。

### 4.什么是未登录词？怎么解决未登录词？

**未登录词**是指**词表**中不存在的**词元**。为了将未登录词也能很好的编码，采用了以下措施：

- **[unk]标记**：将未登录词直接映射为固定的未知词元ID。该方法丢弃了该词元的全部信息，模型无法区分不同的未知词。（例：[unk]或[UNK])

- **字节回退**（Byte-level BPE)： 将未登录词拆解为 UTF-8 字节序列，每个字节（0~255）都是词表中预置的词元。（例：把😊拆分为[0xF0, 0x9F, 0x98, 0x8A]，再进行编码。

### 5.常见的分词方法有哪些？

a. **分词的粒度**有哪些？

分词方法根据粒度大小，可以分为**”词级分词“**，**”字符级分词“**和**”子词分级”**。

- **词级分词**：文本按分隔符（如空格）拆分为单独的单词。

- **字符级分词**：：将文本拆分为单个字符（如字母和标点符号）。

- **子词分级**：将文本拆分为部分单词或字符集。（例：“annoyingly“拆分为”annoying“和”ly“）

b. **常见的分词算法**有哪些？

**BPE（Byte Pair Encoding，字节对编码）**，**核心思想**是采用**贪心合并策略**迭代合并语料库中**最频繁的相邻符号对**，创建**常见的词元**或**新词元**。拆分结果位唯一

**算法步骤**如下：

- **粒度切分**：将基于某个粒度切分后的字符或单词作为词元处理。

- **频率计算**：计算所有词元的频率。

- **词元合并**：找到频率最高的词元对，合并为一个新词元。 

- **词元扩充**：将新词元添加到词汇表中，并从词汇表中删除原来的词元。

- **循环迭代**：重复**词元合并**和 **扩充**，直到词汇表大小达到期望的大小。

**WordPiece**（BPE的改进版），**核心改进**是将基于**语言模型的似然提升**作为**合并策略**。（注语言模型可以是简单的Unigram模型）**改进策略**是每轮尝试一个合并操作，选择让训练语料的**对数似然增加最多的组合**，重复直到词表达到目标大小。

**Unigram**：是**基于概率**的分词方法。Unigram不使用合并策略，而是假设一个大词表，保留其中**概率最高的子词组合**。

**算法步骤**如下：

- **词元初始集**：构造所有可能的子词候选。

- **词元删除**：采用EM模型计算每个词元的似然下降量，删除部分损失小的词元。

- **迭代优化**：循环**词元删除**直至达到词表的容量要求。

**BPE、WordPiece 的分词**是基于固定规则的确定性过程，**拆分结果唯一**；而 **Unigram 的分词**是基于EM模型的概率最优求解过程，**拆分结果不唯一**。

其他分词方法还包括：**基于空格的分词**（空格为分隔符），**基于字符的分词**（每个字符都是词元）和**SentencePiece**（把输入视为原始字节流，统一处理所有的语言）。
