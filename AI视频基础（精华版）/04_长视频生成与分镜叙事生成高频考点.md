<h1 id="4.4.1_长视频生成的核心难点：长程依赖、漂移与上下文爆炸">4.4.1 长视频生成的核心难点：长程依赖、漂移与上下文爆炸</h1>

> 本节聚焦长视频生成相对短视频生成新增的核心困难：长程依赖、误差累积、内容漂移和上下文爆炸。理解这部分内容，是后续学习 Causal、Chunk、Sliding-Window、Hierarchical Generation 以及多镜头叙事生成的基础。

<h2 id="4.4.1.1_面试问题：为什么长视频生成不能简单理解为“短视频生成跑更久”？">4.4.1.1 面试问题：为什么长视频生成不能简单理解为“短视频生成跑更久”？</h2>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐⭐ (5/5)**

长视频生成不能简单理解为“短视频生成跑更久”，核心原因是：**视频长度增加以后，问题性质会从短时运动建模，升级为长程一致性、上下文管理、误差控制和叙事结构建模的系统问题。**

短视频生成通常只需要在几秒范围内维持局部运动自然和相邻帧一致；而长视频生成要求模型在更长时间范围内记住人物、场景、物体状态、动作进展和事件因果关系。也就是说，长视频不是简单增加帧数，而是增加了更长时间尺度上的约束。

从概率建模角度，可以把条件视频生成写成：

```math
p(x_{1:T} \mid c)
```

其中 $x_{1:T}$ 表示长度为 $T$ 的视频序列，$c$ 表示文本、图像、参考主体或其他条件。短视频场景下，$T$ 较小，模型主要面对局部时空一致性；当 $T$ 变大时，模型需要处理更长的历史依赖：

```math
p(x_{1:T} \mid c)
=
\prod_{t=1}^{T} p(x_t \mid x_{<t}, c)
```

这个式子不意味着所有视频扩散模型都严格按帧自回归生成，而是说明长视频的本质约束：**后续内容必须与更早之前的内容保持一致**。例如人物在第 5 秒出现过，第 40 秒再次出现时，身份、服饰、发型和场景关系仍然应该稳定。

长视频生成相比短视频生成主要多出以下几类困难：

1. **长程依赖更强**  
   模型不仅要保证相邻帧连续，还要保证相隔很远的片段之间逻辑一致。例如同一个角色不能中途换脸，同一个房间布局不能无故变化，一个已经发生的事件不能在后面被“遗忘”。

2. **漂移和误差累积更明显**  
   短视频中轻微的外观变化可能不明显，但长视频中这种变化会逐步放大，表现为身份漂移、背景漂移、动作方向漂移或画面风格漂移。时间越长，模型越容易偏离初始条件和原始语义。

3. **上下文长度和计算成本急剧增加**  
   视频 token 数量通常与时间长度、分辨率和空间 patch 数共同相关。若视频 latent token 数近似为：

    ```math
    N = T \times H' \times W'
    ```

    则全局 self-attention 的复杂度大致为：

    ```math
    \mathcal{O}(N^2)
    ```

    因此视频长度翻倍，并不只是生成时间翻倍，还可能显著增加显存、注意力计算和采样成本。

4. **语义与事件结构更复杂**  
   短视频常常只表达一个动作或一个场景变化，而长视频往往包含多个阶段：开场、动作展开、转场、结果、角色再次出现等。模型必须维持事件顺序、因果关系和节奏，而不是只生成局部看起来合理的动态画面。

5. **评估标准发生变化**  
   短视频可以重点看单帧质量、短时运动自然度和 prompt 对齐；长视频还要看长时身份一致性、场景稳定性、事件连贯性、镜头组织和整体可观看性。因此短视频指标不能完全覆盖长视频问题。

所以，面试中可以这样概括：

**短视频生成主要考察“局部时空质量”，长视频生成考察的是“长时间范围内的稳定生成能力”。它不仅要求每一小段视频好看，还要求整段视频在主体、场景、动作、语义和事件逻辑上持续一致。**

因此，长视频生成不是把短视频模型直接外推到更多帧，而是需要围绕长程依赖、上下文压缩、误差控制、条件持续传播和高效推理进行系统设计。
