<h1 id="4.5.1_视频数据与数据管线：为什么数据质量决定训练上限">4.5.1 视频数据与数据管线：为什么数据质量决定训练上限</h1>

> 视频模型训练首先是数据问题。数据质量、文本标注、运动分布、分辨率、时长和过滤策略，会直接决定模型能否学到稳定的时空先验。

<h2 id="4.5.1.1_面试问题：为什么视频数据比图像数据更难构建？">4.5.1.1 面试问题：为什么视频数据比图像数据更难构建？</h2>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐⭐ (5/5)**

视频数据比图像数据更难构建，核心原因是视频多了时间维度。图像主要关注单帧语义和空间质量；视频还要关注帧间运动、时序连续性、镜头变化、音画关系和事件过程。

视频样本可以表示为：

```math
x_{1:T}=\{x_1,x_2,\ldots,x_T\}
```

它不仅要求每一帧质量高，还要求相邻帧和远距离帧之间具有合理关系。低质量视频数据常见问题包括：压缩伪影、字幕水印、镜头切分混乱、动作不完整、文本描述不准、运动过弱或过强、帧率不稳定。

因此，视频数据构建至少需要同时考虑四类质量：

1. **视觉质量**：清晰度、压缩伪影、水印、黑边、噪声。
2. **运动质量**：运动是否真实、是否有足够动态、是否存在抖动。
3. **语义质量**：caption 是否准确描述主体、动作、场景和事件。
4. **时间质量**：片段是否连续、是否包含无关切镜、是否存在时序断裂。

面试中可以概括为：**图像数据主要决定空间先验，视频数据同时决定空间先验和时间先验；视频数据差，模型很难学到稳定运动和长时一致性。**

<h2 id="4.5.1.2_面试问题：视频数据清洗通常需要关注哪些维度？">4.5.1.2 面试问题：视频数据清洗通常需要关注哪些维度？</h2>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐⭐ (5/5)**

视频数据清洗的目标是过滤掉会破坏生成分布或训练稳定性的样本。常见维度包括：

| 维度 | 典型问题 | 影响 |
|---|---|---|
| 视觉质量 | 模糊、水印、字幕、黑边、压缩伪影 | 降低画质上限 |
| 时序质量 | 抖动、跳帧、镜头混剪、重复帧 | 破坏运动先验 |
| 运动分布 | 过静、过快、无意义运动 | 影响动态幅度 |
| 文本质量 | caption 错误、过短、缺动作 | 降低文本遵循 |
| 安全合规 | 敏感、侵权、低俗内容 | 影响部署风险 |
| 去重 | 重复视频、相似片段 | 降低数据多样性 |

清洗并不是简单删除低清晰度视频。对于视频生成，运动质量和 caption 质量同样关键。例如一个高分辨率但几乎静止的视频，对学习运动帮助有限；一个画质不错但文本描述错误的视频，会让模型学到错误的文本-视频对应关系。

面试中可以总结为：**视频数据清洗要同时看画质、运动、语义和时序连续性，而不是只看分辨率。**

<h2 id="4.5.1.3_面试问题：为什么视频-caption-质量会显著影响-T2V-模型能力？">4.5.1.3 面试问题：为什么视频 caption 质量会显著影响 T2V 模型能力？</h2>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐⭐ (5/5)**

T2V 模型学习的是文本条件到视频分布的映射：

```math
p(x_{1:T}\mid c_{\text{text}})
```

如果 caption 只描述静态主体，例如“a dog in a park”，模型很难学习动作、镜头、事件和时间变化。如果 caption 描述错误，例如视频是“狗在奔跑”，文本却写成“狗坐着”，模型会学习到错误对齐。

高质量视频 caption 通常需要覆盖四类信息：

1. **主体**：谁或什么物体出现。
2. **场景**：在哪里、环境是什么。
3. **动作**：主体如何运动、事件如何发生。
4. **镜头与风格**：相机运动、景别、画面风格、节奏。

视频 caption 的难点在于它要描述过程，而不是只描述画面。单帧 caption 往往不足以训练 T2V，因为它忽略了动作演化和事件顺序。

面试中可以概括为：**T2V 模型的文本遵循能力，很大程度取决于 caption 是否准确描述视频中的动态过程。**

<h1 id="4.5.2_训练粒度与训练目标：Frame、Clip、Sequence-与多尺度组织">4.5.2 训练粒度与训练目标：Frame、Clip、Sequence 与多尺度组织</h1>

> 视频训练的粒度决定模型学到什么。按帧训练主要学空间，按 clip 训练学习短时运动，按长序列训练才可能学习更长程的状态和事件。

<h2 id="4.5.2.1_面试问题：视频模型训练为什么不能只按单帧训练？">4.5.2.1 面试问题：视频模型训练为什么不能只按单帧训练？</h2>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐⭐ (5/5)**

单帧训练只能学习空间分布，无法学习帧间运动和时间一致性。若只把视频拆成图像训练，模型可以获得较好的单帧画质，但生成视频时容易出现闪烁、动作不连续、主体漂移和运动不自然。

视频生成目标应建模整个时空序列：

```math
p(x_{1:T}\mid c)
```

而不是只建模每帧独立分布：

```math
\prod_{t=1}^{T}p(x_t\mid c)
```

后一种形式忽略了帧间依赖，容易导致每一帧单独看还可以，但连起来不像视频。

面试中可以这样回答：**单帧训练提供空间先验，clip 或 sequence 训练提供时间先验。视频模型必须学习帧间关系，而不是只学习高质量图像生成。**

<h2 id="4.5.2.2_面试问题：clip-length-对视频生成模型有什么影响？">4.5.2.2 面试问题：clip length 对视频生成模型有什么影响？</h2>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐⭐ (5/5)**

clip length 决定模型在训练时能看到多长的时间上下文。较短 clip 成本低，适合学习局部运动和短时一致性；较长 clip 能覆盖更完整动作和事件，但显存、计算和数据质量要求更高。

如果 latent token 数近似为：

```math
N=T\times H'\times W'
```

则时间长度 $T$ 增大会直接放大 token 数，并进一步增加 attention 成本：

```math
\mathcal{O}(N^2)
```

因此，clip length 是质量和成本之间的关键权衡。

过短的 clip 容易导致模型只能学到局部动态，无法理解动作完整过程；过长的 clip 会显著增加训练成本，也更容易受到镜头切换、caption 不准和长时噪声影响。

面试中可以总结为：**clip length 越长，模型越可能学习长程时序，但训练成本和数据噪声也越高。**

<h2 id="4.5.2.3_面试问题：多分辨率、多帧率、多长宽比训练为什么重要？">4.5.2.3 面试问题：多分辨率、多帧率、多长宽比训练为什么重要？</h2>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

真实视频数据天然具有多分辨率、多帧率和多长宽比。如果训练时只固定一种设置，模型可能对其他输出规格泛化较差，部署时也难以支持不同产品场景。

多分辨率训练可以增强模型对不同细节尺度的适应能力；多帧率训练可以让模型适应不同运动采样密度；多长宽比训练可以支持横屏、竖屏、方图等多种应用场景。

工程上通常会使用 bucketing 策略，把相近分辨率、帧数和长宽比的视频放入同一 batch，减少 padding 浪费：

```math
\text{bucket}=(T,H,W,\text{aspect ratio})
```

面试中可以概括为：**多尺度训练不是为了形式多样，而是为了让模型适应真实视频分布，并提高训练吞吐和部署灵活性。**

<h1 id="4.5.3_图像-视频联合训练与课程学习：从空间先验到时间先验">4.5.3 图像-视频联合训练与课程学习：从空间先验到时间先验</h1>

> 视频模型通常不会从零开始学习全部能力。更常见的路线是先利用图像数据学习空间语义和视觉质量，再用视频数据补充运动和时序先验。

<h2 id="4.5.3.1_面试问题：为什么很多视频模型需要图像-视频联合训练？">4.5.3.1 面试问题：为什么很多视频模型需要图像-视频联合训练？</h2>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐⭐ (5/5)**

图像数据规模大、质量高、标注相对成熟，适合学习物体、场景、构图、风格和纹理等空间先验。视频数据规模和质量通常更难保证，但它包含运动、时序、镜头和事件信息，适合学习时间先验。

图像-视频联合训练的核心价值是：**用图像补空间，用视频补时间。**

可以把训练目标理解为混合分布学习：

```math
\mathcal{L}=\lambda_{\text{img}}\mathcal{L}_{\text{img}}+\lambda_{\text{vid}}\mathcal{L}_{\text{vid}}
```

其中 $\mathcal{L}_{\text{img}}$ 提供空间生成能力，$\mathcal{L}_{\text{vid}}$ 提供时序生成能力。

如果只用视频训练，模型可能受限于视频数据噪声和规模，单帧质量不足；如果只用图像训练，模型缺乏运动先验。联合训练能够在视觉质量和动态能力之间取得更好平衡。

<h2 id="4.5.3.2_面试问题：从图像预训练迁移到视频训练时，哪些能力会继承，哪些能力必须重新学习？">4.5.3.2 面试问题：从图像预训练迁移到视频训练时，哪些能力会继承，哪些能力必须重新学习？</h2>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

从图像预训练迁移到视频训练时，可继承的主要是空间能力，包括语义理解、物体外观、纹理细节、风格表达和文本-图像对齐能力。

必须重新学习的是时间相关能力，包括：

1. **运动连续性**：相邻帧之间如何自然变化。
2. **时序一致性**：身份、背景和纹理如何跨帧稳定。
3. **动作过程**：动作如何开始、发展和结束。
4. **相机运动**：镜头移动和主体运动如何区分。
5. **长程状态**：远距离片段如何保持一致。

因此，从图像到视频不是简单增加时间维度，而是从空间生成扩展到时空生成。面试中可以概括为：**图像模型提供“画得好”的能力，视频训练补充“动得对、连得上”的能力。**

<h2 id="4.5.3.3_面试问题：课程学习在视频生成训练中通常如何设计？">4.5.3.3 面试问题：课程学习在视频生成训练中通常如何设计？</h2>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

课程学习的思想是先训练容易任务，再逐步增加难度。在视频生成中，常见课程包括：

1. **从图像到视频**：先学空间先验，再学时间先验。
2. **从短 clip 到长 clip**：先学短时运动，再扩展长程一致性。
3. **从低分辨率到高分辨率**：先学全局结构和运动，再补细节。
4. **从简单条件到复杂条件**：先 T2V，再扩展到 I2V、V2V、多条件控制。
5. **从粗数据到精选数据**：先大规模预训练，再高质量数据微调。

课程学习的价值在于降低训练难度和不稳定性。直接在长序列、高分辨率、多条件视频上训练，成本高且容易不收敛；分阶段训练可以逐步构建空间、时间和控制能力。

面试中可以总结为：**视频生成训练通常不是一步到位，而是通过任务难度、数据质量、分辨率和时间长度逐步扩展。**

<h1 id="4.5.4_训练稳定性与优化机制：Diffusion、Flow-Matching、EMA-与-CFG">4.5.4 训练稳定性与优化机制：Diffusion、Flow Matching、EMA 与 CFG</h1>

> 视频模型训练不仅成本高，而且更容易不稳定。训练目标、噪声调度、条件 dropout、参数平均和 loss weighting 都会影响最终收敛质量。

<h2 id="4.5.4.1_面试问题：视频扩散模型为什么比图像扩散模型更难训练稳定？">4.5.4.1 面试问题：视频扩散模型为什么比图像扩散模型更难训练稳定？</h2>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐⭐ (5/5)**

视频扩散模型更难训练稳定，主要因为它同时面对更高维输入、更复杂数据噪声和更强的一致性约束。

图像扩散主要学习单帧分布：

```math
p(x\mid c)
```

视频扩散需要学习时空序列分布：

```math
p(x_{1:T}\mid c)
```

相比图像，视频训练多出几类不稳定来源：

1. **token 数更多**：时间维度放大计算和显存。
2. **数据噪声更复杂**：压缩、跳帧、切镜、字幕、水印都会影响训练。
3. **运动分布更难学**：真实运动包含主体运动、相机运动和场景变化。
4. **一致性要求更高**：单帧好不代表视频稳定。
5. **caption 噪声更大**：文本往往难以准确描述完整动态过程。

面试中可以概括为：**视频扩散训练不只是更大的图像扩散训练，而是同时学习空间质量、时间连续性和条件对齐。**

<h2 id="4.5.4.2_面试问题：Diffusion-和-Flow-Matching-在训练目标上有什么核心差异？">4.5.4.2 面试问题：Diffusion 和 Flow Matching 在训练目标上有什么核心差异？</h2>

**难度评分：⭐⭐⭐⭐⭐ (5/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

Diffusion 通常通过逐步加噪和去噪学习数据分布。经典噪声预测目标可写为：

```math
\mathcal{L}_{\text{diff}}=
\mathbb{E}_{x_0,\epsilon,t}
\left[
\lVert \epsilon-\epsilon_\theta(x_t,t,c)\rVert^2
\right]
```

其中 $x_t$ 是加噪后的样本，$\epsilon_\theta$ 预测噪声。

Flow Matching 更关注学习从噪声分布到数据分布的连续向量场。训练目标可抽象为：

```math
\mathcal{L}_{\text{fm}}=
\mathbb{E}_{x_t,t}
\left[
\lVert v_\theta(x_t,t,c)-u_t(x_t)\rVert^2
\right]
```

其中 $v_\theta$ 是模型预测的速度场，$u_t$ 是目标流场。

二者的核心差异在于：Diffusion 更强调去噪过程，Flow Matching 更强调学习从噪声到数据的传输路径。在视频生成中，Flow Matching 常被关注，是因为它有机会在采样效率和训练稳定性上提供更好的工程取舍，但具体效果仍取决于模型结构、数据和采样器设计。

面试中应避免绝对化说法。更稳妥的回答是：**Diffusion 学去噪，Flow Matching 学流场；二者都是生成建模路线，视频场景下重点在于采样效率、稳定性和长序列建模成本。**

<h2 id="4.5.4.3_面试问题：EMA、CFG-dropout-和-loss-weighting-分别解决什么问题？">4.5.4.3 面试问题：EMA、CFG dropout 和 loss weighting 分别解决什么问题？</h2>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐⭐ (5/5)**

这三个机制分别对应训练稳定性、条件控制和不同噪声阶段的优化平衡。

**EMA** 用指数滑动平均维护更稳定的模型参数：

```math
\theta_{\text{ema}}\leftarrow \beta\theta_{\text{ema}}+(1-\beta)\theta
```

它可以降低训练过程中参数震荡，推理时通常使用 EMA 权重获得更稳定质量。

**CFG dropout** 在训练时随机丢弃条件，让模型同时学习有条件和无条件预测。推理时可使用 classifier-free guidance：

```math
\hat{\epsilon}
=\epsilon_\theta(x_t,t,\varnothing)
+s\left(\epsilon_\theta(x_t,t,c)-\epsilon_\theta(x_t,t,\varnothing)\right)
```

其中 $s$ 是 guidance scale。它提升条件遵循，但过大可能导致过饱和、运动僵硬或多样性下降。

**loss weighting** 用于平衡不同时间步、噪声强度或 token 区域的学习难度。视频中不同帧、不同运动区域、不同噪声阶段对训练贡献不同，合理加权有助于稳定收敛。

面试中可以概括为：**EMA 稳参数，CFG dropout 学条件可控，loss weighting 平衡不同训练阶段和区域的优化难度。**

<h1 id="4.5.5_训练效率与分布式系统：显存、吞吐与长序列并行">4.5.5 训练效率与分布式系统：显存、吞吐与长序列并行</h1>

> 视频模型训练的瓶颈通常不是单一算子，而是时空 token、显存、通信、数据读取和并行策略共同决定的系统问题。

<h2 id="4.5.5.1_面试问题：视频模型训练成本为什么远高于图像模型？">4.5.5.1 面试问题：视频模型训练成本为什么远高于图像模型？</h2>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐⭐ (5/5)**

视频比图像多时间维度，训练成本随帧数、分辨率和模型规模共同增长。若 latent token 数为：

```math
N=T\times H'\times W'
```

Transformer attention 成本近似为：

```math
\mathcal{O}(N^2)
```

视频帧数 $T$ 增大，会同时放大计算量、显存占用、中间激活、通信量和数据读取压力。

视频训练成本高还因为：视频 VAE 编解码更重，数据加载 IO 更大，batch 内样本规格更不统一，caption 和过滤成本更高，长序列训练更容易触发显存瓶颈。

面试中可以总结为：**视频训练贵，不只是因为帧多，而是时间维度会放大 token、attention、激活、通信和数据管线的全部成本。**

<h2 id="4.5.5.2_面试问题：视频模型训练中常见的显存优化方法有哪些？">4.5.5.2 面试问题：视频模型训练中常见的显存优化方法有哪些？</h2>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐⭐ (5/5)**

常见显存优化方法包括：

1. **Mixed Precision**：使用 FP16、BF16 或 FP8 降低激活和参数显存。
2. **Activation Checkpointing**：反向传播时重算部分激活，换显存为计算。
3. **Gradient Accumulation**：用多次小 batch 累积模拟大 batch。
4. **ZeRO / FSDP**：切分参数、梯度和优化器状态。
5. **Memory-efficient Attention**：降低 attention 中间矩阵显存。
6. **Token / Frame Reduction**：降低帧数、分辨率或 latent token 数。
7. **Bucketing**：减少 padding，提升有效 token 利用率。

这些方法不是简单叠加。显存优化往往会带来吞吐下降、通信增加或实现复杂度提升。面试中可以概括为：**视频训练显存优化本质是参数、激活、优化器状态、attention 中间量和数据规格之间的系统权衡。**

<h2 id="4.5.5.3_面试问题：数据并行、张量并行、流水线并行和序列并行分别适合解决什么瓶颈？">4.5.5.3 面试问题：数据并行、张量并行、流水线并行和序列并行分别适合解决什么瓶颈？</h2>

**难度评分：⭐⭐⭐⭐⭐ (5/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

不同并行方式解决不同瓶颈：

| 并行方式 | 拆分对象 | 适合瓶颈 |
|---|---|---|
| Data Parallel | batch | 数据规模大、模型单卡可放下 |
| Tensor Parallel | 单层矩阵计算 | 单层参数或计算过大 |
| Pipeline Parallel | 模型层 | 模型深度大、单卡放不下 |
| Sequence Parallel | 序列 token | 长视频 token 过多 |

视频模型中，sequence parallel 尤其重要，因为长视频的瓶颈常来自时空 token 序列过长，而不是只有参数量大。对于 Video DiT 或大规模 Transformer，单纯数据并行往往不足以解决显存和 attention 成本。

实际训练系统通常采用混合并行：

```text
data parallel + tensor parallel + sequence parallel + activation checkpointing
```

面试中可以总结为：**数据并行扩 batch，张量并行拆矩阵，流水线并行拆层，序列并行拆长上下文；视频模型通常需要混合并行。**

<h1 id="4.5.6_推理加速与部署优化：采样步数、延迟、吞吐与成本">4.5.6 推理加速与部署优化：采样步数、延迟、吞吐与成本</h1>

> 视频模型能训练出来不等于能上线。工业部署关心的是生成质量、延迟、吞吐、显存、成本和稳定性之间的综合平衡。

<h2 id="4.5.6.1_面试问题：视频生成推理为什么比图像生成更贵？">4.5.6.1 面试问题：视频生成推理为什么比图像生成更贵？</h2>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐⭐ (5/5)**

视频推理比图像更贵，主要因为一次生成要处理多个时空帧，并且扩散或 flow-based 模型通常需要多步采样。

粗略地看，推理成本可写成：

```math
C_{\text{infer}}\propto S\times T\times H'\times W'\times C_{\text{model}}
```

其中 $S$ 是采样步数，$T$ 是帧数，$H'W'$ 是 latent 空间尺寸，$C_{\text{model}}$ 是单步模型计算成本。

视频还需要额外的 VAE 编解码、帧间一致性处理、可能的条件编码、后处理和安全过滤。因此，视频推理成本不仅来自主干模型，也来自整个生成链路。

面试中可以概括为：**视频推理贵，是采样步数、帧数、分辨率、模型规模和系统后处理共同叠加的结果。**

<h2 id="4.5.6.2_面试问题：视频生成常见推理加速方法有哪些？">4.5.6.2 面试问题：视频生成常见推理加速方法有哪些？</h2>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐⭐ (5/5)**

常见推理加速方法包括：

1. **减少采样步数**：使用更高效 sampler、distillation、consistency model 或 rectified flow 类方法。
2. **模型蒸馏**：将多步大模型压缩为少步或小模型。
3. **缓存机制**：复用文本编码、参考图特征、KV cache 或历史 latent。
4. **VAE 优化**：加速视频编码和解码，减少解码瓶颈。
5. **低精度推理**：FP16、BF16、INT8、FP8 等。
6. **批处理与调度**：通过 batching、dynamic batching、队列调度提升吞吐。
7. **分辨率/帧数自适应**：根据场景需求动态调整输出规格。

需要注意，加速通常会牺牲质量、稳定性或可控性。例如采样步数过少可能导致细节下降，过强蒸馏可能降低多样性，低精度可能影响数值稳定。

面试中可以总结为：**推理加速不是单点优化，而是采样器、模型、VAE、缓存、精度和调度共同优化。**

<h2 id="4.5.6.3_面试问题：工业部署中如何权衡质量、速度、显存和成本？">4.5.6.3 面试问题：工业部署中如何权衡质量、速度、显存和成本？</h2>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐⭐ (5/5)**

工业部署中，视频生成模型需要在质量、速度、显存和成本之间做权衡。可以抽象为约束优化问题：

```math
\max Q_{\text{video}}
\quad \text{s.t.}\quad
L\leq L_{\max},\;
M\leq M_{\max},\;
C\leq C_{\max}
```

其中 $Q_{\text{video}}$ 表示视频质量，$L$ 是延迟，$M$ 是显存占用，$C$ 是单位生成成本。

常见策略包括：

1. **多档模型**：高质量模型用于离线生成，轻量模型用于预览和交互。
2. **分阶段生成**：先低分辨率预览，再高分辨率精修。
3. **动态配置**：根据用户场景调整帧数、分辨率、采样步数和 guidance scale。
4. **异步任务系统**：长任务进入队列，避免阻塞交互。
5. **质量-成本分层**：不同会员等级或产品场景使用不同资源预算。

面试中可以概括为：**工业部署不是追求单次生成最高质量，而是在可接受成本和延迟下稳定地产出足够好的视频。**
