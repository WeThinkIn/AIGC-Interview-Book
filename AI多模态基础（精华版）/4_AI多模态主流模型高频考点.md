### 5.4.1 InternVL系列

InternVL 是开源多模态阵营中演进脉络最清晰的系列之一。从 1.5 到 3.5，它的每一代都在解决一个明确的工程瓶颈：**1.5 解决"高分辨率如何看清"、2 解决"如何统一多任务输出"、2.5 解决"如何低成本扩规模与治理数据"、3 把"先训语言再缝视觉"改为"原生多模态预训练"、3.5 则在推理与效率两端用级联 RL 和 ViR/DvD 收尾。** 

<h3 id="1.介绍一下InternVL-1.5在4K输入时针对分块序列爆炸的优化技术">1.介绍一下InternVL-1.5在4K输入时针对分块序列爆炸的优化技术</h3>

<h4 id="面试问题-高分辨率动态分块为何会导致视觉token序列爆炸？">面试问题-高分辨率动态分块为何会导致视觉token序列爆炸？</h4>

**难度评分：⭐⭐ (2/5)  |  考察频率：⭐⭐⭐ (3/5)**

标准 ViT 的输入分辨率只有 224 或 336，对于文档、表格、票据这类**密集文本**场景，整张图被压成两三百个 token，每个字只占几个像素，根本读不清——这是 OCR/DocVQA 类任务的命门。

**InternVL-1.5 的解法是动态高分辨率分块（Dynamic High-Resolution）。** 它按输入图的长宽比，把图切成 $1\sim40$ 个 $448\times448$ 的子图块（tile）：训练时用 $1\sim12$ 块，推理时可零样本放大到 40 块，**等效支持到 4K 分辨率**。

**序列爆炸正是分块的副作用。** 每个 $448\times448$ 子图过 ViT（patch=14）会得到 $ (448/14)^2 = 32\times32 = 1024 $ 个 patch token。若不做压缩，40 块就是 $40\times1024 \approx 4$ 万个视觉 token，再加一张全局缩略图。Transformer 的注意力为 $O(n^2)$ 复杂度，4 万量级的视觉序列会显著增加显存与算力开销，并挤占 LLM 的上下文窗口。因此高分辨率分块在"提升细节清晰度"与"控制计算开销"之间形成直接矛盾。

---

<h4 id="面试问题-InternVL-1.5用了哪些技术缓解视觉token爆炸？">面试问题-InternVL-1.5用了哪些技术缓解视觉token爆炸？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

InternVL-1.5 用三项配合的设计，把"4K 输入"压到 LLM 可消费的规模。

- **Pixel Unshuffle（像素逆洗牌）下采样——核心降本手段。** 对每个子图块的 patch 特征做一次 $2\times2$ 的 space-to-depth 操作：把 $2\times2$ 的空间块折叠进通道维，空间分辨率减半、通道翻 4 倍，**token 数降到原来的 $1/4$**。于是一个 $448\times448$ 子图从 1024 个 token 压成 **256 个 token**。40 块从约 4 万降到约 1 万 token，量级上立刻可控。（注：官方早期文档写作 "pixel shuffle"，实为笔误，该操作是下采样的 *pixel unshuffle*，OpenGVLab 已在仓库中更正。）
- **长宽比自适应分块 + 缩略图。** 切块时从一组候选长宽比中选最接近原图的方案，尽量减少形变与无效块；同时附一张整图缩略图（thumbnail）保留**全局上下文**，避免只见局部、不见整体。
- **强视觉编码器打底。** 用持续学习增强的 InternViT-6B 作为视觉前端，使得即便经过压缩，单个 token 承载的语义密度也更高。

综合来看，InternVL-1.5 以"动态分块保证细节、Pixel Unshuffle 将每块压缩至 256 token、缩略图保留全局"三项设计，将 4K 输入的视觉序列从约 4 万压缩到约 1 万量级。其压缩比例固定为 $1/4$，对信息密集区与背景区一视同仁，这是该方案的主要局限。

<h3 id="2.介绍一下InternVL-2如何在一个模型中支持框坐标、掩码等不同输出">2.介绍一下InternVL-2如何在一个模型中支持框坐标、掩码等不同输出</h3>

<h4 id="面试问题-InternVL-2如何用统一接口产出框坐标与掩码等不同输出？">面试问题-InternVL-2如何用统一接口产出框坐标与掩码等不同输出？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐ (3/5)**

InternVL-2 的目标是用**一个模型**覆盖检测、定位、分割等数百种视觉-语言任务，而非为每类任务训一个专家模型。它的统一靠两条互补的机制。

- **框坐标：直接当作文本 token 生成。** 视觉定位（grounding）被重述为"看图写坐标"——给定描述句，模型以纯文本形式输出归一化的边界框坐标（官方评测 prompt 即 `Please provide the bounding box coordinates of the region this sentence describes: {}`）。这里**没有任何检测专用 head**，框就是语言序列里的几个数字 token，由 LLM 自回归生成。
- **掩码/图像等结构化输出：特殊 token 路由到下游解码器。** 借助 VisionLLMv2 式设计，InternVL-2 把 MLLM 与多个下游任务解码器相连：模型生成一个特殊 token（如 `[SEG]`），**取该 token 在 LLM 最后一层的隐藏状态作为"提示向量"**，送入掩码解码器（如 SAM/SAM-2）生成分割掩码。后续 Sa2VA 正是用 InternVL-2/2.5 + SAM-2 的 `[SEG]` token 实现图像/视频的指代分割，且梯度可经 `[SEG]` 回传到 MLLM。

**本质是一个"语言接口"统领多任务**：要么把答案直接写成坐标文本，要么吐出特殊 token 去驱动专用解码器——两种方式都不破坏"以 LLM 为核心、自回归生成"的统一范式。

---

<h4 id="面试问题-输出统一为序列/路由token相比为每个任务配专用head有何优势？">面试问题-输出统一为序列/路由token相比为每个任务配专用head有何优势？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐ (3/5)**

把输出统一成"序列 + 路由 token"，相比传统"每任务一个专用 head"，优势集中在通用性与可扩展性。

- **一个模型、数百任务，免去逐任务改结构。** 新增任务往往只需调整指令与数据，而不必新设计输出头并重训，天然契合"指令驱动"的多模态助手范式。
- **充分复用 LLM 的语言与推理能力。** 坐标、类别、关系都以语言形式表达，检测/定位因此能借力 LLM 的常识与上下文理解（例如"穿红衣服那个人"这种需要语义消歧的指代）。
- **跨任务知识共享。** 检测、描述、问答共享同一套主干，任务间可互相增益，而非彼此割裂。

这种统一方式同样存在代价。其一，纯文本坐标的精度上限有限——逐 token 生成坐标在像素级精确度上通常不及专用检测器。其二，`[SEG]` 路由方案需要额外挂载并训练下游解码器，并非零成本的统一。

<h3 id="3.介绍一下InternVL-2.5的渐进式缩放策略">3.介绍一下InternVL-2.5的渐进式缩放策略</h3>

<h4 id="面试问题-渐进式缩放策略的执行流程是怎样的？">面试问题-渐进式缩放策略的执行流程是怎样的？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

渐进式缩放（Progressive Scaling）是 InternVL-2.5 把"训练大模型"成本压下来的核心策略。它的思路是**"先用小 LLM 把视觉编码器训好，再无痛迁移到大 LLM"**，而非一上来就用大 LLM 联合训练视觉端。

**执行流程分两步：**

- **第一步：小 LLM 对齐视觉编码器（Stage 1.5）。** 把 InternViT 与一个**资源友好的小 LLM（如 20B）**联合训练，集中优化"基础视觉能力 + 跨模态对齐"。这一步刻意避开"直接用大 LLM 训视觉"的高昂算力。
- **第二步：迁移到大 LLM、跳过重训。** 借助共享权重机制，把上一步训好的 InternViT **直接迁移到大 LLM（如 72B）上，无需重训视觉端**。换言之，训大模型时 Stage 1.5 可以整段跳过，直接复用已优化的 InternViT。

**这套做法成立的关键观察是**：即便 ViT 与 LLM 是用 next-token-prediction 损失联合训练得到的，**学出的视觉特征仍是"可泛化的通用表示"，能被其它 LLM 直接看懂**。正因为视觉特征不与某个特定 LLM 强绑定，"换大脑不换眼睛"才行得通。

---

<h4 id="面试问题-为什么渐进式缩放能大幅降低训练成本？关键技术点是什么？">面试问题-为什么渐进式缩放能大幅降低训练成本？关键技术点是什么？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐ (3/5)**

渐进式缩放的降本效果非常直观：**InternVL2.5-78B 全程只用了约 1200 亿 token，相比 Qwen2-VL 的 1.4 万亿 token 不到其 1/10。**

**降本来自三个关键点：**

- **避开"大 LLM × 视觉"的昂贵联合训练。** 视觉-语言对齐这件最耗算力的事，被放到便宜的小 LLM 上完成；大 LLM 阶段只需做高层任务适配，不必从零重学视觉。
- **视觉编码器复用，消除冗余。** InternViT 一次训成、跨规模反复使用，大模型训练得以跳过中间阶段，数据需求显著下降。
- **配套的数据打包（data-packing）。** 通过 select-search-pack-maintain 把样本高效拼装，提升 GPU 利用率，进一步压缩训练时间。

该策略的前提是视觉特征可跨 LLM 迁移。若目标大 LLM 的嵌入空间与小 LLM 差异较大，迁移后可能仍需少量再对齐；它节省的是重训视觉端的成本，而非完全免除对齐。

总体而言，渐进式缩放将"先小后大、先对齐后扩展"固化为可复用的训练流水线，用视觉编码器的一次性投入换取大模型训练阶段近十倍的数据与算力节省。

<h3 id="4.分析一下InternVL-2.5如何通过数据清洗解决重复生成问题">4.分析一下InternVL-2.5如何通过数据清洗解决重复生成问题</h3>

<h4 id="面试问题-重复生成问题的成因是什么？为何对CoT推理尤其有害？">面试问题-重复生成问题的成因是什么？为何对CoT推理尤其有害？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐ (3/5)**

重复生成（repetitive generation）指模型在推理时陷入"循环复读"，反复输出相同词句或段落。InternVL 团队在研发中观察到一个反直觉的现象：**哪怕训练集中只混入极小比例的异常样本，也足以在推理时诱发这种异常行为。**

**成因在于模型会忠实模仿训练数据里的"病态模式"。** 网络数据中本就存在大量重复内容（模板化文本、复读的字幕、堆砌的关键词），模型一旦学到这种分布，就可能在生成时复现循环。

**为何对 CoT 推理尤其致命？** 长链思维（Chain-of-Thought）和长文本生成的输出序列很长，**重复模式会被逐步放大**：一旦在中途陷入某个循环，整条推理链就此跑偏、无法收敛到正确答案。因此 InternVL-2.5 明确指出，重复生成在长文本与 CoT 推理任务上危害最大——这也是它把数据清洗当作一等公民的直接动机。

---

<h4 id="面试问题-InternVL-2.5的数据过滤流水线如何抑制重复生成？">面试问题-InternVL-2.5的数据过滤流水线如何抑制重复生成？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐ (3/5)**

InternVL-2.5 设计了一条三道闸的数据过滤流水线，专门压制异常样本、尤其是重复样本。

- **① LLM 质量打分（LLM-Based Quality Scoring）。** 用预训练 LLM 配合领域专用 prompt 给每条样本打 $0\sim10$ 分，**低于阈值（如 7 分）直接剔除**，保证整体数据质量底线。
- **② 重复检测（Repetition Detection）。** 用 LLM prompt 专门标记重复样本，**低于更严格阈值（如 3 分）的予以排除**；高质量学术数据集可豁免该流程，避免误杀。被标记样本再走人工复核。
- **③ 启发式规则过滤（Heuristic Rule-Based Filtering）。** 用规则检测"异常句长、重复行"等显式异常，配合人工验证后移除。多模态侧也用类似规则检测视觉异常。

论文报告，这套流水线**显著降低了异常行为发生率，尤其在 CoT 推理任务上带来明显提升**。

概括而言，InternVL-2.5 将重复生成的治理落实为"LLM 打分 + 重复检测 + 规则过滤 + 人工复核"的数据工程闭环，其核心判断是：少量异常数据即可显著毒化推理行为，必须在训练前完成清洗。

<h3 id="5.介绍一下InternVL-3的原生多模态预训练及其与BLIP-2两阶段方法的不同">5.介绍一下InternVL-3的原生多模态预训练及其与BLIP-2两阶段方法的不同</h3>

<h4 id="面试问题-InternVL-3的原生多模态预训练是怎么做的？">面试问题-InternVL-3的原生多模态预训练是怎么做的？</h4>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐⭐ (5/5)**

InternVL-3 最具标志性的改变，是把"先训纯文本 LLM、再缝上视觉"的传统路线，换成了**原生多模态预训练（Native Multimodal Pre-Training）**——把语言预训练与多模态对齐**合并进同一个预训练阶段**。

**具体做法：在预训练阶段就交错喂入两类数据。** 大规模纯文本语料与多样的多模态数据（图文、视频-文本、图文交错序列）被**混合在一起联合预训练**，让模型从一开始就同时习得语言能力与多模态能力，而非事后再嫁接。

**关键工程点是数据配比。** 异构数据混合的最优采样比并不好定，InternVL-3 用两阶段策略确定：先分别在纯多模态、纯语言数据上训独立模型、各自找到最优内部配比，再在固定总预算下确定两者的相对比例。实验结论是 **语言:多模态 = 1:3** 时整体最优；总训练量约 **2000 亿 token（语言约 500 亿 + 多模态约 1500 亿）**。

**配套设计：** 引入 **V2PE（可变视觉位置编码）** 支撑更长的多模态上下文；后训练沿用 **SFT + MPO（混合偏好优化）** 进一步强化对话与推理。一个有力佐证是——得益于原生预训练，InternVL-3 的**纯文本能力甚至超过了作为语言初始化的 Qwen2.5 基座**，说明联合训练并未牺牲语言能力，反而有所增益。

---

<h4 id="面试问题-原生多模态预训练与BLIP-2先对齐再生成的两阶段方法根本不同在哪？">面试问题-原生多模态预训练与BLIP-2先对齐再生成的两阶段方法根本不同在哪？</h4>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

两者的根本分歧，是"**视觉能力是原生长出来的，还是事后缝上去的**"。

**BLIP-2 是典型的"事后改装（retrofit）"路线。** 它冻结现成的视觉编码器、冻结现成的 LLM，**只训中间的 Q-Former 桥接模块**，且分两阶段（视觉-语言表示学习 → 视觉-语言生成学习）。LLM 自始至终没有在预训练中原生接触过视觉，视觉信息只是作为软提示被外挂注入。

**InternVL-3 是"原生（native）"路线。** 不需要桥接模块、不需要事后对齐，视觉能力在预训练阶段就与语言能力一同习得。

**核心差异对照：**

| 维度 | BLIP-2（两阶段改装） | InternVL-3（原生预训练） |
|---|---|---|
| 视觉能力来源 | 预训练后外挂，事后对齐 | 预训练阶段原生习得 |
| 是否需桥接模块 | 需要 Q-Former（约 188M） | 不需要额外桥接/对齐模块 |
| 训练阶段 | 表示学习 → 生成学习两阶段 | 语言+多模态单阶段联合 |
| LLM 是否动 | 冻结 LLM | LLM 参与联合预训练 |
| 主要风险 | 模态鸿沟、对齐债（distribution shift） | 预训练成本高、需掌控基座 |
| 适用方 | 想低成本给现成 LLM 加视觉 | 有基座掌控权的头部团队 |

**为什么原生路线更优？** 事后对齐天然存在"对齐债"——冻结的视觉空间与语言空间靠一个小模块强行拼接，细粒度交互与分布一致性都受限；原生预训练让两种模态从同一起点共同优化，避免了这种系统性偏差，这也是 InternVL-3 在多模态与纯文本双榜都更强的根因。

原生预训练的代价在于成本高昂：需要从预训练阶段起就投入大规模算力，且必须掌控语言基座，通常只有头部团队能够承担。相较之下，BLIP-2 式"冻结两端 + 轻量桥接"在为现成强 LLM 低成本增加视觉能力这一目标上仍具性价比。

从 BLIP-2 到 InternVL-3，体现了多模态范式从"事后改装"向"原生一体"的演进：前者成本较低但存在对齐债，后者成本较高但能力上限更高。

<h3 id="6.分析一下InternVL-3.5为何采用级联式强化学习而非单阶段方法">6.分析一下InternVL-3.5为何采用级联式强化学习而非单阶段方法</h3>

<h4 id="面试问题-级联式强化学习（Cascade-RL）的流程是什么？">面试问题-级联式强化学习（Cascade RL）的流程是什么？</h4>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

级联式强化学习（Cascade RL）是 InternVL-3.5 提升推理能力的核心框架，本质是一套**"由粗到细（coarse-to-fine）"的两阶段 RL**。

- **第一阶段：离线 RL（Offline RL）做粗对齐。** 采用 **MPO（Mixed Preference Optimization，混合偏好优化）**，在**预先收集好的偏好数据**上训练。离线数据无需实时采样，**收敛稳定、成本低**，负责把模型快速带到一个不错的初始策略附近。
- **第二阶段：在线 RL（Online RL）做精对齐。** 采用 **GSPO** 等在线算法，**从当前策略实时采样**生成数据再优化。在线数据质量更高、更贴合模型当前分布，负责在最优解附近做精细打磨。

**效果**：这套粗到细策略在下游推理基准上带来显著提升，整体推理性能相比 InternVL-3 **最高提升约 +16%（如 MMMU、MathVista）**。

---

<h4 id="面试问题-为什么用级联（离线→在线）而非单阶段强化学习？">面试问题-为什么用级联（离线→在线）而非单阶段强化学习？</h4>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐ (3/5)**

级联之所以优于单阶段，是因为离线与在线 RL 各有致命短板，而两者恰好互补。

- **纯在线 RL 的问题：不稳、昂贵、冷启动差。** 全程实时采样训练，对超参与初始策略极敏感，容易震荡甚至崩溃；若从一个未对齐的策略直接在线训，冷启动阶段样本质量低、训练效率差、算力消耗巨大。
- **纯离线 RL 的问题：被静态数据封顶。** MPO/DPO 这类离线方法稳定便宜，但**能力上界被预收集数据死死限制**——模型无法通过与环境交互"自我进化"，难以突破数据本身的天花板。
- **级联的取舍：离线打底（稳）+ 在线精修（高）。** 先用离线 RL 廉价、稳定地把策略带到优质初始点（粗），再用在线 RL 在最优解附近精细提升（细）。这样既规避了在线冷启动的不稳，又突破了离线静态数据的上界。

概括而言，Cascade RL 以"离线稳定起步 + 在线提升上限"的组合，化解了单阶段强化学习"要么不稳定、要么受静态数据封顶"的两难。

<h3 id="7.介绍一下InternVL-3.5的ViR与DvD效率优化技术及其协同工作方式">7.介绍一下InternVL-3.5的ViR与DvD效率优化技术及其协同工作方式</h3>

<h4 id="面试问题-ViR（视觉分辨率路由）如何按内容动态压缩视觉token？">面试问题-ViR（视觉分辨率路由）如何按内容动态压缩视觉token？</h4>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

ViR（Visual Resolution Router，视觉分辨率路由）是 InternVL-3.5 为缓解视觉 token 过多问题提出的**自适应**压缩方案，针对的是固定比例压缩对所有区域一视同仁的不足。

**核心思想：按语义内容逐块决定压缩率。** ViR 会评估每个视觉块（patch）的语义信息量——**文字、OCR 等信息密集区保留高分辨率（更多 token）**，**背景、纯色等简单区施加高压缩（更少 token）**。这与早期只看图像宽高决定切几块的做法本质不同：ViR 看的是"内容值不值得多花 token"。

**集成方式轻量。** ViR 通过一个轻量训练阶段 **ViCO（Visual Consistency Learning，视觉一致性学习）** 接入模型，无需重训主干。**效果**：视觉 token 数最多可减少约 **50%**，而在文档/OCR 等基准上几乎无精度损失。

---

<h4 id="面试问题-DvD（解耦视觉-语言部署）如何提效？它与ViR如何协同？">面试问题-DvD（解耦视觉-语言部署）如何提效？它与ViR如何协同？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐ (3/5)**

DvD（Decoupled Vision-Language Deployment，解耦视觉-语言部署）是从**系统部署**层面提速的工程方案，重点优化预填充（prefilling）阶段的阻塞。

**做法：视觉与语言分服务器、异步流水线。** 把 **ViT + MLP（+ ViR）部署在"视觉服务器"GPU 上**，把 **LLM 单独部署在"语言服务器"GPU 上**。视觉端批量处理图像、产出紧凑特征，再以 **BF16 经 TCP（可选 RDMA 提速）单向传给**语言端融合解码。整个过程组织成"视觉处理 → 特征传输 → 语言处理"的**异步三段流水线**，相互重叠执行，最大限度减少流水线停顿。**单靠 DvD 即可带来近 2× 吞吐提升。**

**ViR 与 DvD 如何协同？** 两者一个减负载、一个提并行，叠加增益明显：

- **ViR 减少 token 数** → 既缩短了语言端的序列长度，又减少了视觉端到语言端的**特征传输量**；
- **DvD 重叠视觉/语言计算** → 让被压缩后的流水线进一步并行化。
- 二者结合，相比 InternVL-3 实现了 **最高 4.05× 的推理加速**。

概括而言，ViR 从算法侧按内容自适应压缩 token，DvD 从系统侧解耦部署实现异步流水线，两者一软一硬相互配合，将多模态推理的延迟与成本显著降低。

### 5.4.2 GLM-VL系列

GLM-VL 是智谱（Z.ai）推出的多模态系列。从 GLM-4.1V-Thinking 到 GLM-4.5V、GLM-4.6V，其主线是**"以推理为中心"**：4.1V-Thinking 确立了推理导向的训练框架与 RLCS，4.5V 换用 MoE 基座把能力扩展到全谱系视觉推理与智能体场景，4.6V 进一步引入原生多模态工具调用，将"视觉感知"与"可执行动作"打通。

<h3 id="1.介绍一下GLM-4.1V-Thinking的核心设计目标">1.介绍一下GLM-4.1V-Thinking的核心设计目标</h3>

<h4 id="面试问题-GLM-4.1V-Thinking的核心设计目标是什么？">面试问题-GLM-4.1V-Thinking的核心设计目标是什么？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

GLM-4.1V-Thinking 的核心目标是**推动通用多模态推理（general-purpose multimodal reasoning）**，开源型号为 GLM-4.1V-9B-Thinking。它的设计哲学是"以推理为中心（reasoning-centric）"——不满足于感知与描述，而是显式引入"思考（thinking）"过程，让模型先推理、再作答。

**目标的实现路径分两步：**

- **第一步：大规模预训练打造强视觉基座。** 团队认为基座的潜力"几乎设定了最终性能的上界"，因此先通过大规模预训练得到一个具备充分潜力的视觉基础模型。
- **第二步：用 RLCS 释放潜力。** 在基座之上，通过带课程采样的强化学习（RLCS）全面提升跨域能力，覆盖 STEM 解题、视频理解、内容识别、代码、grounding、GUI 智能体、长文档理解等。

**架构形态**为标准的"ViT + MLP 适配器 + 语言解码器"，支持图像与视频输入，并具备显式的时间建模能力。

**能力佐证**：在 28 个公开基准的综合评测中，GLM-4.1V-9B-Thinking 几乎在所有任务上超越 Qwen2.5-VL-7B，并在其中 18 个基准上达到与参数量大得多的 Qwen2.5-VL-72B 相当甚至更优的水平；在长文档理解与 STEM 推理等高难任务上，与 GPT-4o 等闭源模型相比也具备竞争力。

---

<h4 id="面试问题-RLCS（带课程采样的强化学习）如何支撑以推理为中心的目标？">面试问题-RLCS（带课程采样的强化学习）如何支撑以推理为中心的目标？</h4>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

RLCS（Reinforcement Learning with Curriculum Sampling，带课程采样的强化学习）把课程学习的思想引入在线采样，让训练样本的难度持续匹配模型当前能力，从而最大化每次更新的信息量。

- **难度分级：离线 + 在线双重评估。** 训练前，用多个成熟 VLM（或早期 RL checkpoint）在全量数据上跑 pass@k，并融合专家人工难度标注，把数据切分为由易到难的多个难度档；训练中，对每个 rollout 记录 pass@k 结果并映射到对应难度档，与离线标签合并，实时反映模型当前水平。
- **采样再加权：聚焦中等难度。** 按训练迭代的粒度持续重加权各难度类别——**下采样过于简单与当前过难的样本，加大中等难度样本的曝光**，因为模型在这一区间收益最大。
- **奖励体系：可验证为主、人类反馈为辅。** RLVR（可验证奖励）对不同域采用不同校验：数学用 sympy 符号校验、grounding 用 IoU、开放式问题用 LLM 判分，并通过明确的答案标记规避 reward hacking；在主观或不可验证域则用 RLHF。多域数据按预设比例混合，以 GRPO 目标优化。
- **训练稳定性工程。** 关闭 KL 与熵惩罚、采用大批量并对 rollout 动态打包、使用 per-sample loss（比 per-token 更稳）；以 EMA 进行动态批量扩展保持效率；对过长 CoT 注入答案标记，引导更短、更聚焦的输出。

**一个重要的工程教训**：奖励基础设施必须足够稳健——任一单域校验器的缺陷都可能毒化乃至拖垮整个 RL 训练，这在实验中被明确观察到。RLCS 通过"难度自适应 + 稳健奖励"两条腿，支撑了 GLM-4.1V-Thinking 跨域推理能力的规模化提升。

<h3 id="2.介绍一下GLM-4.5V的核心设计目标及其与GLM-4.1V-Thinking的架构差异">2.介绍一下GLM-4.5V的核心设计目标及其与GLM-4.1V-Thinking的架构差异</h3>

<h4 id="面试问题-GLM-4.5V的核心设计目标是什么？">面试问题-GLM-4.5V的核心设计目标是什么？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐ (3/5)**

GLM-4.5V 延续 GLM-4.1V-Thinking 的推理路线，基于智谱 GLM-4.5-Air 构建，在 42 个公开视觉-语言基准上达到同规模 SOTA。它的目标重心从"验证推理范式可行"转向**"全谱系视觉推理 + 实用性 + 智能体"**。

**能力覆盖更全面：**

- **图像推理**：场景理解、复杂多图分析、空间识别。
- **视频理解**：长视频分段与事件识别。
- **GUI 任务**：读屏、图标识别、桌面操作辅助。
- **复杂图表与长文档解析**：研究报告分析、信息抽取。
- **Grounding**：精确的视觉元素定位。

**实用性导向的关键设计是 hybrid（混合）推理模式**：提供 thinking（深度推理）与 non-thinking（快速响应）两种模式，可通过开关切换，从而在"回答质量"与"响应延迟"之间灵活权衡，使模型更贴近真实业务的可用性要求。

<h4 id="面试问题-GLM-4.5V在架构上与GLM-4.1V-Thinking有哪些关键差异？">面试问题-GLM-4.5V在架构上与GLM-4.1V-Thinking有哪些关键差异？</h4>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

两者整体都遵循"视觉编码器 + 适配器 + 语言解码器"的范式，但在语言基座、位置编码与多媒体处理上有显著差异。

- **语言基座：稠密 → 稀疏 MoE。** GLM-4.1V-Thinking 基于约 9B 的稠密模型；GLM-4.5V 换为 **GLM-4.5-Air 稀疏 MoE**——总参约 106B、每 token 激活约 12B、128 个路由专家加共享专家、46 层解码、采用 GQA（8 个 query group）与 SiLU 门控线性单元。MoE 用稀疏激活在"高容量"与"可控推理成本"之间取得平衡。
- **位置编码：引入 3D-RoPE / MRoPE。** 采用多分辨率旋转位置编码（分段 [8, 12, 12]，分别对应时间、高、宽三个维度），强化空间与时序建模。
- **视频处理：3D 卷积编码器。** 专门捕捉视频的时序特征（运动、序列变化），而非仅做逐帧空间编码。
- **输入与上下文：** 支持任意宽高比、最高 4K 分辨率；官方支持 64K 上下文（架构序列长度上限达 131072）。

**GLM-4.1V-Thinking 与 GLM-4.5V 架构对照：**

| 维度 | GLM-4.1V-Thinking | GLM-4.5V |
|---|---|---|
| 语言基座 | 约 9B 稠密（GLM-4-9B 系） | GLM-4.5-Air 稀疏 MoE（106B 总参 / 12B 激活） |
| 专家结构 | 无 | 128 路由专家 + 共享专家 |
| 位置编码 | 常规位置编码 | 3D-RoPE / MRoPE（分段 [8,12,12]） |
| 视频编码 | 逐帧 + 时间建模 | 3D 卷积编码器 |
| 输入分辨率 | 图像/视频 | 任意宽高比、最高 4K |
| 上下文长度 | 较短 | 64K（架构上限 131072） |
| 推理模式 | thinking 为主 | hybrid（thinking / non-thinking 可切换） |

差异的核心逻辑是：4.5V 用 MoE 基座扩大了能力容量，用 3D 位置编码与 3D 卷积强化了时空建模，用 hybrid 模式提升了落地可用性，从而把 4.1V-Thinking 验证过的推理范式扩展到了更大规模与更广任务面。

<h3 id="3.介绍一下GLM-4.5V在视觉定位任务中如何实现边界框的精确生成">3.介绍一下GLM-4.5V在视觉定位任务中如何实现边界框的精确生成</h3>

<h4 id="面试问题-GLM-4.5V如何表示并生成边界框坐标？">面试问题-GLM-4.5V如何表示并生成边界框坐标？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

GLM-4.5V 把视觉定位的坐标表示与生成方式做了明确规范化，以保证可解析、可训练、对分辨率稳健。

- **坐标表示：千分制相对坐标。** 边界框为四元组 $[x_1, y_1, x_2, y_2]$，由左上角与右下角坐标构成；每个数值按图像宽度（对 $x$）或高度（对 $y$）归一化后**乘以 1000**。这种千分制相对坐标与绝对像素解耦，对任意分辨率与宽高比都保持一致语义。
- **特殊 token 包裹坐标。** 用 `<|begin_of_box|>` 与 `<|end_of_box|>` 两个特殊 token 标记边界框区段（其内括号样式可为 `[]`、`[[]]`、`()`、`<>` 等），使坐标在输出序列中边界清晰，便于解析与训练。
- **生成方式：先推理、后定位。** 给定包含目标描述的 prompt（支持对目标的复杂描述与指定输出格式），模型先进行 step-by-step 推理（thinking），再输出边界框。定位被纳入"先思考、后作答"的推理范式，而非简单地直接回归坐标。

<h4 id="面试问题-GLM-4.5V通过哪些训练与输入设计保证定位精度？">面试问题-GLM-4.5V通过哪些训练与输入设计保证定位精度？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐ (3/5)**

边界框"能输出"不等于"输出得准"，GLM-4.5V 的定位精度由训练目标与输入条件共同保障。

- **可验证奖励直接对齐 IoU。** 在强化学习中，grounding 任务以 **IoU（交并比）作为奖励信号**，让模型朝"框得更准"的方向优化；配合带课程采样的难度调度，使定位训练的难度持续匹配模型能力。
- **标准化输出抑制抽取误差。** 通过统一的坐标格式与明确的答案标记，降低坐标被错误抽取的概率，同时规避 reward hacking（用取巧方式骗过校验器而非真正定位）。
- **高分辨率输入提供感知前提。** 支持任意宽高比与最高 4K 分辨率，使小目标、密集排布与精细边缘得以保留——定位精度的上限首先取决于"是否看得清"，高分辨率输入是细粒度定位的必要条件。

<h3 id="4.分析一下GLM-4.5V在长上下文训练中如何处理超长序列以理解复杂文档">4.分析一下GLM-4.5V在长上下文训练中如何处理超长序列以理解复杂文档</h3>

<h4 id="面试问题-复杂文档理解为何依赖长上下文？长序列带来哪些挑战？">面试问题-复杂文档理解为何依赖长上下文？长序列带来哪些挑战？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐ (3/5)**

复杂文档（多页 PDF、研究报告、含表格与图表的长文）的理解，本质上要求模型在**单次输入内同时容纳大量文本与多张高分辨率页面图像**，并在其间建立跨页、跨模态的引用关系。这天然依赖很长的上下文窗口。

**长序列带来的挑战集中在两点：**

- **token 预算极易被突破。** 高分辨率页面图本身就消耗大量视觉 token，多页叠加再加上正文文本，序列长度迅速膨胀，很容易超出常规上下文窗口。
- **算力与位置编码的双重压力。** Transformer 注意力为 $O(n^2)$ 复杂度，序列越长算力开销越大；同时，位置编码能否在远超训练长度时仍保持稳定外推，直接决定长序列建模质量。

---

<h4 id="面试问题-GLM-4.5V用哪些设计支撑长序列建模？">面试问题-GLM-4.5V用哪些设计支撑长序列建模？</h4>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐ (3/5)**

GLM-4.5V 通过基座、位置编码、计算结构与输入适配四个层面共同支撑长序列。

- **长上下文基座。** 基于原生支持长上下文的 GLM-4.5-Air，GLM-4.5V 官方支持 **64K 上下文**，其架构序列长度上限可达 **131072（即 128K）**，为容纳多页文档提供了底层窗口。
- **3D-RoPE / MRoPE 位置编码。** 采用分段为 [8, 12, 12] 的多分辨率旋转位置编码，对时间、高、宽分别建模。旋转位置编码有利于长序列的外推稳定性，并能在图文交错的长序列中保持一致的相对位置语义。
- **MoE 控制长序列成本。** 106B 总参中每 token 仅激活约 12B，使长序列推理的计算开销相对可控，让"长窗口"在工程上可负担。
- **高分辨率与任意宽高比适配。** 支持最高 4K 分辨率与任意宽高比，配合视觉 token 的压缩处理，在"看清文档细节"与"序列不超限"之间取得平衡。

四者结合，使 GLM-4.5V 能够在长上下文下对复杂文档进行连贯的跨页、跨模态理解。

<h3 id="5.分析一下GLM-4.6V为何需要引入原生多模态工具调用能力">5.分析一下GLM-4.6V为何需要引入原生多模态工具调用能力</h3>

<h4 id="面试问题-传统文本式工具调用在多模态场景下有何不足？">面试问题-传统文本式工具调用在多模态场景下有何不足？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐ (3/5)**

传统的工具调用是为纯文本场景设计的：要么由 VLM 先把图像转成文本描述、再以文本形式驱动工具，要么用"纯文本 LLM + 旁挂视觉模型"的级联管线。把它直接套到多模态智能体场景，会暴露三类不足。

- **输入侧信息损失。** 图像、截图、文档页被迫先转换为文本描述才能进入工具调用，版式、细节与视觉证据在转换中大量丢失。
- **输出侧无法消费视觉返回。** 工具返回的统计图表、渲染后的网页截图、检索到的商品图等本身是图像，纯文本模型无法理解，难以纳入后续推理。
- **感知与动作割裂。** "看懂"与"执行"分属不同模型或环节，管线复杂、误差逐级累积，形成明显的感知-动作鸿沟（perception-action gap）。

---

<h4 id="面试问题-GLM-4.6V的原生多模态工具调用如何弥补这些不足？">面试问题-GLM-4.6V的原生多模态工具调用如何弥补这些不足？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

GLM-4.6V 首次集成**原生多模态工具调用（native multimodal tool calling）**，目标是打通"视觉感知"与"可执行动作"，为真实业务中的多模态智能体提供统一技术底座。

- **多模态输入：图像直接入参。** 图像、截图、文档页可**直接作为工具参数**传入，无需预先转为文本描述，从而最大限度避免信号损失、简化管线。
- **多模态输出：视觉理解工具返回。** 模型能**视觉理解工具返回的结果**（搜索结果、统计图表、渲染网页截图、检索商品图等），并将其纳入后续推理链与最终输出。
- **闭合"感知 → 理解 → 执行"循环。** 同一个模型完成图像优先（image-first）的工具输入输出，支撑视觉网页搜索、含图表的文档问答、图文混排内容创作等复杂任务。
- **训练方式。** 把工具调用行为纳入统一的强化学习目标，对齐模型在复杂工具链中的任务规划、指令遵循与格式遵循能力。

在规模上，GLM-4.6V 采用 128K 多模态上下文训练，并提供 GLM-4.6V（106B）与 GLM-4.6V-Flash（9B）等版本。原生多模态工具调用使其能够以单一模型完成"图像优先"的端到端智能体流程，而非依赖多模型拼接。

<h3 id="6.介绍一下GLM-4.6V在“设计到代码”任务中如何实现像素级复现与交互式编辑">6.介绍一下GLM-4.6V在"设计到代码"任务中如何实现像素级复现与交互式编辑</h3>

<h4 id="面试问题-GLM-4.6V如何实现从设计稿/截图到代码的像素级复现？">面试问题-GLM-4.6V如何实现从设计稿/截图到代码的像素级复现？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

GLM-4.6V 针对前端开发做了专门优化，显著缩短"设计到代码（design to code）"的周期。

**像素级复现（Pixel-Level Replication）的流程是**：用户上传一张网页截图或设计稿，模型识别其中的**布局、组件与配色方案**，进而生成高保真的 HTML/CSS/JS 代码。它不仅能复刻单屏，还**支持长网页生成与多轮迭代**。

**支撑这一能力的关键训练设计包括：**

- **大规模长上下文图文数据的持续预训练**，使模型熟悉真实网页的版式与代码对应关系。
- **视觉-语言压缩对齐**（受 Glyph 工作启发），提升模型对密集视觉信息的高效表示与对齐能力。
- **十亿级多模态世界知识数据**与**智能体合成数据训练**，丰富其对界面元素与交互模式的先验。

借助这些设计，GLM-4.6V 能把"看到的界面"较为忠实地还原为"可运行的前端代码"。

<h4 id="面试问题-GLM-4.6V的交互式编辑与视觉反馈闭环如何工作？">面试问题-GLM-4.6V的交互式编辑与视觉反馈闭环如何工作？</h4>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐ (3/5)**

在一次性生成之外，GLM-4.6V 还支持基于视觉的迭代修改，核心是交互式编辑与视觉反馈闭环两项机制。

- **交互式编辑（Interactive Editing）。** 用户可在生成页面的截图上**圈选某个区域并给出自然语言指令**（如"把这个按钮左移并改成深蓝色"），模型自动**定位到对应的代码片段并进行修改**。其本质是把视觉 grounding 能力作用于"截图区域 ↔ 代码片段"的映射，将视觉指代转化为精确的代码定位与编辑。
- **视觉反馈闭环（Visual Feedback Loop）。** 受 UI2Code$^N$ 工作启发，该闭环让模型**产出代码 → 系统渲染 → 截图回灌给模型 → 模型检视渲染结果并与目标比对 → 自我纠正与精修代码**，从而保证布局与视觉一致性。
- **训练方式。** 在强化学习中针对该闭环进行优化，目标是支持多步、视觉对齐的修正，而非一次性生成，进而验证"自我改进多模态智能体"的可行性。

这两项机制使 GLM-4.6V 的前端能力从"截图生成代码"升级为"可持续迭代、自我校正"的视觉编程，更贴近真实开发中的多轮修改流程。

### 5.4.3 Qwen-VL系列

Qwen-VL 是阿里通义推出的多模态系列。从 Qwen-VL、Qwen2-VL、Qwen2.5-VL 到 Qwen3-VL，其主线围绕**"原生动态分辨率 + 时空位置建模"**持续演进：Qwen-VL 确立了"编码器 + 适配器 + LLM"框架与三阶段训练，Qwen2-VL 引入朴素动态分辨率与 M-RoPE，Qwen2.5-VL 重做高效 ViT 并把动态性扩展到时间维（绝对坐标、动态 FPS、绝对时间对齐），Qwen3-VL 再以 Interleaved-MRoPE、DeepStack 与文本时间戳对齐强化长视频与细粒度理解。

<h3 id="1.介绍一下Qwen-VL系列视觉编码器的核心演进路径">1.介绍一下Qwen-VL系列视觉编码器的核心演进路径</h3>

<h4 id="面试问题-从Qwen-VL到Qwen3-VL视觉编码器经历了哪些关键演进？">面试问题-从Qwen-VL到Qwen3-VL视觉编码器经历了哪些关键演进？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

Qwen-VL 系列的视觉编码器经历了从"固定分辨率 + 压缩适配器"到"原生动态分辨率 + 高效 ViT + 多层特征融合"的连续演进。

- **Qwen-VL（2023）：固定分辨率 + 位置感知适配器。** 视觉编码器初始化自 OpenCLIP 的 ViT-bigG；输入分辨率固定（224，后提至 448）。其特色是一个**位置感知的视觉-语言适配器**：用单层 cross-attention 配 256 个可学习 query，把变长图像特征压缩为固定的 256 个视觉 token，并加入 2D 绝对位置编码以保留空间信息。
- **Qwen2-VL（2024）：朴素动态分辨率 + 2D-RoPE。** 引入 Naive Dynamic Resolution，可处理任意分辨率、动态产出可变数量的视觉 token；移除 ViT 的绝对位置编码、改用 **2D-RoPE**；ViT 后接一个简单 MLP，将相邻 $2\times2$ 的 token 合并为一个（例如 $224\times224$ 图经 patch=14 编码后压缩为 66 个 token），并以 `<|vision_start|>`、`<|vision_end|>` 标记视觉段。ViT 规模约 675M。
- **Qwen2.5-VL（2025）：从零训练的高效原生 ViT。** 从头训练原生动态分辨率 ViT，引入 **Window Attention**——仅 4 层为全注意力、其余为窗口注意力（最大窗口 $8\times8$ 即 $112\times112$，小于该尺寸的区域保留原始尺度、不做填充），使计算量随 patch 数线性增长；并采用 **RMSNorm 与 SwiGLU** 使 ViT 结构与 LLM 更一致。
- **Qwen3-VL（2025）：多层次特征融合（DeepStack）。** 在保持原生动态分辨率的基础上引入 **DeepStack**，把 ViT 不同深度的多层特征融合进 LLM 的早期层；patch 尺寸由 14 调整为 16。

---

<h4 id="面试问题-视觉编码器的各次演进分别解决了什么问题？">面试问题-视觉编码器的各次演进分别解决了什么问题？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐ (3/5)**

每一次演进都针对前一代的一个具体瓶颈。

- **从固定分辨率到动态分辨率：解决"形变与细节丢失"。** Qwen-VL 的固定分辨率会对非标准长宽比的图像造成缩放形变、对高分辨率文档丢失细节；Qwen2-VL 的朴素动态分辨率让任意分辨率图按原比例转为可变 token，从根本上保留原始信息。
- **2D-RoPE 替代绝对位置编码：解决"分辨率可变下的位置表达"。** 动态分辨率意味着 token 网格尺寸不固定，固定长度的绝对位置编码不再适用；2D-RoPE 以相对方式编码二维位置，天然适配可变网格并利于外推。
- **Window Attention：解决"高分辨率下 ViT 的二次复杂度"。** 处理大图时全注意力的 $O(n^2)$ 成本难以承受；窗口注意力把大部分层限制在局部窗口内，使计算量近似线性，仅少数全注意力层保留全局视野。
- **RMSNorm/SwiGLU 对齐 LLM：解决"结构异构带来的工程复杂度"。** 让 ViT 与 LLM 采用一致的归一化与激活结构，简化整体网络、便于联合训练与优化。
- **DeepStack 多层融合：解决"单层特征细粒度不足"。** 仅用 ViT 末层特征会损失低层细节，多层融合同时保留高层语义与低层细节，强化细粒度感知与图文对齐。

<h3 id="2.介绍一下Qwen-VL系列三阶段训练范式的运作机制与各阶段数据策略">2.介绍一下Qwen-VL系列三阶段训练范式的运作机制与各阶段数据策略</h3>

<h4 id="面试问题-Qwen-VL三阶段训练的运作机制是什么？">面试问题-Qwen-VL三阶段训练的运作机制是什么？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

Qwen-VL 确立了"预训练 → 多任务预训练 → 监督微调"的三阶段训练范式，三阶段在"训练哪些模块、用什么数据、达到什么目标"上层层递进。

- **阶段一：预训练（Pretraining）。** **冻结 LLM，只训练视觉编码器与适配器**。目标是建立基础的图文对齐，让视觉特征进入语言空间。此阶段使用低分辨率（如 224）以提升吞吐。
- **阶段二：多任务预训练（Multi-task Pretraining）。** **解冻全部参数、联合训练整个模型**，并将分辨率提升（如 448）。目标是在多种视觉-语言任务上同时学习，全面提升能力。
- **阶段三：监督微调（Supervised Fine-Tuning）。** **冻结视觉编码器，训练 LLM 与适配器**，在指令与对话数据上微调，使模型具备指令跟随与多轮对话能力（得到 Qwen-VL-Chat）。

三阶段的逻辑是"先对齐、再多任务、后对齐人类指令"——先用大规模弱标注数据打好图文对齐底座，再用多任务数据扩展能力，最后用高质量指令数据塑造交互行为。

---

<h4 id="面试问题-三阶段训练各自的数据策略有何差异？">面试问题-三阶段训练各自的数据策略有何差异？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐ (3/5)**

三阶段的数据在"规模、质量、任务结构"上呈现明显的递进差异。

- **阶段一：大规模、弱标注、单一形式。** 以海量图文对为主，数据来自网络爬取，规模庞大但标注质量参差；组织形式以"图 → 文"为主，目的是用规模换对齐。
- **阶段二：多任务、结构化、较高质量。** 数据覆盖多种任务形式——图像描述、视觉问答、视觉定位（grounding/referring）、OCR、文本生成、图文交错等；并提升分辨率以适配 OCR、定位等细粒度任务。数据从"单一图文对"转为"多任务混合"，是能力广度的主要来源。
- **阶段三：指令与对话、精而少。** 使用高质量的指令跟随与多轮对话数据，规模相对小但质量高，目的是塑造模型的交互方式与指令遵循能力，而非继续灌输知识。

整体上，数据策略从"大而粗"逐步走向"小而精"，与三阶段"先打底、再拓宽、后精修"的目标一一对应。

<h3 id="3.介绍一下Qwen2-VL中M-RoPE的设计原理及如何统一位置编码">3.介绍一下Qwen2-VL中M-RoPE的设计原理及如何统一位置编码</h3>

<h4 id="面试问题-M-RoPE的设计原理是什么？">面试问题-M-RoPE的设计原理是什么？</h4>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐⭐ (5/5)**

M-RoPE（Multimodal Rotary Position Embedding，多模态旋转位置编码）是 Qwen2-VL 的核心设计之一。传统 1D-RoPE 只能编码一维序列位置，无法表达图像的二维空间与视频的时间维度。

**M-RoPE 的设计原理是把旋转位置编码分解为三个分量：时间（temporal）、高（height）、宽（width）。** 它将隐藏维度按 `mrope_section` 切分为三块，分别对时间、高、宽三个轴施加各自的旋转频率，从而用一套位置编码同时表达"第几帧、第几行、第几列"。

这样设计的一个附带收益是：图像与视频的位置 ID 数值被显著压低（不再像把二维/三维强行展平成一维那样线性累积），**有利于模型在推理时外推到更长序列**。

---

<h4 id="面试问题-M-RoPE如何统一文本、图像与视频的位置编码？">面试问题-M-RoPE如何统一文本、图像与视频的位置编码？</h4>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

M-RoPE 通过对不同模态分配不同的"时间/高/宽"ID 规则，把三种模态统一进同一套位置编码框架。

- **文本：三分量取相同 ID，退化为 1D-RoPE。** 文本 token 的时间、高、宽 ID 完全相同并逐 token 加 1，此时 M-RoPE 在功能上等价于传统 1D-RoPE，从而**完整保留语言模型的原有行为**。
- **图像：时间 ID 恒定，高/宽 ID 按位置赋值。** 同一张图内所有视觉 token 的时间 ID 保持不变，而高、宽分量按 token 在网格中的行列位置分别赋 ID（如高 ID 逐行重复、宽 ID 逐列交替）。
- **视频：时间 ID 逐帧递增，高/宽与图像一致。** 视频被视为帧序列，时间分量随帧递增，高、宽分量沿用图像的赋值方式。
- **跨模态衔接：以上一模态的最大位置 ID 加 1 初始化下一模态。** 当输入混合多种模态时，每种模态的位置编号都从前一模态最大 ID 加 1 开始，保证整段序列位置连续。

通过这套规则，文本、图像、视频被纳入同一位置编码体系，既统一了多模态位置表达，又因数值更紧凑而增强了长序列外推能力。

<h3 id="4.分析一下Qwen2.5-VL使用绝对位置坐标相比归一化坐标在目标检测中的优势">4.分析一下Qwen2.5-VL使用绝对位置坐标相比归一化坐标在目标检测中的优势</h3>

<h4 id="面试问题-Qwen2.5-VL的绝对位置坐标表示是怎么做的？">面试问题-Qwen2.5-VL的绝对位置坐标表示是怎么做的？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

在视觉定位上，Qwen2.5-VL 改变了坐标的表示方式：**直接使用图像的实际尺寸来表示检测框、点等坐标，而不做传统的坐标归一化。**

也就是说，检测框坐标取的是基于输入图像真实像素尺度的数值，而非把坐标缩放到 $[0,1]$ 或千分制等相对区间。配合 Qwen2.5-VL 的原生动态分辨率（图像按原始尺度转为可变 token），模型在训练时见到的坐标与图像的真实尺度一致，因此**能够直接学习图像的真实尺度信息**。

---

<h4 id="面试问题-绝对坐标相比归一化坐标在目标检测中有何优势？">面试问题-绝对坐标相比归一化坐标在目标检测中有何优势？</h4>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐ (3/5)**

绝对坐标相比归一化坐标，在目标检测与定位任务上具有几方面优势。

- **直接学习真实尺度与空间关系。** 归一化会把坐标压到与图像尺寸无关的相对区间，丢失"物体实际有多大、相距多远"的尺度信息；绝对坐标保留了真实像素尺度，使模型能学到物体的真实大小与空间关系，这对检测与定位尤为关键。
- **与原生分辨率天然一致。** Qwen2.5-VL 本就按图像原始尺度处理输入，绝对坐标与这一表示直接对齐，避免了"输入按原尺度、标签却被归一化"之间的尺度错配。
- **更利于细粒度定位。** 在小目标、密集排布的场景中，绝对坐标对位置的刻画更直接，有助于提升检测框的精度。

其代价在于：绝对坐标的数值范围随图像尺寸变化，需要模型适应不同量级的坐标取值；Qwen2.5-VL 通过原生分辨率训练与大规模定位数据来消化这一点，从而在检测、定位类任务上获得更强表现。

<h3 id="5.介绍一下Qwen2.5-VL的动态FPS采样与3D patch划分策略">5.介绍一下Qwen2.5-VL的动态FPS采样与3D patch划分策略</h3>

<h4 id="面试问题-动态FPS采样如何工作？为何需要？">面试问题-动态FPS采样如何工作？为何需要？</h4>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

动态 FPS（Frames Per Second）采样是 Qwen2.5-VL 把"动态分辨率"从空间维扩展到时间维的关键设计。

**它的工作方式是：训练时以变化的帧率对视频采样，并把 M-RoPE 的时间维 ID 直接与绝对时间对齐。** 模型据此能通过"时间维 ID 之间的间隔"感知时间的快慢节奏，从而在不同 FPS 采样率的视频之间保持一致的时间对齐。

**为何需要它：** 其一，真实视频帧率各异，固定帧率采样难以兼顾长短视频与快慢动作；动态 FPS 让模型适应不同采样率，增强鲁棒性。其二，仅按"帧序号"编码时间（如 Qwen2-VL）无法反映内容变化的速度或事件的绝对时刻；把时间 ID 与绝对时间对齐后，模型能理解"事件发生在第几秒"，且这一对齐不引入额外计算开销。

---

<h4 id="面试问题-3D patch（时序分块）如何处理视频输入？">面试问题-3D patch（时序分块）如何处理视频输入？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐ (3/5)**

为高效处理视频，Qwen2.5-VL 采用 3D patch（时序分块）的方式，把时间维一并纳入 patch 划分。

**做法是把相邻的连续帧绑定为一个时序 patch（temporal patch size = 2，即每两帧合并），patch 形状为时间 × 高 × 宽（如 $2\times14\times14$）。** 这样视频不再是逐帧独立编码，而是以"时空小块"为单位编码，既压缩了时间维的 token 数量，又显式保留了短时的时序信息。

配合动态 FPS 与绝对时间对齐，3D patch 让模型在可控的 token 预算下处理较长视频：时序分块降低冗余、绝对时间对齐保证时间语义，两者共同支撑了对长视频的高效理解与时刻定位。

<h3 id="6.分析一下Qwen2.5-VL在后训练阶段如何结合监督微调与直接偏好优化">6.分析一下Qwen2.5-VL在后训练阶段如何结合监督微调与直接偏好优化</h3>

<h4 id="面试问题-Qwen2.5-VL后训练的SFT与DPO两阶段如何运作？">面试问题-Qwen2.5-VL后训练的SFT与DPO两阶段如何运作？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

Qwen2.5-VL 的后训练由两阶段组成：监督微调（SFT）与直接偏好优化（DPO），**两阶段均冻结视觉编码器（ViT）**，只更新语言侧。

- **SFT 阶段：在多样多模态数据上做监督微调。** 数据涵盖图文对、视频、纯文本，来源包括通用 VQA、拒绝采样（Rejection Sampling）以及文档/OCR、Grounding、视频、Agent 等专项数据集。该阶段让模型学会各类任务技能与指令跟随，数据在文本与多模态之间较为均衡（约各占一半，规模在百万量级）。
- **DPO 阶段：用偏好数据对齐人类偏好。** DPO 仅在图文与纯文本数据上进行，使用偏好对（preferred/rejected）来对齐模型与人类偏好，且**每个样本只处理一次**以保证优化效率。

两阶段串联的结果是：先用 SFT 把任务能力和指令遵循补齐，再用 DPO 低成本地把输出风格与人类偏好对齐。

---

<h4 id="面试问题-为何采用SFT与DPO的组合？两者各自承担什么作用？">面试问题-为何采用SFT与DPO的组合？两者各自承担什么作用？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐ (3/5)**

SFT 与 DPO 解决的是两类不同的问题，组合使用能在能力与对齐之间取得平衡。

- **SFT 负责"会做"。** 通过在带标准答案的多任务数据上模仿学习，让模型掌握 VQA、文档解析、定位、视频理解、Agent 等具体技能，并建立指令跟随的基本格式。它决定了模型的能力边界。
- **DPO 负责"做得合人意"。** SFT 只能学到"标准答案"，却难以表达"两个都对的回答中哪个更好"。DPO 用偏好对直接优化模型对"更优回答"的倾向，且无需训练独立的奖励模型与复杂的在线采样，工程上比 PPO 类方法更轻量、更稳定。
- **冻结 ViT 的考量。** 后训练阶段视觉感知能力已在预训练中建立，冻结 ViT 既能保护视觉表示不被少量后训练数据扰动，又能降低训练开销，让后训练聚焦于语言侧的任务适配与偏好对齐。

这一组合以较低成本完成了"任务能力 + 偏好对齐"的双重目标，是当前多模态后训练的常见范式。

<h3 id="7.解释一下Qwen3-VL中MRoPE-Interleave的设计原理及其作用">7.解释一下Qwen3-VL中MRoPE-Interleave的设计原理及其作用</h3>

<h4 id="面试问题-MRoPE-Interleave的设计原理是什么？">面试问题-MRoPE-Interleave的设计原理是什么？</h4>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

MRoPE-Interleave（Interleaved-MRoPE）是 Qwen3-VL 对多模态旋转位置编码的改进。早期的 M-RoPE 把隐藏维度按时间、高、宽**分块（chunked）**切分——前一段维度全给时间、中间给高、后面给宽。这样的分块方式存在一个问题：时间分量集中在低频（或某一段）维度，频率在各轴之间分配不均。

**MRoPE-Interleave 的设计是按通道交错（interleave）分配三个轴的频率。** 它不再整段切分，而是让各通道循环轮流承担时间、高、宽（如 $t, h, w, t, h, w, \dots$），使**全频段在时间、高、宽三个轴上均匀分配**。这样每个注意力头与通道都能同时观察到时间与空间信息，位置表达更一致、频率利用更充分。对于纯文本数据，该方案退化为标准 RoPE（非时间轴置零），从而完整保留预训练语言模型的先验。

---

<h4 id="面试问题-MRoPE-Interleave相比M-RoPE带来哪些作用？">面试问题-MRoPE-Interleave相比M-RoPE带来哪些作用？</h4>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐ (3/5)**

相比分块式的 M-RoPE，交错式分配带来几方面收益。

- **更强的长时程视频推理。** 时间频率不再被局限在某一段维度，而是贯穿全频段，使模型对跨越长时间的事件能保持稳定的时序建模，长视频推理能力增强。
- **更好的时空一致性与跨模态泛化。** 每个通道都同时感知时间与空间，位置编码在三轴间更协调，对图像、视频、grounding 等任务的时空对齐更稳健。
- **更长的上下文外推。** Interleaved-MRoPE 的位置 ID 增长比普通 RoPE 更慢，因此在做长度外推时只需较小的缩放因子；Qwen3-VL 原生支持 256K 上下文，并可借助 YaRN 扩展到约 1M。
- **保留纯文本能力。** 对纯文本退化为标准 RoPE，几乎零工程代价地保护了语言模型的原有能力。

<h3 id="8.介绍一下Qwen3-VL的DeepStack技术如何实现ViT多层次特征融合">8.介绍一下Qwen3-VL的DeepStack技术如何实现ViT多层次特征融合</h3>

<h4 id="面试问题-DeepStack如何实现ViT多层次特征融合？">面试问题-DeepStack如何实现ViT多层次特征融合？</h4>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

DeepStack 是 Qwen3-VL 用来融合 ViT 多层次特征的机制。传统做法只取 ViT 最后一层的特征送入 LLM，会丢失中间层的细节信息。

**DeepStack 的做法是：从 ViT 的多个不同深度层（如 $l_1, l_2, l_3$）分别抽取 patch 特征，各自经独立的 MLP merger 投影到 LLM 的隐藏维度，再以残差相加的方式注入 LLM 的前若干层（如第 1–3 层）。** 关键在于，这种注入是把不同层的视觉特征叠加进对应的 LLM 早期层隐藏状态，**不增加序列长度**——既引入了多层视觉语义，又不带来额外的 token 开销。

需要注意的是，由于 DeepStack 把视觉特征融合进了 LLM 的早期隐藏状态，Qwen3-VL 的语言部分不再是纯文本通路，而是从早期层起就携带视觉信息。

---

<h4 id="面试问题-多层次特征融合为何能提升细粒度理解与图文对齐？">面试问题-多层次特征融合为何能提升细粒度理解与图文对齐？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐ (3/5)**

多层次特征融合之所以有效，源于 ViT 不同深度层编码的信息粒度不同。

- **低层与高层语义互补。** ViT 浅层更多保留边缘、纹理、局部结构等低层细节，深层更偏向类别、语义等高层抽象；只用末层会丢失细节，多层融合让模型**同时获得低层细节与高层语义**。
- **提升细粒度与定位精度。** 文档、小目标、密集排布等任务高度依赖低层细节，多层特征的引入直接增强了细粒度感知与 grounding 精度。
- **强化图文对应。** 在 LLM 早期层就注入丰富的多层视觉信息，有助于视觉与文本在更早阶段建立更紧密的对应关系，从而改善图文对齐质量。

<h3 id="9.说明一下Qwen3-VL将T-RoPE升级为文本时间戳对齐机制的实现方式及好处">9.说明一下Qwen3-VL将T-RoPE升级为文本时间戳对齐机制的实现方式及好处</h3>

<h4 id="面试问题-文本时间戳对齐机制的实现方式是什么？">面试问题-文本时间戳对齐机制的实现方式是什么？</h4>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐ (3/5)**

文本时间戳对齐（Text-Timestamp Alignment）是 Qwen3-VL 在视频时序建模上的改进。此前的 T-RoPE 把时间信息编码在位置编码的时间分量中、与帧索引（绝对时间对齐的帧位置）绑定。

**文本时间戳对齐的实现方式是：在每组视频帧之前插入一个显式的文本时间戳 token（如 `<3.0 seconds>` 或 `<00:03:00>`），用标准的文本嵌入机制进行编码。** 也就是说，时间信息不再仅依赖位置编码，而是被显式地写成可读的文本标记，直接锚定每段画面对应的时刻。训练时还会交替使用不同的时间戳格式（秒、时:分:秒等），以增强模型对多种时间表达的鲁棒性。

---

<h4 id="面试问题-相比T-RoPE，文本时间戳对齐带来哪些好处？">面试问题-相比T-RoPE，文本时间戳对齐带来哪些好处？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐ (3/5)**

把时间从"隐式的位置编码"改为"显式的文本标记"，带来更精确、更灵活的时序能力。

- **帧级精确的事件定位。** 显式时间戳把每段画面与具体时刻直接关联，使模型能做到帧级精度的事件定位与秒级的视频索引，而非仅依赖位置 ID 的相对间隔去推断时刻。
- **更强的时序推理。** 时间以可读文本形式参与建模，模型可以像处理普通文本一样对时刻进行比较、计算与表述，时序推理更直接。
- **格式灵活、表达统一。** 支持秒、时:分:秒等多种时间格式并在训练中交替，模型既能理解也能按不同格式输出时间，适配多样的下游需求。
- **实现简洁。** 复用标准文本嵌入机制即可注入时间信息，无需为时序定位额外设计专用结构或位置编码分量。
