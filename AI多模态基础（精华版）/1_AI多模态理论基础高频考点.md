<h3 id="1.介绍一下多模态与语言大模型的区别">1.介绍一下多模态与语言大模型的区别</h3>

<h4 id="面试问题-多模态大模型与LLM在架构上有哪些关键差异？">面试问题-多模态大模型与LLM在架构上有哪些关键差异？</h4>

**难度评分：⭐⭐ (2/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

从本质上看，多模态大模型（MLLM）并非独立于语言大模型（LLM）的全新体系，而是在 LLM 基础上扩展了视觉、听觉等多模态感知能力。架构差异集中体现在两个层面。

**模态边界不同。** LLM 仅接受文本输入并生成文本输出，GPT-3.5、LLaMA 3 均属此类；整体是一个纯粹的 Decoder-only Transformer，以 token 序列作为输入和输出。MLLM 则能够同时处理文本、图像、音频甚至视频——例如 LLaVA、Qwen-VL、BLIP-2 可以输入一张图并回答"这是什么"。

**架构新增两个关键模块。** 相比 LLM 的单一 Transformer 主干，MLLM 在输入端引入了两层额外组件：

- **模态编码器**：例如 CLIP-ViT-L 将图像编码为 256 或 576 个 1024 维的视觉 token，Whisper 编码器将音频编码为 1500 个语音 token。
- **投影器（Projector）**：负责将非文本模态的向量映射到 LLM 的文本嵌入空间。例如 LLaVA-1.5 采用两层 GELU-MLP，将 CLIP-ViT-L 的 1024 维输出投射到 LLaMA-7B 的 4096 维隐藏空间。

LLM 基座在训练时可以冻结（LLaVA 1.0 阶段一即冻结）、也可以参与联合微调（LLaVA 1.5 阶段二解冻 LLM）。三段式"编码器 → 投影器 → LLM"已成为当前 VLM 的事实标准架构。

**LLM 与 MLLM 架构对照：**

| 维度 | LLM | MLLM |
|---|---|---|
| 输入模态 | 仅文本 | 文本 + 图像 + 视频 + 音频 |
| 输出模态 | 文本 | 文本（部分支持图/音/视频生成） |
| 主干结构 | Decoder-only Transformer | 模态编码器 + 投影器 + LLM 主干 |
| 关键新增模块 | 无 | 视觉编码器（CLIP-ViT、SigLIP）+ 音频编码器（Whisper、Mimi）+ 投影器（MLP、Q-Former、Pixel Shuffle） |
| 输入 token 形式 | 文本 token | 文本 token + 模态 token（拼接为统一序列） |
| 代表模型 | GPT-5、DeepSeek-V3、LLaMA 3、Qwen3.6 | Qwen3-VL-235B-A22B-Thinking、InternVL3.5-241B-A28B、Gemini 3.5 Flash、LLaVA |

<h4 id="面试问题-多模态模型的训练数据与LLM有什么本质不同？">面试问题-多模态模型的训练数据与LLM有什么本质不同？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

LLM 与 MLLM 的数据差异不仅体现在"多了一种模态"，而是体现在"数据形态"与"训练组合比例"两个层面。

**LLM 的数据形态是纯文本序列。** 预训练依赖书籍、网页、论文等海量纯文本语料，规模通常达万亿 token 级别——例如 LLaMA 2 使用 2 万亿 token、Common Crawl + C4 + RedPajama 等是典型组合，数据本身即标签（next-token prediction）。

**MLLM 的数据形态是跨模态配对数据。** 核心训练数据是"模态 A + 模态 B"的配对，典型例子：

- **图文对**：LAION-5B 包含约 58 亿组图文配对、LAION-COCO 6 亿组、CC12M 1200 万组，主要通过互联网爬取 alt-text 获得，**标注质量参差不齐但规模庞大**。
- **视频-字幕对**：WebVid-10M 等。
- **指令微调数据**：LLaVA-Instruct-150K、ShareGPT4V、ALLaVA-4V 等 GPT-4V 蒸馏生成的高质量对话数据，规模在 15万~500万条之间。

**关键工程细节是数据配比。** 多模态训练通常**不会只用图文对**，而是以一定比例（常见 10%~30%）混入纯文本数据，以防止 LLM 基座的语言能力在多模态训练过程中发生退化——这一现象在 LLaVA 1.0 的早期实验中被明确观察到，Qwen-VL 训练配方中也专门保留了纯文本比例。数据质量与配比的权衡，是多模态训练最关键的工程点之一。

<h4 id="面试问题-多模态模型与LLM在应用能力边界上有何差异？">面试问题-多模态模型与LLM在应用能力边界上有何差异？</h4>

**难度评分：⭐⭐ (2/5)  |  考察频率：⭐⭐⭐ (3/5)**

两者在应用能力上的差异可以用一句话概括：**LLM 覆盖"语言相关任务"，MLLM 覆盖"需要跨模态信息整合的任务"，但后者的覆盖是"上界扩展"，并不代表单点性能全面超越专家模型**。

**LLM 的能力核心是语言任务。** 文本生成、代码编写、文本摘要、翻译、对话等，代表模型包括 GPT-4、Claude、DeepSeek-V3。这些任务的共性是"输入与输出都是文本序列"。

**MLLM 的能力扩展到跨模态场景。** 典型落地包括：

- **以图搜物**：电商平台的"拍照搜商品"，需要同时理解图像语义和商品库文本描述。
- **医学影像辅助诊断**：结合 CT/MRI 图像与病历文本生成诊断建议，代表如 Med-PaLM M。
- **短视频内容合规审核**：综合画面、配文、语音进行多模态违规判断。
- **文档/OCR 理解**：解析财报表格、流程图、发票等富结构文档，代表如 GPT-5、InternVL3.5、Qwen3-VL。

**值得澄清的两个常见误解：**

- **"MLLM 一定比 LLM 强"是错的。** 在纯文本任务上，MLLM 通常会比同规模 LLM 稍弱，因为一部分模型容量被视觉对齐占用。这也是为何 GPT-4o 在纯文本榜单上与 GPT-4 Turbo 相近而非显著超越。
- **"MLLM 能替代所有视觉专家模型"也是错的。** 在 OCR、细粒度分类、目标检测等单点任务上，通用 MLLM 通常比不过专用模型（如 PaddleOCR、EVA-CLIP 下游微调模型）。MLLM 的真正价值在于**综合推理能力**——需要"看懂图像 + 理解问题 + 结合常识生成答案"时才不可替代。

<h3 id="2.介绍一下多模态词嵌入的定义">2.介绍一下多模态词嵌入的定义</h3>

<h4 id="面试问题-多模态词嵌入相比传统词嵌入的本质区别是什么？">面试问题-多模态词嵌入相比传统词嵌入的本质区别是什么？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

理解多模态词嵌入，需要先回顾传统词嵌入的基本概念。Word2Vec（2013）、GloVe、BERT 等模型的核心任务是将每个词映射为一个固定维度的向量（典型 300~768 维），使语义相近的词在向量空间中具有较小的距离。例如，"猫"与"猫咪"的向量距离较近，而"猫"与"汽车"的向量距离较远。

**传统词嵌入的根本局限是"单模态封闭"。** 文本嵌入、图像嵌入、音频嵌入分属三个互不相通的向量空间，无法直接比较——CLIP 之前，没有任何方法能计算"文本'猫'"与"一张猫的照片"之间的相似度。

**多模态词嵌入（也称跨模态嵌入）的本质突破是：将文本、图像、音频等不同类型的信息统一映射到同一个向量空间中。** 扩展是在两个方向上同时进行的：

- **扩展输入的模态**：不再仅支持文本，而是任意模态都能转换为同一空间中的向量。
- **扩展相似度的语义**：原本"语义相近 → 向量接近"的内核不变，但"语义"的含义从"同义词"扩展到"跨模态同一概念"——一段"猫叫声"音频、一张猫的照片、文本"猫"，三者的向量都应当聚集在同一区域，并远离"汽车"相关的向量。

**代表性工作 CLIP** 在 4 亿图文对上训练，首次大规模验证了"统一向量空间"的可行性，并将之用于零样本图像分类——无需任何标注的微调数据，仅通过计算"图像向量"与"一组候选文本向量"的相似度即可完成分类。这一能力是传统单模态词嵌入无法实现的。

<h4 id="面试问题-多模态词嵌入需要满足哪些核心条件？如何形式化描述？">面试问题-多模态词嵌入需要满足哪些核心条件？如何形式化描述？</h4>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

"统一向量空间"这一目标可以进一步拆解为两个必须同时满足的条件。

- **模态内一致性**：同一模态中语义相近的内容在向量空间中彼此接近。例如两张不同角度的猫照片应当向量接近，这与传统词嵌入的要求一致。
- **跨模态对齐性**：不同模态中语义一致的内容也应在向量空间中彼此接近。例如文本"猫"、一张猫的照片、一段猫叫的音频，三者的向量应当聚集在一起，并远离"汽车"相关内容的向量。**跨模态对齐性才是多模态词嵌入区别于传统词嵌入的核心要求**。

**形式化表述如下：** 文本编码器 $f_t(\cdot)$ 和图像编码器 $f_v(\cdot)$ 分别将文本 $t$ 和图像 $v$ 映射到同一个 $d$ 维空间，对于匹配的图文对 $\langle t, v \rangle$ 和不匹配的 $\langle t, v' \rangle$，需要满足：

$$\text{sim}(f_t(t), f_v(v)) > \text{sim}(f_t(t), f_v(v'))$$

该不等式正是对比学习损失函数的核心思想——拉近匹配对的距离、推远不匹配对的距离。**CLIP 的 InfoNCE 损失即基于该原理**，通过在一个 batch（典型 32768）内互相作为负样本计算相似度矩阵，用交叉熵拟合对角线上的匹配对。InfoNCE 在多模态预训练中的具体形式与公式展开，详见本节"预训练阶段的核心目标与训练任务"子问题。

**为什么两个条件缺一不可？** 只满足模态内一致性而缺失跨模态对齐（如 Word2Vec + 单独训练的 ResNet），无法支撑跨模态检索；只强调跨模态对齐而忽略模态内一致性（如训练数据高度偏斜），则会导致"向量空间塌陷"——不同语义的样本被错误拉近，下游所有相似度度量失效。CLIP 在训练时通过**大 batch + 对称对比损失**同时保证两个条件，这也是其成功的关键工程设计。

多模态词嵌入是众多下游任务的理论基础：**跨模态检索**依赖于嵌入向量之间的相似度计算，**多模态理解**依赖于对齐后的嵌入进行语义交互，**跨模态生成**（如 Stable Diffusion 的文本编码器）则依赖于在嵌入空间中完成条件控制。

<h3 id="3.介绍一下多模态中预训练和微调的区别">3.介绍一下多模态中预训练和微调的区别</h3>

<h4 id="面试问题-多模态预训练阶段的核心目标与训练任务是什么？">面试问题-多模态预训练阶段的核心目标与训练任务是什么？</h4>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐⭐ (5/5)**

多模态大模型的训练遵循"先预训练、再微调"的两阶段范式，这一点与纯文本 LLM 一致。但多模态训练的特殊之处在于，**"跨模态对齐"贯穿整个训练流程**。

**预训练阶段的核心目标可以用一个词概括：对齐。** 即让模型学会将图像、文本、音频等不同模态的信息映射到同一语义空间中，预训练所使用的数据以大规模弱标注配对数据为主，例如 LAION-5B 包含约 58 亿组图文对，主要通过互联网爬取获得，标注质量参差不齐但规模庞大。同时会混入一定比例的纯文本语料，以维持模型的语言能力。

**预训练阶段主要包含三类训练任务：**

- **跨模态对比学习（ITC，Image-Text Contrastive）**：采用 InfoNCE 损失函数，拉近匹配图文对的距离、推远不匹配对的距离。代表模型如 CLIP、ALIGN。
- **图文匹配（ITM，Image-Text Matching）**：输入一组图文对，由模型判断二者是否匹配，本质上是一个二分类任务。BLIP、ALBEF 在 ITC 基础上叠加 ITM 以学到更精细的跨模态交互。
- **掩码建模（MLM / MIM）**：随机遮蔽文本中的部分 token 或图像中的部分 patch，要求模型预测被遮蔽的内容，从而学习模态内的语义表示。BEIT-3、Flamingo 等模型采用此类任务。

**InfoNCE 损失公式：**

![ITC Loss](https://latex.codecogs.com/svg.image?\mathcal{L}_{\text{ITC}}%20=%20-\frac{1}{2N}\sum_{i=1}^N%20\left[%20\log\frac{e^{\text{sim}(t_i,%20v_i)/\tau}}{\sum_{j=1}^N%20e^{\text{sim}(t_i,%20v_j)/\tau}}%20+%20\log\frac{e^{\text{sim}(t_i,%20v_i)/\tau}}{\sum_{j=1}^N%20e^{\text{sim}(t_j,%20v_i)/\tau}}%20\right])

其中 $\tau$ 为温度系数（典型取 0.07），$N$ 为批次内样本数（CLIP 默认 32768）。公式前后两项分别对应"图像→文本"和"文本→图像"两个方向的对比损失，对称设计保证双向对齐质量。

**预训练的计算代价十分高昂——**需要对编码器、投影层、LLM 基座进行全参数或大规模参数更新。千亿参数级别的模型，单次预训练成本可达**数百万美元**，训练周期通常为数周乃至数月。这是为什么"从零预训练一个多模态基座"仅由头部厂商承担。

<h4 id="面试问题-多模态微调阶段的核心目标与常用方法是什么？">面试问题-多模态微调阶段的核心目标与常用方法是什么？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐⭐ (5/5)**

**微调阶段的核心目标是将通用基座模型适配到特定下游任务。** 例如针对医疗影像检索任务，可使用 10 万条标注好的影像-报告配对数据进行微调；针对某电商平台的商品识别场景，则用平台特有的商品图文对做领域迁移。

**训练目标与下游任务直接对齐：**

- **视觉问答（VQA）**：优化 next-token prediction 损失，以问答对话的形式微调，代表数据集 VQAv2、GQA、ScienceQA。
- **跨模态检索**：优化对比损失或排序损失（triplet loss），常见于电商搜索场景。
- **图像描述生成（Image Captioning）**：优化语言模型的交叉熵损失，代表数据集 COCO Captions、Flickr30k。
- **文档理解**：混合 VQA 与 OCR-aware 损失，代表数据集 DocVQA、ChartQA。

**微调阶段注重计算效率**，当前主流方案有三类：

- **参数高效微调（PEFT）**：采用 LoRA、Adapter、Prefix-Tuning 等方法，**仅更新不到 1% 的参数**。LoRA 是当前最主流的方案，几乎所有开源 VLM 的社区微调都基于 LoRA；2024~2025 年涌现的 **DoRA**（权重分解 LoRA，进一步缩小与全参微调的差距）、**LoftQ**（低比特量化 + LoRA 联合初始化）、**PiSSA**（主奇异值初始化加速收敛）等改进方案，在 Qwen3-VL、InternVL3.5 等新一代基座的社区微调中已被广泛采用。
- **冻结 + 仅训映射器**：冻结 LLM 基座与视觉编码器，仅训练投影层（几 M 参数）。这是 LLaVA 1.0 阶段一的做法，适合算力极度受限或只需小幅领域迁移的场景。
- **全参数微调**：在特定高价值场景（如医疗、金融）下对 LLM 基座做全参数更新，但需要配合"混入通用数据"来缓解灾难性遗忘。

**成本对比：** 微调成本约为预训练的**千分之一到百分之一**——训练周期通常为数小时至数天，单卡或小规模 8 卡 A100/H100 即可完成。这也是为什么开源社区能贡献大量基于 LLaVA、Qwen-VL 的领域变体。

<h4 id="面试问题-预训练与微调在通用性、成本、能力保持上有何关键权衡？">面试问题-预训练与微调在通用性、成本、能力保持上有何关键权衡？</h4>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

两个阶段各有侧重，理解其权衡是选择训练路线的关键。

**维度一：通用性 vs 专用性。** 预训练产出的模型具备较强的通用能力——LLaVA-1.5、Qwen-VL、InternVL 等基座模型在 MMBench、MME、ScienceQA 等通用榜单上表现均衡，但在垂直领域（医疗影像、工业质检）直接使用往往难以达到工业部署标准。微调后的模型在目标任务上性能显著提升，但通用能力可能出现退化。

**维度二：成本差距接近三个数量级。** 预训练单次数百万美元、训练周期数周；微调成本千分之一到百分之一、训练周期数小时到数天。这一成本差距直接决定了"谁做预训练 vs 谁做微调"的产业分工——基座模型由少数头部团队生产，下游应用团队只做微调。

**维度三：灾难性遗忘现象。** 这是微调阶段最需要警惕的问题——在专用任务上训练后，模型在通用任务上的表现会大幅下降。例如在医疗影像上全参数微调后，模型可能对"描述一下这张自然风景照"的通用能力严重退化。

**缓解灾难性遗忘的常见策略：**

- **混入通用数据**：微调数据集中掺入 10%~30% 的通用指令数据（如 LLaVA-Instruct），维持模型的基础能力。
- **LoRA 冻结基座**：由于 LoRA 只修改低秩增量，基座的原始权重完全保留，遗忘程度显著低于全参微调。
- **选择性冻结**：冻结视觉编码器和 LLM 基座，仅微调投影层和部分 top layer，最大限度保留预训练知识。

**工程选型建议：** 如果下游任务与预训练分布差异不大（如通用 VQA、图像描述），优先使用 LoRA + 少量领域数据；如果任务分布差异较大（如医疗、工业场景），需要全参数微调但务必混入通用数据；如果仅做轻量领域迁移（如风格定制），甚至可以只训投影层。

**预训练 vs 微调多维对照：**

| 维度 | 预训练 | 微调 |
|---|---|---|
| 核心目标 | 跨模态对齐 + 通用语义建模 | 适配特定下游任务/领域 |
| 数据规模 | 数十亿~百亿级配对数据（LAION-5B、DataComp-1B） | 数千~百万条任务数据（VQAv2、DocVQA、领域私有数据） |
| 数据来源 | 互联网弱标注爬取 + 合成 caption（蒸馏） | 人工标注 + 蒸馏 + 领域专家标注 |
| 训练参数量 | 编码器 + 投影器 + LLM 基座全量或大规模更新 | PEFT < 1%（LoRA/DoRA）/ 仅投影器 ~M 级 / 全参微调 |
| 单次成本 | 数百万~千万美元 | 数百~数千美元 |
| 训练周期 | 数周~数月 | 数小时~数天 |
| 算力需求 | 千卡 A100/H100/H200 集群 | 单卡或 8 卡 A100/H100 |
| 主要风险 | 数据配比失衡导致能力失衡、训练崩溃 | 灾难性遗忘 |
| 产业角色 | 头部团队（OpenAI、Google、Anthropic、阿里、上海 AI Lab、智谱） | 应用厂商、开源社区、垂直行业团队 |

<h3 id="4.介绍一下多模态解决的代表性任务">4.介绍一下多模态解决的代表性任务</h3>

<h4 id="面试问题-多模态任务可以如何整体分类？分类依据是什么？">面试问题-多模态任务可以如何整体分类？分类依据是什么？</h4>

**难度评分：⭐⭐ (2/5)  |  考察频率：⭐⭐⭐ (3/5)**

多模态技术所涵盖的任务类型十分广泛，**最清晰的分类视角是"输入输出模态的关系"**——即任务的输入属于哪种模态、输出属于哪种模态、两者之间是检索、转换还是整合。按此视角可归纳为四个主要类别：

- **跨模态检索（Cross-modal Retrieval）**：输入一种模态、输出另一种模态中的匹配结果。典型如文搜图、图搜文。
- **跨模态生成（Cross-modal Generation）**：输入一种模态、生成另一种模态的全新内容。典型如文生图、文生视频、TTS。
- **多模态理解与推理（Multimodal Understanding & Reasoning）**：同时接收多种模态作为输入，输出一个综合理解或推理结果（通常是文本）。典型如 VQA、图像描述。
- **跨模态对齐与转换（Cross-modal Alignment & Translation）**：实现同一内容在不同模态间的转换或同步。典型如风格迁移、音画同步。

**为什么用"输入输出模态关系"作为分类依据？** 主要有两个工程原因：

- **技术路线差异大**：检索任务依赖对比学习嵌入空间（CLIP 系），生成任务依赖扩散模型或自回归解码器（SD、Sora、GPT-4o 系），理解推理任务依赖 VLM 三段式架构（LLaVA 系），对齐转换任务依赖专门的对齐网络（Wav2Lip 等）。分类即代表了"技术选型"。
- **评测基准不同**：检索看 R@1/R@5，生成看 FID/CLIP-Score，理解推理看 VQA Accuracy/MMBench，对齐看 Sync 指标。分类清晰了才能找对评测基准。

按其他维度（如按模态组合分为图文/视频/音频等）分类也可以，但会导致技术边界模糊——因此工业界与学术界通常采用上述"输入输出关系"视角。

<h4 id="面试问题-四类代表性任务各自的典型落地场景与技术特征是什么？">面试问题-四类代表性任务各自的典型落地场景与技术特征是什么？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

下面逐类分析各自的落地场景与技术核心。

**第一类：跨模态检索。** 最常见的形式是图文检索——电商平台的"拍照搜商品"功能属于以图搜文/搜商品（代表如淘宝图搜、京东拍照购），搜索引擎的图片搜索属于以文搜图。视频检索的典型场景是安防系统，通过输入文字描述定位监控视频中的目标片段。**技术特征：** 依赖统一嵌入空间（CLIP 或其中文版 Chinese-CLIP、图搜领域用 EVA-CLIP），推理时只需计算相似度，延迟低、可大规模部署，**是最早实现产业化落地的多模态任务**。

**第二类：跨模态生成。** 当前 AIGC 领域最活跃的方向，代表性任务与代表作品包括：

- **文本生成图像**：Midjourney、SD 3.5、DALL·E 3、Flux.2（20~80B）、Z-Image（6B Single-Stream DiT）、Qwen-Image。
- **文本生成视频**：Sora 2、Veo 3.1（4K@60fps + 原生音频）、Kling 3.0（单条 3 分钟）、Wan 2.6、Runway Gen-4.5、Seedance 2。
- **图像描述生成**：Qwen3-VL-235B-A22B-Thinking、InternVL3.5-241B-A28B、Gemini 3.5 Flash、GPT-5。
- **文字转语音（TTS）**：ElevenLabs、Voxtral TTS（3 秒克隆 + 9 语种）、OmniVoice（600+ 语种）、CosyVoice、GPT-4o 语音。
- **语音转文字（ASR）**：Whisper、Paraformer、SenseVoice。
- **语音驱动数字人**：SadTalker、EchoMimic、Hedra、HeyGen。

**技术特征：** 主流采用扩散模型（图像/视频）或自回归解码器（语音/对话），训练成本远高于检索类任务，但创造价值最直接。**各类生成器的具体结构、训练目标与推理加速方法详见 5.2 节"多模态生成器"。**

**第三类：多模态理解与推理。** 典型任务是视觉问答（VQA）：输入一张图像及一个相关问题（如"图中的猫戴了帽子吗？"），模型生成对应答案。**工业落地场景：**

- **短视频内容合规审核**：综合画面、配文、语音判断内容是否违规，抖音、快手等平台均有大规模部署。
- **医疗辅助诊断**：结合 CT 影像与病历文本进行辅助诊断，代表如 Med-PaLM M、RadFM。
- **文档理解**：解析财报、发票、合同等富结构文档，代表如 InternVL3.5、Qwen3-VL 在 DocVQA、ChartQA、TableVQA 上的工业落地。
- **GUI Agent**：基于多模态理解操控浏览器、桌面 App，代表如 Qwen3-VL Thinking、InternVL3.5 的 GUI 交互能力、Gemini 3.5 Flash 的 Agent 行动。

**技术特征：** 依赖三段式 VLM 架构（编码器 + 投影器 + LLM），LLaVA 系、Qwen-VL 系（最新 Qwen3-VL）、InternVL 系（最新 InternVL3.5，引入 Cascade RL 与 Visual Resolution Router）、Gemini 3.5、GPT-4o/GPT-5 等均主攻此类任务；2026 年起开始向**"统一多模态"（UMM，理解 + 生成同栈）**演进，代表如 InternVL-U（4B 单一框架支持理解/推理/生成/编辑）、Janus-Pro、BLIP3-o、Skywork UniPic。

**第四类：跨模态对齐与转换。** 代表性任务：

- **图像风格迁移**：将照片转换为梵高风格的画作，代表如 Style Transfer、AdaIN、InstantStyle。
- **音画对齐（唇形同步）**：将语音与视频中人物的唇形同步，代表如 Wav2Lip、SadTalker，广泛应用于视频配音、数字人直播、外语配音自动化。
- **跨语言对齐**：多语言多模态模型（如 MuMu、PaLI-X）实现同一图像在不同语言描述下的一致性。

**技术特征：** 此类任务常作为其他多模态任务的中间环节（如视频生成中的音画同步），单独部署时通常采用专用模型而非通用 VLM，以获得更精确的时序/空间控制。

**四类多模态任务对照：**

| 任务类型 | 输入→输出关系 | 主流技术路线 | 评测指标 | 代表落地场景 |
|---|---|---|---|---|
| 跨模态检索 | 一种模态 → 另一种模态匹配结果 | 双塔编码器 + 对比学习 | R@1 / R@5 / R@10 | 拍照搜商品、图片搜索、安防视频检索 |
| 跨模态生成 | 一种模态 → 另一种模态全新内容 | 扩散 / 流匹配 / 自回归 / Vocoder | FID / FVD / CLIP-Score / MOS | 文生图/视频/语音、数字人 |
| 多模态理解与推理 | 多种模态 → 文本回答 | 三段式 VLM（编码器+投影器+LLM） | VQA Accuracy / MMBench / MMMU / DocVQA | VQA、医疗诊断、文档理解、GUI Agent |
| 跨模态对齐与转换 | 同一内容在模态间转换/同步 | 专用对齐网络 / Adapter / 风格 ControlNet | Sync / LSE-D / 风格相似度 | 音画同步、风格迁移、配音 |
