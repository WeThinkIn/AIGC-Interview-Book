<h3 id="1.Qwen-VL系列视觉编码器经历了哪些关键演进，各解决什么问题">1.Qwen-VL系列视觉编码器经历了哪些关键演进，各解决什么问题</h3>

<h4 id="面试问题-从固定分辨率到原生动态分辨率，解决了什么问题？">面试问题-从固定分辨率到原生动态分辨率，解决了什么问题？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

视觉编码器的第一条演进主线，是从"把任意图缩到固定尺寸"走向"按原始尺度处理"。Qwen-VL 系列是这条线最完整的样本。

**Qwen-VL（2023）用的是固定分辨率 + 位置感知适配器。** 视觉编码器初始化自 OpenCLIP 的 ViT-bigG，输入分辨率固定为 224、后提到 448；它的特色是一个位置感知的视觉-语言适配器——用单层 cross-attention 配 256 个可学习 query，把变长图像特征压成固定的 256 个视觉 token，并加 2D 绝对位置编码保留空间信息。问题在于，固定分辨率会对非标准长宽比的图造成缩放形变，对文档、票据这类高分辨率密集文本丢失细节。

**Qwen2-VL（2024）引入朴素动态分辨率（Naive Dynamic Resolution）。** 它能处理任意分辨率、动态产出可变数量的视觉 token，从根本上按原比例保留原始信息；同时移除 ViT 的绝对位置编码、改用 2D-RoPE。ViT 后接一个简单 MLP，把相邻 $2\times2$ 的 token 合并为一个（例如 $224\times224$ 图经 patch=14 编码后压缩为 66 个 token），并用 `<|vision_start|>`、`<|vision_end|>` 标记视觉段，ViT 规模约 675M。

**核心结论是：固定分辨率的形变与细节丢失，被动态分辨率彻底解决。** 而动态分辨率又反过来要求位置编码必须能适配可变网格，这正是 2D-RoPE 替换绝对位置编码的直接动因（详见专题二"位置编码与长序列建模"）。

<h4 id="面试问题-Qwen2.5-VL为什么要做Window-Attention与RMSNorm/SwiGLU等高效化改造？">面试问题-Qwen2.5-VL为什么要做Window-Attention与RMSNorm/SwiGLU等高效化改造？</h4>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐ (3/5)**

动态分辨率解决了"看清"，但直接后果是大图会产生大量 patch，ViT 全注意力的 $O(n^2)$ 成本随之失控。Qwen2.5-VL（2025）从零训练原生动态分辨率 ViT，把高效化落到了结构层面。

**Window Attention 把复杂度从二次压到近线性。** Qwen2.5-VL 的 ViT 里仅 4 层是全注意力、其余全部为窗口注意力，最大窗口 $8\times8$（即 $112\times112$）；小于该尺寸的区域保留原始尺度、不做填充。大部分层只在局部窗口内算注意力，计算量随 patch 数线性增长，仅少数全注意力层保留全局视野。

**RMSNorm 与 SwiGLU 让 ViT 与 LLM 结构对齐。** 早期 ViT 与 LLM 在归一化、激活函数上各用一套，联合训练时结构异构、工程复杂。Qwen2.5-VL 让 ViT 也采用 RMSNorm 与 SwiGLU，与语言侧一致，简化了整体网络、便于联合优化。

这两项改造的共同逻辑是：**在保住动态分辨率"看得清"的前提下，把大图带来的算力与工程代价压回可控区间。**

<h4 id="面试问题-DeepStack多层次特征融合如何在不增加token的情况下提升细粒度？">面试问题-DeepStack多层次特征融合如何在不增加token的情况下提升细粒度？</h4>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

演进到 Qwen3-VL（2025），瓶颈从"效率"转向"特征质量"：传统做法只取 ViT 最后一层特征送入 LLM，会丢失中间层的细节信息。DeepStack 就是针对这一点的多层次特征融合机制（patch 尺寸也由 14 调整为 16）。

**DeepStack 的做法是跨层抽取、投影、残差注入。** 从 ViT 的多个不同深度层（如 $l_1, l_2, l_3$）分别抽 patch 特征，各自经独立的 MLP merger 投影到 LLM 隐藏维度，再以残差相加注入 LLM 的前若干层（如第 1–3 层）。关键在于，这种注入是把不同层的视觉特征叠加进对应的 LLM 早期层隐藏状态，**不增加序列长度**——既引入了多层视觉语义，又不带来额外 token 开销，因此它同时是"提质"和"控 token"两件事。

**为什么多层融合能提升细粒度与图文对齐？** 根源在于 ViT 不同深度编码的粒度不同：浅层更多保留边缘、纹理、局部结构等低层细节，深层更偏类别、语义等高层抽象。只用末层会丢细节；多层融合让模型同时拿到低层细节与高层语义，对文档、小目标、密集排布等任务的细粒度感知与 grounding 精度直接受益。而且在 LLM 早期层就注入丰富视觉信息，视觉与文本能更早建立紧密对应，图文对齐质量随之改善。

需要注意一个副作用：由于 DeepStack 把视觉特征融进了 LLM 的早期隐藏状态，Qwen3-VL 的语言部分不再是纯文本通路，而是从早期层起就携带视觉信息。

**Qwen-VL 视觉编码器演进对照：**

| 代次 | 分辨率处理 | 位置编码 | 效率/结构关键点 | ViT 规模 |
|---|---|---|---|---|
| Qwen-VL | 固定 224/448 | 2D 绝对位置编码 | cross-attn 压成 256 token | ViT-bigG |
| Qwen2-VL | 朴素动态分辨率 | 2D-RoPE | MLP 合并 $2\times2$ token | 约 675M |
| Qwen2.5-VL | 原生动态分辨率（从零训） | 2D-RoPE | Window Attention + RMSNorm/SwiGLU | 高效原生 ViT |
| Qwen3-VL | 原生动态分辨率 | 2D-RoPE | DeepStack 多层融合、patch 14→16 | — |

<h3 id="2.高分辨率输入下视觉token压缩的演进路线是怎样的">2.高分辨率输入下视觉token压缩的演进路线是怎样的</h3>

<h4 id="面试问题-视觉token爆炸的根源是什么？早期的固定比例压缩方案怎么做？">面试问题-视觉token爆炸的根源是什么？早期的固定比例压缩方案怎么做？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

高分辨率与 token 数量是一对天然矛盾，InternVL-1.5 把它暴露得最典型。

**爆炸的根源是分块。** 标准 ViT 输入只有 224 或 336，对文档、表格、票据这类密集文本场景，整张图被压成两三百个 token、每个字只占几个像素，根本读不清——这是 OCR/DocVQA 类任务的命门。InternVL-1.5 的解法是动态高分辨率分块：按长宽比把图切成 $1\sim40$ 个 $448\times448$ 子图块（训练用 $1\sim12$ 块、推理零样本放大到 40 块），等效支持到 4K。但每个 $448\times448$ 子图过 ViT（patch=14）得到 $(448/14)^2 = 1024$ 个 patch token，40 块不压缩就是约 4 万个视觉 token；注意力 $O(n^2)$ 复杂度下，这个量级会显著吃显存、挤占 LLM 上下文。

**第一代解法是固定比例的空间下采样。** InternVL-1.5 对每个子图块的 patch 特征做一次 $2\times2$ 的 space-to-depth 操作：把 $2\times2$ 空间块折叠进通道维，空间分辨率减半、通道翻 4 倍，token 数降到原来的 $1/4$——一个 $448\times448$ 子图从 1024 压到 256 token，40 块从约 4 万降到约 1 万。（注：官方早期文档写作 "pixel shuffle"，实为笔误，该操作是下采样的 pixel unshuffle，OpenGVLab 已在仓库更正。）配套还有两点：长宽比自适应分块（从候选长宽比里选最接近原图的，减少形变与无效块）+ 一张全局缩略图保留整体上下文；再加上持续学习增强的 InternViT-6B 打底，让压缩后单 token 的语义密度更高。

**这类方案的共同局限是"一视同仁"。** Qwen2-VL 的 MLP 合并 $2\times2$、InternVL 的 Pixel Unshuffle，压缩比都固定为 $1/4$，对信息密集区与背景区用同样的力度压——这正是下一代内容自适应压缩要解决的问题。

<h4 id="面试问题-从固定压缩到内容自适应，ViR与系统级DvD分别解决了什么？">面试问题-从固定压缩到内容自适应，ViR与系统级DvD分别解决了什么？</h4>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

到 InternVL-3.5，token 效率被拆成"算法侧自适应压缩"和"系统侧解耦部署"两条腿，分别是 ViR 与 DvD。

**ViR（Visual Resolution Router，视觉分辨率路由）按内容逐块决定压缩率。** 它评估每个视觉块的语义信息量——文字、OCR 等信息密集区保留高分辨率（更多 token），背景、纯色等简单区施加高压缩（更少 token）。这与早期只看图像宽高决定切几块本质不同：ViR 看的是"这块内容值不值得多花 token"。集成方式很轻，通过一个轻量训练阶段 ViCO（Visual Consistency Learning，视觉一致性学习）接入，无需重训主干；效果是视觉 token 最多减约 50%，而文档/OCR 基准几乎无精度损失。

**DvD（Decoupled Vision-Language Deployment，解耦视觉-语言部署）从部署层面提速。** 把 ViT + MLP（+ ViR）部署在"视觉服务器"GPU、把 LLM 单独部署在"语言服务器"GPU；视觉端批量处理图像、产出紧凑特征，再以 BF16 经 TCP（可选 RDMA 提速）单向传给语言端融合解码。整个过程组织成"视觉处理 → 特征传输 → 语言处理"的异步三段流水线、相互重叠，重点优化预填充（prefilling）阶段的阻塞，单靠 DvD 就能带来近 2× 吞吐提升。

**两者一软一硬、叠加增益明显。** ViR 减少 token 数，既缩短语言端序列长度，又减少视觉端到语言端的特征传输量；DvD 重叠视觉/语言计算，让被压缩后的流水线进一步并行化。二者结合相比 InternVL-3 实现最高 4.05× 推理加速。从 Pixel Unshuffle 的固定 $1/4$ 到 ViR 的按需压缩、再到 DvD 的系统级解耦，这条效率演进线的方向很清晰：**从"无差别省"走向"按内容省 + 按系统并行"。**

<h3 id="3.多模态位置编码RoPE家族如何统一图文视频并支持外推">3.多模态位置编码RoPE家族如何统一图文视频并支持外推</h3>

<h4 id="面试问题-M-RoPE的三分量设计是怎样的，如何统一图文视频？">面试问题-M-RoPE的三分量设计是怎样的，如何统一图文视频？</h4>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐⭐ (5/5)**

多模态位置编码的起点问题是：传统 1D-RoPE 只能编码一维序列位置，无法表达图像的二维空间与视频的时间维。M-RoPE（Multimodal Rotary Position Embedding）是 Qwen2-VL 给出的标准答案，也是该系列几乎必问的考点。

**M-RoPE 的设计原理是把旋转位置编码分解为三个分量：时间（temporal）、高（height）、宽（width）。** 它将隐藏维度按 `mrope_section` 切分为三块，分别对时间、高、宽三个轴施加各自的旋转频率，从而用一套编码同时表达"第几帧、第几行、第几列"。

**它靠"给不同模态分配不同 ID 规则"来统一三种模态：**

- **文本：三分量取相同 ID，退化为 1D-RoPE。** 时间、高、宽 ID 完全相同并逐 token 加 1，此时 M-RoPE 功能上等价于传统 1D-RoPE，完整保留语言模型的原有行为。
- **图像：时间 ID 恒定，高/宽 ID 按位置赋值。** 同一张图内所有视觉 token 时间 ID 不变，高、宽分量按 token 在网格中的行列位置分别赋 ID。
- **视频：时间 ID 逐帧递增，高/宽与图像一致。** 视频被视为帧序列，时间分量随帧递增，高、宽沿用图像赋值方式。
- **跨模态衔接：以上一模态的最大位置 ID 加 1 初始化下一模态。** 混合输入时每种模态从前一模态最大 ID 加 1 开始，保证整段序列位置连续。

**一个附带收益是有利于长序列外推。** 图像/视频的位置 ID 数值被显著压低（不再像把二维/三维强行展平成一维那样线性累积），推理时更容易外推到更长序列。

<h4 id="面试问题-MRoPE-Interleave的交错设计带来了哪些作用？">面试问题-MRoPE-Interleave的交错设计带来了哪些作用？</h4>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

M-RoPE 之后，Qwen3-VL 的 MRoPE-Interleave（Interleaved-MRoPE）解决的是"频率在各轴之间分配不均"的问题。

**问题出在分块。** 早期 M-RoPE 把隐藏维度按时间、高、宽分块（chunked）切分——前一段维度全给时间、中间给高、后面给宽。结果时间分量集中在某一段（偏低频）维度，导致模型对时间的建模被局限在局部频段。

**MRoPE-Interleave 改为按通道交错分配三个轴的频率。** 它不再整段切分，而是让各通道循环轮流承担时间、高、宽（如 $t, h, w, t, h, w, \dots$），使全频段在三个轴上均匀分配。这样每个注意力头与通道都能同时观察到时间与空间信息，位置表达更一致、频率利用更充分。对纯文本数据，该方案退化为标准 RoPE（非时间轴置零），完整保留预训练语言模型的先验。

**相比分块式 M-RoPE，交错式的收益集中在四点：**

- **更强的长时程视频推理。** 时间频率贯穿全频段而非局限一段，跨越长时间的事件也能保持稳定时序建模。
- **更好的时空一致性与跨模态泛化。** 每个通道同时感知时间与空间，图像、视频、grounding 等任务的时空对齐更稳健。
- **更长的上下文外推。** Interleaved-MRoPE 位置 ID 增长比普通 RoPE 更慢，长度外推只需较小缩放因子；Qwen3-VL 原生支持 256K 上下文、可借 YaRN 扩到约 1M。
- **保留纯文本能力。** 对纯文本退化为标准 RoPE，几乎零工程代价地保护语言原有能力。

需要补充的是，位置编码这条线并非 Qwen 独有：GLM-4.5V 采用分段 [8, 12, 12] 的 3D-RoPE/MRoPE（分别对应时间、高、宽），InternVL-3 则用 V2PE（可变视觉位置编码）支撑更长的多模态上下文——三家殊途同归，都在把"多轴 + 外推友好"作为多模态位置编码的共同取向。

**多模态位置编码方案对照：**

| 方案 | 代表模型 | 核心设计 | 主要收益 |
|---|---|---|---|
| 2D-RoPE | Qwen2-VL ViT | 二维相对位置 | 适配可变网格、利于外推 |
| M-RoPE | Qwen2-VL | 时间/高/宽分块三分量 | 统一图文视频、压低位置 ID |
| MRoPE-Interleave | Qwen3-VL | 三轴频率通道交错 | 长视频、外推、时空一致 |
| 3D-RoPE / MRoPE | GLM-4.5V | 分段 [8,12,12] 多分辨率 | 强化空间与时序建模 |
| V2PE | InternVL-3 | 可变视觉位置编码 | 支撑更长多模态上下文 |

<h3 id="4.复杂文档理解如何依赖长上下文并支撑超长序列">4.复杂文档理解如何依赖长上下文并支撑超长序列</h3>

<h4 id="面试问题-复杂文档为何依赖长上下文？长序列又会带来哪些挑战？">面试问题-复杂文档为何依赖长上下文？长序列又会带来哪些挑战？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐ (3/5)**

长上下文是位置编码之外、支撑复杂文档理解的另一根支柱，GLM-4.5V 是这方面的代表。

**复杂文档天然依赖长上下文。** 多页 PDF、研究报告、含表格与图表的长文，本质上要求模型在单次输入内同时容纳大量文本与多张高分辨率页面图像，并在其间建立跨页、跨模态的引用关系，这只能靠很长的上下文窗口承载。

**长序列带来的挑战集中在两点：**

- **token 预算极易被突破。** 高分辨率页面图本身就消耗大量视觉 token，多页叠加再加正文文本，序列长度迅速膨胀，很容易超出常规上下文窗口。
- **算力与位置编码的双重压力。** 注意力 $O(n^2)$ 复杂度下序列越长开销越大；同时位置编码能否在远超训练长度时保持稳定外推，直接决定长序列建模质量——这也是专题内 RoPE 外推能力被反复强调的原因。

---

<h4 id="面试问题-长序列建模有哪些多层支撑设计？">面试问题-长序列建模有哪些多层支撑设计？</h4>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐ (3/5)**

GLM-4.5V 从基座、位置编码、计算结构、输入适配四个层面共同支撑长序列，这也是一套可复用的通用配方。

- **长上下文基座。** 基于原生支持长上下文的 GLM-4.5-Air，官方支持 64K 上下文，架构序列长度上限可达 131072（即 128K），为容纳多页文档提供底层窗口。
- **3D-RoPE / MRoPE 位置编码。** 分段 [8, 12, 12] 的多分辨率旋转位置编码，对时间、高、宽分别建模；旋转位置编码利于长序列外推稳定性，并在图文交错长序列中保持一致的相对位置语义。
- **MoE 控制长序列成本。** 106B 总参中每 token 仅激活约 12B，使长序列推理的计算开销相对可控，让"长窗口"在工程上可负担。
- **高分辨率与任意宽高比适配。** 支持最高 4K 分辨率与任意宽高比，配合视觉 token 的压缩处理（呼应专题一），在"看清文档细节"与"序列不超限"之间取得平衡。

四者结合，使模型能够在长上下文下对复杂文档做连贯的跨页、跨模态理解。作为对比，Qwen3-VL 走的是另一条相同目标的路径：靠 Interleaved-MRoPE 的慢速位置 ID 增长 + YaRN，把原生 256K 外推到约 1M——**长序列能力最终都落在"基座窗口 + RoPE 外推 + 稀疏激活控本"这三件事上。**

<h3 id="5.多模态模型如何用统一语言接口产出框与掩码等结构化输出">5.多模态模型如何用统一语言接口产出框与掩码等结构化输出</h3>

<h4 id="面试问题-框坐标文本化与SEG-token路由到下游解码器是怎么做的？">面试问题-框坐标文本化与SEG-token路由到下游解码器是怎么做的？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐ (3/5)**

多模态模型要用一个模型覆盖检测、定位、分割等数百种任务，而非为每类任务训一个专家模型。InternVL-2 的统一靠两条互补机制。

- **框坐标：直接当作文本 token 生成。** 视觉定位（grounding）被重述为"看图写坐标"——给定描述句，模型以纯文本形式输出归一化的边界框坐标（官方评测 prompt 即 `Please provide the bounding box coordinates of the region this sentence describes: {}`）。这里没有任何检测专用 head，框就是语言序列里的几个数字 token，由 LLM 自回归生成。
- **掩码/图像等结构化输出：特殊 token 路由到下游解码器。** 借助 VisionLLMv2 式设计，InternVL-2 把 MLLM 与多个下游任务解码器相连：模型生成一个特殊 token（如 `[SEG]`），取该 token 在 LLM 最后一层的隐藏状态作为"提示向量"，送入掩码解码器（如 SAM/SAM-2）生成分割掩码。后续 Sa2VA 正是用 InternVL-2/2.5 + SAM-2 的 `[SEG]` token 实现图像/视频指代分割，且梯度可经 `[SEG]` 回传到 MLLM。

**本质是一个"语言接口"统领多任务：** 要么把答案直接写成坐标文本，要么吐出特殊 token 去驱动专用解码器——两种方式都不破坏"以 LLM 为核心、自回归生成"的统一范式。

---

<h4 id="面试问题-统一序列/路由token相比专用head有哪些优劣？">面试问题-统一序列/路由token相比专用head有哪些优劣？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐ (3/5)**

把输出统一成"序列 + 路由 token"，相比传统"每任务一个专用 head"，优势集中在通用性与可扩展性。

- **一个模型、数百任务，免去逐任务改结构。** 新增任务往往只需调整指令与数据，不必新设计输出头并重训，天然契合"指令驱动"的多模态助手范式。
- **充分复用 LLM 的语言与推理能力。** 坐标、类别、关系都以语言形式表达，检测/定位因此能借力 LLM 的常识与上下文理解（例如"穿红衣服那个人"这种需要语义消歧的指代）。
- **跨任务知识共享。** 检测、描述、问答共享同一套主干，任务间可互相增益而非彼此割裂。

**代价也很明确。** 其一，纯文本坐标的精度上限有限——逐 token 生成坐标在像素级精确度上通常不及专用检测器；其二，`[SEG]` 路由方案需要额外挂载并训练下游解码器，并非零成本的统一。正因为"纯文本坐标精度有限"，坐标到底怎么表示，就成了下一个主问题的焦点。

<h3 id="6.视觉定位的坐标怎么表示才准">6.视觉定位的坐标怎么表示才准</h3>

<h4 id="面试问题-归一化、千分制、绝对坐标三种表示如何对比选型？">面试问题-归一化、千分制、绝对坐标三种表示如何对比选型？</h4>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

同样是"把框写成文本"，三家选了三种坐标表示，背后是不同的取舍。

- **归一化坐标（早期通用做法，如 InternVL-2）。** 把坐标缩放到 $[0,1]$ 相对区间，与图像尺寸解耦。好处是数值范围稳定、跨分辨率一致；代价是丢失"物体实际有多大、相距多远"的真实尺度信息。
- **千分制相对坐标（GLM-4.5V）。** 边界框为四元组 $[x_1, y_1, x_2, y_2]$，由左上角与右下角构成，每个数值按图像宽（对 $x$）或高（对 $y$）归一化后乘以 1000。它与绝对像素解耦、对任意分辨率与宽高比保持一致语义，同时用整数千分值兼顾精度与可解析性；并用 `<|begin_of_box|>`、`<|end_of_box|>` 两个特殊 token 包裹坐标区段（内括号样式可为 `[]`、`[[]]`、`()`、`<>` 等），使坐标在输出序列中边界清晰、便于解析与训练。
- **绝对位置坐标（Qwen2.5-VL）。** 直接用图像真实像素尺度表示检测框、点等坐标，不做归一化。配合原生动态分辨率（图像按原尺度转为可变 token），模型训练时见到的坐标与图像真实尺度一致，从而直接学习真实尺度信息。

**绝对坐标在目标检测上的独特优势在于保留尺度与空间关系：**

- **直接学习真实尺度与空间关系。** 归一化会丢失物体真实大小与间距，绝对坐标保留真实像素尺度，对检测与定位尤为关键。
- **与原生分辨率天然一致。** Qwen2.5-VL 本就按原始尺度处理输入，绝对坐标与之直接对齐，避免"输入按原尺度、标签却被归一化"的尺度错配。
- **更利于细粒度定位。** 小目标、密集排布场景中，绝对坐标对位置刻画更直接，有助于提升框的精度。

其代价是数值范围随图像尺寸变化、需要模型适应不同量级的坐标取值；Qwen2.5-VL 用原生分辨率训练与大规模定位数据来消化这一点。

**三种坐标表示对比：**

| 表示方式 | 代表模型 | 与分辨率关系 | 尺度信息 | 主要代价 |
|---|---|---|---|---|
| 归一化 $[0,1]$ | InternVL-2 | 解耦 | 丢失真实尺度 | 尺度信息缺失 |
| 千分制 ×1000 | GLM-4.5V | 解耦、语义一致 | 相对为主 | 仍是相对量 |
| 绝对像素 | Qwen2.5-VL | 直接对齐原尺度 | 保留真实尺度 | 数值量级随图变化 |

<h4 id="面试问题-IoU奖励、特殊token与高分辨率如何共同保证定位精度？">面试问题-IoU奖励、特殊token与高分辨率如何共同保证定位精度？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐ (3/5)**

"能输出框"不等于"框得准"。定位精度由训练目标、输出规范与输入条件三者共同保障，GLM-4.5V 是较完整的样本。

- **可验证奖励直接对齐 IoU。** 在强化学习中，grounding 任务以 IoU（交并比）作为奖励信号，让模型朝"框得更准"优化；配合带课程采样的难度调度，使定位训练难度持续匹配模型能力（详见专题六）。
- **标准化输出抑制抽取误差。** 通过统一坐标格式与明确答案标记（如 `<|begin_of_box|>` 包裹），降低坐标被错误抽取的概率，同时规避 reward hacking——用取巧方式骗过校验器而非真正定位。
- **高分辨率输入提供感知前提。** 支持任意宽高比与最高 4K 分辨率，使小目标、密集排布与精细边缘得以保留。定位精度的上限首先取决于"是否看得清"，高分辨率输入是细粒度定位的必要条件（呼应专题一的分辨率与压缩权衡）。

此外，GLM-4.5V 把定位纳入"先思考、后作答"的推理范式：给定含目标描述的 prompt，模型先 step-by-step 推理（thinking）再输出边界框，而非简单直接回归坐标——这让复杂指代（需要语义消歧的目标）也能被稳定定位。

<h3 id="7.视频输入如何在token预算内保留时序信息">7.视频输入如何在token预算内保留时序信息</h3>

<h4 id="面试问题-3D-patch时序分块与动态FPS采样如何保留时序信息？">面试问题-3D-patch时序分块与动态FPS采样如何保留时序信息？</h4>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

视频比图像多了时间维，直接逐帧编码会让 token 数量随帧数线性爆炸。Qwen2.5-VL 把"动态分辨率"从空间维扩展到时间维，用两项设计压住 token 又保住时序。

**3D patch（时序分块）把时间维一并纳入 patch 划分。** 做法是把相邻连续帧绑定为一个时序 patch（temporal patch size = 2，即每两帧合并），patch 形状为时间 × 高 × 宽（如 $2\times14\times14$）。视频不再逐帧独立编码，而是以"时空小块"为单位编码，既压缩了时间维的 token 数量，又显式保留了短时的时序信息。

**动态 FPS（Frames Per Second）采样让模型适应不同帧率。** 训练时以变化的帧率对视频采样，并把 M-RoPE 的时间维 ID 直接与绝对时间对齐。模型据此能通过"时间维 ID 之间的间隔"感知时间的快慢节奏，从而在不同 FPS 采样率的视频之间保持一致的时间对齐。之所以需要它：其一，真实视频帧率各异，固定帧率难以兼顾长短视频与快慢动作；其二，仅按"帧序号"编码时间（如 Qwen2-VL）无法反映内容变化速度或事件绝对时刻，把时间 ID 与绝对时间对齐后模型能理解"事件发生在第几秒"，且这一对齐不引入额外计算开销。

两者配合，3D patch 降低时间维冗余、动态 FPS + 绝对时间对齐保证时间语义，共同支撑在可控 token 预算下处理较长视频与做时刻定位。

<h4 id="面试问题-3D卷积编码器与逐帧编码有什么区别？">面试问题-3D卷积编码器与逐帧编码有什么区别？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐ (3/5)**

除了 patch 层面的时序分块，编码器结构本身也在往"显式建模时序"演进。

**逐帧编码只做空间理解。** 早期做法把视频当成一串独立图像，每帧各自过 ViT 提空间特征，帧与帧之间的运动、序列变化要靠后续 LLM 或位置编码去补——时序信息是"事后拼"出来的。

**3D 卷积编码器直接在编码阶段捕捉时序。** GLM-4.5V 用专门的 3D 卷积编码器处理视频，卷积核在时间维上滑动，能直接捕捉运动、序列变化等时序特征，而非仅做逐帧空间编码。这与 Qwen2.5-VL 的 3D patch 是同一目标的两种落点：一个在 patch 划分上绑定相邻帧，一个在卷积结构上跨帧感受——**共识是视频理解必须让时序信息"在进入 LLM 之前"就被编码进特征，而不是全部丢给 LLM 事后推断。**

<h3 id="8.视频事件的时刻如何被精确表达">8.视频事件的时刻如何被精确表达</h3>

<h4 id="面试问题-从帧索引时间对齐到文本时间戳对齐经历了怎样的演进？">面试问题-从帧索引时间对齐到文本时间戳对齐经历了怎样的演进？</h4>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐ (3/5)**

时序建模的第二层问题是"事件到底发生在第几秒"，Qwen3-VL 的文本时间戳对齐是这条线的最新一步。

**此前是把时间藏在位置编码里。** T-RoPE 把时间信息编码在位置编码的时间分量中、与帧索引（绝对时间对齐的帧位置）绑定。时间是"隐式"的，模型只能靠位置 ID 的相对间隔去推断时刻。

**文本时间戳对齐（Text-Timestamp Alignment）把时间显式写成文本。** 实现方式是：在每组视频帧之前插入一个显式的文本时间戳 token（如 `<3.0 seconds>` 或 `<00:03:00>`），用标准的文本嵌入机制编码。也就是说，时间信息不再仅依赖位置编码，而是被显式写成可读的文本标记，直接锚定每段画面对应的时刻。训练时还会交替使用不同时间戳格式（秒、时:分:秒等），增强模型对多种时间表达的鲁棒性。

从"隐式位置分量"到"显式文本标记"，本质是把时间从模型内部的编码细节，提升为可读、可比较、可运算的一等信息。

<h4 id="面试问题-显式文本时间戳带来了哪些时序推理好处？">面试问题-显式文本时间戳带来了哪些时序推理好处？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐ (3/5)**

把时间从"隐式位置编码"改为"显式文本标记"，带来更精确、更灵活的时序能力。

- **帧级精确的事件定位。** 显式时间戳把每段画面与具体时刻直接关联，使模型能做到帧级精度的事件定位与秒级视频索引，而非仅靠位置 ID 的相对间隔推断。
- **更强的时序推理。** 时间以可读文本参与建模，模型可以像处理普通文本一样对时刻做比较、计算与表述，时序推理更直接。
- **格式灵活、表达统一。** 支持秒、时:分:秒等多种格式并在训练中交替，模型既能理解也能按不同格式输出时间，适配多样的下游需求。
- **实现简洁。** 复用标准文本嵌入即可注入时间信息，无需为时序定位额外设计专用结构或位置编码分量。

把专题四串起来看：3D patch / 3D 卷积解决"时序特征怎么进模型"，动态 FPS + 绝对时间对齐解决"快慢节奏怎么感知"，文本时间戳对齐解决"绝对时刻怎么精确表达"——三层叠加，才有对长视频的高效理解 + 秒级定位。

<h3 id="9.视觉能力是事后缝合还是原生习得">9.视觉能力是事后缝合还是原生习得</h3>

<h4 id="面试问题-BLIP-2两阶段改装与InternVL-3原生多模态预训练有何本质区别？">面试问题-BLIP-2两阶段改装与InternVL-3原生多模态预训练有何本质区别？</h4>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐⭐ (5/5)**

预训练范式的根本分歧只有一句话：**视觉能力是原生长出来的，还是事后缝上去的。** 这是该专题几乎必问的考点。

**BLIP-2 是典型的"事后改装（retrofit）"路线。** 它冻结现成的视觉编码器、冻结现成的 LLM，只训中间的 Q-Former 桥接模块（约 188M），且分两阶段（视觉-语言表示学习 → 视觉-语言生成学习）。LLM 自始至终没在预训练中原生接触过视觉，视觉信息只是作为软提示被外挂注入。

**InternVL-3 是"原生（native）"路线。** 不需要桥接模块、不需要事后对齐，视觉能力在预训练阶段就与语言能力一同习得。

**为什么原生路线上限更高？** 事后对齐天然存在"对齐债"——冻结的视觉空间与语言空间靠一个小模块强行拼接，细粒度交互与分布一致性都受限；原生预训练让两种模态从同一起点共同优化，避免这种系统性偏差。一个有力佐证是：得益于原生预训练，InternVL-3 的纯文本能力甚至超过了作为语言初始化的 Qwen2.5 基座，说明联合训练不仅没牺牲语言能力，反而有所增益。

**核心差异对照：**

| 维度 | BLIP-2（两阶段改装） | InternVL-3（原生预训练） |
|---|---|---|
| 视觉能力来源 | 预训练后外挂，事后对齐 | 预训练阶段原生习得 |
| 是否需桥接模块 | 需要 Q-Former（约 188M） | 不需要额外桥接/对齐模块 |
| 训练阶段 | 表示学习 → 生成学习两阶段 | 语言+多模态单阶段联合 |
| LLM 是否动 | 冻结 LLM | LLM 参与联合预训练 |
| 主要风险 | 模态鸿沟、对齐债 | 预训练成本高、需掌控基座 |
| 适用方 | 想低成本给现成 LLM 加视觉 | 有基座掌控权的头部团队 |

原生路线的代价是成本高昂：需要从预训练阶段起就投入大规模算力，且必须掌控语言基座，通常只有头部团队能承担。相较之下，BLIP-2 式"冻结两端 + 轻量桥接"在为现成强 LLM 低成本增加视觉能力这一目标上仍具性价比。从 BLIP-2 到 InternVL-3，体现的是多模态范式从"事后改装"向"原生一体"的演进。

<h4 id="面试问题-原生多模态预训练的数据配比与工程要点是什么？">面试问题-原生多模态预训练的数据配比与工程要点是什么？</h4>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

原生多模态预训练（Native Multimodal Pre-Training）的做法，是把语言预训练与多模态对齐合并进同一个预训练阶段。

**具体做法是在预训练阶段就交错喂入两类数据。** 大规模纯文本语料与多样的多模态数据（图文、视频-文本、图文交错序列）被混合在一起联合预训练，让模型从一开始就同时习得语言能力与多模态能力，而非事后嫁接。

**关键工程点是数据配比。** 异构数据混合的最优采样比不好定，InternVL-3 用两阶段策略确定：先分别在纯多模态、纯语言数据上训独立模型、各自找到最优内部配比，再在固定总预算下确定两者的相对比例。实验结论是语言:多模态 = 1:3 时整体最优；总训练量约 2000 亿 token（语言约 500 亿 + 多模态约 1500 亿）。

**配套设计还有两项：** 引入 V2PE（可变视觉位置编码）支撑更长的多模态上下文（呼应专题二）；后训练沿用 SFT + MPO（混合偏好优化）进一步强化对话与推理（呼应专题六）。原生预训练在数据侧的核心难点，就是把"语言"和"多模态"这两种异构分布用一个可控的配比调和好。

<h3 id="10.大模型的视觉预训练如何降本">10.大模型的视觉预训练如何降本</h3>

<h4 id="面试问题-渐进式缩放为何要先用小LLM训好视觉编码器再迁移？">面试问题-渐进式缩放为何要先用小LLM训好视觉编码器再迁移？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

原生预训练上限高但贵，如何把大模型的视觉预训练成本压下来，就是 InternVL-2.5 渐进式缩放（Progressive Scaling）要解决的。它的思路是"先用小 LLM 把视觉编码器训好，再无痛迁移到大 LLM"，而非一上来就用大 LLM 联合训练视觉端。

**执行流程分两步：**

- **第一步：小 LLM 对齐视觉编码器（Stage 1.5）。** 把 InternViT 与一个资源友好的小 LLM（如 20B）联合训练，集中优化"基础视觉能力 + 跨模态对齐"，刻意避开"直接用大 LLM 训视觉"的高昂算力。
- **第二步：迁移到大 LLM、跳过重训。** 借助共享权重机制，把上一步训好的 InternViT 直接迁移到大 LLM（如 72B）上，无需重训视觉端；训大模型时 Stage 1.5 可整段跳过，直接复用已优化的 InternViT。

**这套做法成立的关键观察是：** 即便 ViT 与 LLM 是用 next-token-prediction 损失联合训练得到的，学出的视觉特征仍是"可泛化的通用表示"，能被其它 LLM 直接看懂。正因为视觉特征不与某个特定 LLM 强绑定，"换大脑不换眼睛"才行得通。

**降本效果非常直观：** InternVL2.5-78B 全程只用约 1200 亿 token，相比 Qwen2-VL 的 1.4 万亿 token 不到其 1/10。降本来自三点——避开"大 LLM × 视觉"的昂贵联合训练；视觉编码器一次训成、跨规模复用消除冗余；配套的数据打包（select-search-pack-maintain）提升 GPU 利用率。其前提是视觉特征可跨 LLM 迁移，若目标大 LLM 的嵌入空间差异较大，迁移后可能仍需少量再对齐——它节省的是重训视觉端的成本，而非完全免除对齐。

<h4 id="面试问题-Qwen-VL三阶段训练范式及各阶段数据策略是怎样的？">面试问题-Qwen-VL三阶段训练范式及各阶段数据策略是怎样的？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

作为对比，Qwen-VL 确立的是更经典的"预训练 → 多任务预训练 → 监督微调"三阶段范式，三阶段在"训练哪些模块、用什么数据、达到什么目标"上层层递进。

- **阶段一：预训练（Pretraining）。** 冻结 LLM，只训练视觉编码器与适配器，建立基础图文对齐、让视觉特征进入语言空间；使用低分辨率（如 224）提升吞吐。数据以海量图文对为主、来自网络爬取，规模庞大但标注质量参差，组织形式以"图 → 文"为主，目的是用规模换对齐。
- **阶段二：多任务预训练（Multi-task Pretraining）。** 解冻全部参数、联合训练整个模型，并将分辨率提升（如 448）。数据覆盖图像描述、视觉问答、视觉定位、OCR、文本生成、图文交错等多任务，从"单一图文对"转为"多任务混合"，是能力广度的主要来源。
- **阶段三：监督微调（Supervised Fine-Tuning）。** 冻结视觉编码器、训练 LLM 与适配器，在高质量指令与多轮对话数据上微调（规模相对小但质量高），塑造指令跟随与交互行为，得到 Qwen-VL-Chat。

三阶段的逻辑是"先对齐、再多任务、后对齐人类指令"，数据策略从"大而粗"逐步走向"小而精"。**把两条降本/训练路线放在一起看：渐进式缩放优化的是"视觉编码器一次训成、跨规模复用"，三阶段范式优化的是"模块解冻顺序 + 数据由粗到精"——前者省的是规模扩展的钱，后者定的是能力构建的序。**

<h3 id="11.后训练如何在能力与对齐之间取平衡">11.后训练如何在能力与对齐之间取平衡</h3>

<h4 id="面试问题-SFT加DPO两阶段组合与冻结ViT是出于什么考量？">面试问题-SFT加DPO两阶段组合与冻结ViT是出于什么考量？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

后训练的第一类范式是"监督微调打能力 + 偏好优化调对齐"，Qwen2.5-VL 是标准样本，两阶段均冻结视觉编码器（ViT）、只更新语言侧。

- **SFT 阶段负责"会做"。** 在多样多模态数据上做监督微调，数据涵盖图文对、视频、纯文本，来源包括通用 VQA、拒绝采样（Rejection Sampling）以及文档/OCR、Grounding、视频、Agent 等专项数据集；文本与多模态较为均衡（约各占一半、规模在百万量级）。它让模型掌握各类任务技能与指令跟随的基本格式，决定能力边界。
- **DPO 阶段负责"做得合人意"。** SFT 只能学到"标准答案"，却难以表达"两个都对的回答里哪个更好"。DPO 仅在图文与纯文本数据上进行，用偏好对（preferred/rejected）直接优化模型对"更优回答"的倾向，且每个样本只处理一次以保证效率；无需训练独立奖励模型与复杂在线采样，工程上比 PPO 类方法更轻量、更稳定。

**冻结 ViT 的考量在于保护与省钱。** 后训练阶段视觉感知能力已在预训练中建立，冻结 ViT 既能保护视觉表示不被少量后训练数据扰动，又能降低训练开销，让后训练聚焦于语言侧的任务适配与偏好对齐。这套"SFT + DPO"组合以较低成本完成"任务能力 + 偏好对齐"双重目标，是当前多模态后训练的常见范式。

<h4 id="面试问题-RLCS带课程采样如何支撑以推理为中心的目标？">面试问题-RLCS带课程采样如何支撑以推理为中心的目标？</h4>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

当后训练目标从"对齐偏好"升级为"提升推理"时，就需要更强的强化学习。GLM-4.1V-Thinking 以"推理为中心（reasoning-centric）"，用 RLCS 释放基座潜力——团队认为基座潜力几乎设定了最终性能上界，先靠大规模预训练打造强视觉基座，再用 RLCS 全面提升跨域能力（STEM 解题、视频理解、内容识别、代码、grounding、GUI 智能体、长文档理解等）。

RLCS（Reinforcement Learning with Curriculum Sampling，带课程采样的强化学习）把课程学习引入在线采样，让训练样本难度持续匹配模型当前能力，从而最大化每次更新的信息量。

- **难度分级：离线 + 在线双重评估。** 训练前用多个成熟 VLM（或早期 RL checkpoint）在全量数据上跑 pass@k，并融合专家人工难度标注，把数据切成由易到难的多个难度档；训练中对每个 rollout 记录 pass@k 并映射到对应难度档，与离线标签合并，实时反映当前水平。
- **采样再加权：聚焦中等难度。** 按训练迭代粒度持续重加权各难度类别——下采样过于简单与当前过难的样本，加大中等难度样本曝光，因为模型在这一区间收益最大。
- **奖励体系：可验证为主、人类反馈为辅。** RLVR（可验证奖励）对不同域采用不同校验：数学用 sympy 符号校验、grounding 用 IoU、开放式问题用 LLM 判分，并通过明确答案标记规避 reward hacking；主观或不可验证域则用 RLHF；多域数据按预设比例混合，以 GRPO 目标优化。
- **训练稳定性工程。** 关闭 KL 与熵惩罚、采用大批量并对 rollout 动态打包、使用 per-sample loss（比 per-token 更稳）；以 EMA 做动态批量扩展保持效率；对过长 CoT 注入答案标记，引导更短、更聚焦的输出。

一个重要工程教训是：奖励基础设施必须足够稳健——任一单域校验器的缺陷都可能毒化乃至拖垮整个 RL 训练，这在实验中被明确观察到。

<h3 id="12.强化学习如何既稳又能突破上界">12.强化学习如何既稳又能突破上界</h3>

<h4 id="面试问题-级联RL从离线到在线，为何优于单阶段？">面试问题-级联RL从离线到在线，为何优于单阶段？</h4>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

RL 的核心两难是"要么不稳定、要么被静态数据封顶"。InternVL-3.5 的级联式强化学习（Cascade RL）用"由粗到细"的两阶段化解这一两难。

- **第一阶段：离线 RL（Offline RL）做粗对齐。** 采用 MPO（Mixed Preference Optimization，混合偏好优化），在预先收集好的偏好数据上训练。离线数据无需实时采样，收敛稳定、成本低，负责把模型快速带到一个不错的初始策略附近。
- **第二阶段：在线 RL（Online RL）做精对齐。** 采用 GSPO 等在线算法，从当前策略实时采样生成数据再优化。在线数据质量更高、更贴合模型当前分布，负责在最优解附近做精细打磨。

**为什么级联优于单阶段？** 因为离线与在线各有致命短板、恰好互补：

- **纯在线 RL 不稳、昂贵、冷启动差。** 全程实时采样对超参与初始策略极敏感，容易震荡甚至崩溃；从未对齐策略直接在线训，冷启动阶段样本质量低、效率差、算力消耗巨大。
- **纯离线 RL 被静态数据封顶。** MPO/DPO 这类离线方法稳定便宜，但能力上界被预收集数据死死限制，模型无法通过与环境交互"自我进化"。
- **级联的取舍是离线打底（稳）+ 在线精修（高）。** 先用离线 RL 廉价、稳定地把策略带到优质初始点，再用在线 RL 在最优解附近精细提升，既规避在线冷启动的不稳，又突破离线静态数据的上界。

这套粗到细策略在下游推理基准上带来显著提升，整体推理性能相比 InternVL-3 最高提升约 +16%（如 MMMU、MathVista）。把三家放一起看，后训练/RL 这条线的演进很清晰：**从 Qwen2.5-VL 的 SFT+DPO（对齐偏好），到 GLM-4.1V 的 RLCS（难度自适应提升推理），再到 InternVL-3.5 的级联 RL（离线稳 + 在线破上界）——目标从"合人意"一路走向"更能推理且训练可控"。**

<h4 id="面试问题-可验证奖励RLVR如何做多域校验并防范reward-hacking？">面试问题-可验证奖励RLVR如何做多域校验并防范reward-hacking？</h4>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐ (3/5)**

无论 RLCS 还是级联 RL，奖励信号是否可靠都是训练成败的命门，这就是 RLVR（可验证奖励，Reinforcement Learning with Verifiable Rewards）要解决的。

**RLVR 的核心是给不同域配不同的自动校验器。** 数学题用 sympy 做符号校验、grounding 用 IoU 判定框准不准、开放式问题用 LLM 判分——每个域都尽量用"可自动验证对错"的信号取代人工打分，从而让 RL 能大规模、低成本地跑起来。

**reward hacking 是最需要防范的失败模式。** 模型可能学会用取巧方式骗过校验器，而非真正把任务做对（例如输出恰好能通过格式校验但语义错误的答案）。主流防范手段有两条：一是明确的答案标记与标准化输出格式，把"正确答案"和"格式技巧"解耦，降低被钻空子的空间；二是校验器本身要足够稳健——GLM-4.1V 的实验明确观察到，任一单域校验器的缺陷都可能毒化乃至拖垮整个 RL 训练，因此奖励基础设施的工程质量，和算法本身同等重要。

对不可验证的主观域（如文风、有用性），则退回 RLHF 用人类反馈补位。RLVR + RLHF 的组合，本质是"能自动验证的用可验证奖励、验证不了的才用人类反馈"，在成本与覆盖面之间取平衡。

<h3 id="13.训练数据如何治理才能不毒化推理">13.训练数据如何治理才能不毒化推理</h3>

<h4 id="面试问题-重复生成的成因是什么？为何对CoT推理尤其致命？">面试问题-重复生成的成因是什么？为何对CoT推理尤其致命？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐ (3/5)**

数据工程直接决定模型的行为质量，重复生成是最典型的"数据毒化"案例。InternVL 团队在研发中观察到一个反直觉现象：哪怕训练集中只混入极小比例的异常样本，也足以在推理时诱发这种异常行为。

**成因在于模型会忠实模仿训练数据里的"病态模式"。** 重复生成（repetitive generation）指模型推理时陷入"循环复读"，反复输出相同词句或段落。网络数据中本就存在大量重复内容（模板化文本、复读的字幕、堆砌的关键词），模型一旦学到这种分布，就可能在生成时复现循环。

**为何对 CoT 推理尤其致命？** 长链思维（Chain-of-Thought）和长文本生成的输出序列很长，重复模式会被逐步放大：一旦在中途陷入某个循环，整条推理链就此跑偏、无法收敛到正确答案。因此 InternVL-2.5 明确指出，重复生成在长文本与 CoT 推理任务上危害最大——这也是它把数据清洗当作一等公民的直接动机。

<h4 id="面试问题-三道闸数据过滤流水线如何抑制重复生成？">面试问题-三道闸数据过滤流水线如何抑制重复生成？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐ (3/5)**

InternVL-2.5 设计了一条三道闸的数据过滤流水线，专门压制异常样本、尤其是重复样本。

- **① LLM 质量打分（LLM-Based Quality Scoring）。** 用预训练 LLM 配合领域专用 prompt 给每条样本打 $0\sim10$ 分，低于阈值（如 7 分）直接剔除，保证整体数据质量底线。
- **② 重复检测（Repetition Detection）。** 用 LLM prompt 专门标记重复样本，低于更严格阈值（如 3 分）的予以排除；高质量学术数据集可豁免该流程、避免误杀，被标记样本再走人工复核。
- **③ 启发式规则过滤（Heuristic Rule-Based Filtering）。** 用规则检测"异常句长、重复行"等显式异常，配合人工验证后移除；多模态侧也用类似规则检测视觉异常。

论文报告，这套流水线显著降低了异常行为发生率，尤其在 CoT 推理任务上带来明显提升。概括而言，它把重复生成的治理落实为"LLM 打分 + 重复检测 + 规则过滤 + 人工复核"的数据工程闭环，核心判断是：少量异常数据即可显著毒化推理行为，必须在训练前完成清洗。这与专题五 InternVL-3 的"数据配比"、专题六的"奖励校验器稳健性"共同构成一条主线——**数据与信号的质量，往往比算法本身更能决定多模态模型的行为上限。**

<h3 id="14.多模态模型如何从感知走向可执行动作">14.多模态模型如何从感知走向可执行动作</h3>

<h4 id="面试问题-原生多模态工具调用如何弥补文本式工具调用的不足？">面试问题-原生多模态工具调用如何弥补文本式工具调用的不足？</h4>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

多模态模型的下一步，是从"看懂"走向"能执行"，即多模态智能体。GLM-4.6V 的原生多模态工具调用是这条线的代表。

**先看传统文本式工具调用的不足。** 它为纯文本场景设计：要么由 VLM 先把图像转成文本描述、再以文本驱动工具，要么用"纯文本 LLM + 旁挂视觉模型"的级联管线。套到多模态智能体场景会暴露三类问题：

- **输入侧信息损失。** 图像、截图、文档页被迫先转成文本描述才能进入工具调用，版式、细节与视觉证据大量丢失。
- **输出侧无法消费视觉返回。** 工具返回的统计图表、渲染网页截图、检索商品图本身是图像，纯文本模型无法理解，难以纳入后续推理。
- **感知与动作割裂。** "看懂"与"执行"分属不同模型或环节，管线复杂、误差逐级累积，形成明显的感知-动作鸿沟（perception-action gap）。

**GLM-4.6V 用原生多模态工具调用（native multimodal tool calling）打通感知与动作：**

- **多模态输入：图像直接入参。** 图像、截图、文档页可直接作为工具参数传入，无需预先转文本，最大限度避免信号损失、简化管线。
- **多模态输出：视觉理解工具返回。** 模型能视觉理解工具返回的结果（搜索结果、统计图表、渲染网页截图、检索商品图等），并纳入后续推理链与最终输出。
- **闭合"感知 → 理解 → 执行"循环。** 同一模型完成图像优先（image-first）的工具输入输出，支撑视觉网页搜索、含图表的文档问答、图文混排内容创作等复杂任务。
- **训练方式。** 把工具调用行为纳入统一的强化学习目标，对齐模型在复杂工具链中的任务规划、指令遵循与格式遵循能力。

在规模上，GLM-4.6V 采用 128K 多模态上下文训练，并提供 GLM-4.6V（106B）与 GLM-4.6V-Flash（9B）等版本，使其能以单一模型完成"图像优先"的端到端智能体流程，而非依赖多模型拼接。

<h4 id="面试问题-设计到代码如何实现像素级复现与视觉反馈闭环？">面试问题-设计到代码如何实现像素级复现与视觉反馈闭环？</h4>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

"设计到代码（design to code）"是多模态智能体最具代表性的落地任务，GLM-4.6V 在这上面把"感知 → 生成 → 自我校正"完整跑通。

**像素级复现（Pixel-Level Replication）解决"截图变代码"。** 用户上传一张网页截图或设计稿，模型识别其中的布局、组件与配色方案，进而生成高保真的 HTML/CSS/JS 代码，不仅能复刻单屏，还支持长网页生成与多轮迭代。支撑它的关键训练设计包括：大规模长上下文图文数据的持续预训练（熟悉真实网页版式与代码的对应关系）、视觉-语言压缩对齐（受 Glyph 工作启发，提升对密集视觉信息的高效表示）、十亿级多模态世界知识数据与智能体合成数据训练（丰富对界面元素与交互模式的先验）。

**交互式编辑 + 视觉反馈闭环解决"持续迭代与自我校正"：**

- **交互式编辑（Interactive Editing）。** 用户可在生成页面的截图上圈选某个区域并给出自然语言指令（如"把这个按钮左移并改成深蓝色"），模型自动定位到对应代码片段并修改。其本质是把视觉 grounding 能力（呼应专题三）作用于"截图区域 ↔ 代码片段"的映射，将视觉指代转化为精确的代码定位与编辑。
- **视觉反馈闭环（Visual Feedback Loop）。** 受 UI2Code$^N$ 工作启发，该闭环让模型产出代码 → 系统渲染 → 截图回灌给模型 → 模型检视渲染结果并与目标比对 → 自我纠正与精修代码，从而保证布局与视觉一致性。
- **训练方式。** 在强化学习中针对该闭环优化，目标是支持多步、视觉对齐的修正，而非一次性生成，进而验证"自我改进多模态智能体"的可行性。

这两项机制让前端能力从"截图生成代码"升级为"可持续迭代、自我校正"的视觉编程，更贴近真实开发中的多轮修改流程。至此七个专题形成闭环：**从看清（专题一）、定位时空（专题二~四）、到训练范式（专题五~六）与数据治理（专题七），最终收束到"感知-动作闭环"的多模态智能体——这正是主流多模态模型正在共同奔赴的方向。**
